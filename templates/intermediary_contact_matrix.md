## 十、会议安排模板

### 10.1 周例会（Weekly Coordination Meeting）

| 属性 | 内容 |
|:---|:---|
| **时间** | 每周三 14:00-15:00（或双周） |
| **形式** | 发行人会议室 + 线上（腾讯会议/Teams/Zoom） |
| **参会方** | 保荐人项目组 + 发行人IPO协调团队 + 各中介日常联络人 |
| **议程** | ①上周年报事项回顾；②本周工作安排；③DD Issue进展情况；④风险与问题；⑤下周计划 |
| **输出** | 会议纪要（24h内发出） + 行动项跟踪表 |

### 10.2 专题会（Thematic Meeting）

| 专题类型 | 触发条件 | 参会方 | 时长 |
|:---|:---|:---|:---:|
| 审计协调会 | 审计进场/审计调整讨论/年报审计启动 | 保荐人财务组 + 审计师 + 发行人CFO | 60-90' |
| 法律尽调会 | 法律尽调启动/重大法律问题讨论 | 保荐人法律组 + 律师 + 发行人法务 | 60' |
| 估值讨论会 | 估值模型搭建/估值假设讨论 | 保荐人行业组 + 行业顾问 + 发行人CFO | 60' |
| 内核准备会 | 内核前1周 | 保荐人全体 + 各中介主管合伙人 | 120' |
| 反馈回复会 | 收到监管反馈后48h内 | 保荐人 + 发行人 + 各中介 | 90' |
| 定价会（IPO） | T-1日 | 保荐人MD + 发行人CEO/CFO + 承销团代表 | 60' |

### 10.3 内核会（Internal Review Committee Meeting）

| 属性 | 内容 |
|:---|:---|
| **频率** | 按项目进展（通常申报前、重大反馈回复前各1次） |
| **参会方** | 保荐人内核委员 + 项目团队 + 各中介主管合伙人 |
| **议程** | ①项目基本情况汇报；②尽调发现和重大问题；③内核委员质询；④表决 |
| **输出** | 内核意见（书面） + 整改要求 + 内核通过/暂缓/否决结论 |

## 十一、通讯录更新流程

### 11.1 更新触发条件

```
触发更新的情形：
  · 任何中介团队成员发生变动（离职/换人/新增）
  · 任何发行人对接人发生变动
  · 电话号码/邮箱/微信变更
  · 新增中介机构（如增加行业顾问/财经公关）
  · 项目阶段转换（如从尽调阶段进入发行阶段，联系人可能有变化）
```

### 11.2 更新频率要求

| 更新类型 | 频率 | 负责人 |
|:---|:---|:---|
| 定期更新 | 每周五（与项目周报同步发出更新版） | 保荐人执行负责人 |
| 临时更新 | 人员变动后24h内 | 保荐人执行负责人 + 变更方联系人 |
| 重大更新 | 项目阶段转换时 | 保荐人执行负责人 |

### 11.3 通讯录分发规则

```
分发方式：
  · 邮件附件（PDF版 + Excel版）
  · 项目共享文件夹（最新版覆盖）
  · 项目微信群（通知"通讯录已更新至VXX"）

权限管理：
  · 🔴 最高密级：含个人手机号/微信号的完整版 → 仅项目核心成员
  · 🟡 中等密级：不含个人联系方式的机构版 → 项目全体成员
  · 🟢 公开版：仅机构名称+角色 → 可根据需要扩大分发范围
```

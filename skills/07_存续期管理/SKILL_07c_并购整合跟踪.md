# SKILL_07c：并购整合跟踪

## 一、概述

并购重组完成后，独立财务顾问需要在业绩承诺期内持续跟踪标的资产的经营和业绩实现情况，关注商誉减值风险，监测整合进展。本技能定义跟踪框架和关键指标体系。

---

## 二、并购整合评估框架（100天计划）

### 2.0 整合阶段总览

并购整合的"100天计划"是业界公认的最佳实践框架，将收购后整合划分为四个阶段：

```
阶段划分：

Day 1 - Day 30  │ 文化融合与稳定期
                │ 目标：稳定军心、传递整合愿景、防止人才流失

Day 30 - Day 60 │ 组织融合与重构期
                │ 目标：建立新组织架构、统一报告线、关键岗位到位

Day 60 - Day 100│ 业务融合与协同期
                │ 目标：实现客户/供应商/IP的初步整合

贯穿全程       │ 财务融合
                │ 目标：统一会计政策、财务系统对接、关联交易清理
```

### 2.1 第一阶段：文化融合（Day 1 - Day 30）

```
核心目标：稳定人心，传递愿景，防止核心人才流失

关键行动：

1. Day 1：各层级沟通启动
   ├── CEO致全体员工公开信（说明收购意图、整合愿景、对员工的承诺）
   ├── 管理层全体会议（由收购方CEO/并购负责人主讲）
   ├── 部门级沟通会（传达具体影响和安排）
   └── 建立"整合答疑通道"（专属邮箱/微信群，回应员工关切）

2. Day 1-7：愿景共识
   ├── 明确整合后的企业使命、愿景、价值观
   ├── 识别两方文化的差异点（决策风格、沟通方式、激励导向）
   └── 制定文化融合的"双方互学"计划（而非单方面"改造"）

3. Day 1-14：管理团队融合
   ├── 联合领导力工作坊（收购方+标的方关键管理层）
   ├── 明确管理层级和角色
   └── 建立整合指导委员会（Integration Steering Committee）

4. Day 1-30：人才保留
   ├── 识别关键人才（Key Person List，通常≤标的公司10%员工）
   ├── 一对一沟通（了解个人关切、职业规划）
   ├── 制定保留方案（如果涉及股权激励调整）
   └── 启动竞业限制/保密协议审查和补充签署

5. Day 15-30：文化差异问卷与分析
   ├── 向标的公司员工发放文化诊断问卷
   ├── 分析文化差异的核心维度（权力距离、个人/集体主义等）
   └── 形成文化融合行动计划

交付物：
  □ 整合沟通计划
  □ 关键人才名单及保留方案
  □ 文化诊断报告
  □ 整合指导委员会会议纪要
```

### 2.2 第二阶段：组织融合（Day 30 - Day 60）

```
核心目标：搭建新组织架构，统一报告线和管控体系

关键行动：

1. Day 30-40：组织架构调整
   ├── 新组织架构图发布
   ├── 明确的汇报线和层级关系
   ├── 各部门职责边界重新划分（注意避免"真空地带"）
   └── 冗余岗位识别和人员优化方案（如需）

2. Day 30-45：报告线整合
   ├── 统一KPI考核体系（将标的公司纳入上市公司绩效管理体系）
   ├── 统一财务报告口径（月报/季报格式对齐）
   ├── 统一审批权限表（什么层级可以审批什么金额/事项）
   └── 建立信息报送制度（定期向上市公司报告经营情况）

3. Day 40-50：关键岗位补位
   ├── 新任命管理层（董事长/总经理/CFO等由收购方派驻或联合任命）
   ├── 内审/合规岗位建立
   ├── 缺失岗位的外部招聘启动
   └── 过渡期双轨制（原管理团队暂留配合交接）

4. Day 45-60：制度建设
   ├── 统一核心管理制度（采购、销售、人力、财务）
   ├── 内控制度建设（如标的公司原本内控薄弱）
   └── 合规培训（防腐败、信息安全、出口管制等）

交付物：
  □ 新组织架构图
  □ 审批权限表
  □ 统一管理制度文件包
  □ 关键岗位到位确认表
```

### 2.3 第三阶段：业务融合（Day 60 - Day 100）

```
核心目标：实现客户、供应商、IP等核心业务要素的初步整合

关键行动：

1. Day 60-75：客户交接与整合
   ├── 客户关系梳理（重叠客户/交叉销售机会分析）
   ├── 客户沟通计划（告知收购不影响服务质量/产品供应）
   ├── 销售团队整合（区域/行业重新划分）
   ├── 统一客户信用管理政策
   └── CRM系统对接

2. Day 65-80：供应商接口统一
   ├── 供应商关系梳理（重叠供应商识别、集中采购机会分析）
   ├── 采购协同启动（合并采购量，重谈合作价格）
   ├── 供应商统一管理（准入标准、评审体系对齐）
   └── 供应链系统的初步对接

3. Day 70-90：知识产权（IP）整合
   ├── 双方IP资产全面盘点
   ├── 识别IP互补和交叉许可机会
   ├── 统一IP管理制度（专利申请/维护/维权）
   └── 技术团队融合（研发路线对齐、避免重复投入）

4. Day 75-100：业务系统互通
   ├── IT系统互联互通评估
   ├── 优先打通关键系统（邮件、ERP、CRM）
   ├── 统一通讯和协作平台（邮件域名归一等）
   └── 制定中长期系统全面整合的路线图

交付物：
  □ 客户交叉销售机会分析报告
  □ 采购协同节约预估报告
  □ IP资产整合清单
  □ 系统互通实施计划
```

### 2.4 贯穿全程：财务融合

```
贯穿100天全程的财务融合任务：

Day 1-15：
  ├── 财务团队初步对接
  ├── 会计政策差异识别（收入确认、折旧政策、存货计价等）
  └── 财务系统初步评估

Day 15-45：
  ├── 统一会计政策（制定统一会计政策手册）
  ├── 统一财务科目（统一科目表Chart of Accounts）
  └── 关联交易识别和清理方案制定

Day 45-70：
  ├── 财务系统切换准备（数据迁移方案、接口测试）
  ├── 合并报表编制流程建立
  └── 统一预算编制模板和流程

Day 70-100：
  ├── 首期合并财务报表试编
  ├── 关联交易定价公允性审查
  └── 统一资金管理制度（资金池/资金归集）

交付物：
  □ 统一会计政策手册
  □ 统一科目表（Chart of Accounts）
  □ 关联交易清理方案
  □ 首次合并报表试编底稿
```

## 三、整合成效KPI体系

### 3.1 财务指标

| 指标 | 计算方法 | 目标设定 | 数据来源 | 跟踪频率 |
|:---|:---|:---|:---|:---:|
| **收入协同** | 交叉销售收入 + 新渠道/新区域收入 | 按整合方案中的协同目标 | 销售系统/财务系统 | 月度 |
| **成本协同** | 采购成本节约 + 人员优化节约 + 运营效率提升 | 基于基准年，逐年递减（如第一年节约20%、第二年50%） | 财务系统 | 季度 |
| **毛利率改善** | 整合后毛利率 - 整合前加权利润率 | ≥整合方案预期 | 财务报表 | 季度 |
| **期间费用率** | （销售+管理+研发+财务费用）÷ 营业收入 | 不高于整合前或逐年降低 | 财务报表 | 季度 |
| **经营活动现金流** | 经营现金净流量 ÷ 净利润 | >1（净利润质量好） | 财务报表 | 季度 |
| **ROIC改善** | 税后净营业利润 ÷ 投入资本 | ≥并购时的预期回报 | 财务报表 | 年度 |
| **协同实现率** | 实际协同金额 ÷ 计划协同金额 | ≥80%（第一年），≥100%（第二年起） | 专项分析 | 季度 |

### 3.2 运营指标

| 指标 | 目标 | 预警线 | 数据来源 | 跟踪频率 |
|:---|:---|:---|:---|:---:|
| **客户流失率** | <5% | >10% | CRM系统 | 月度 |
| **核心人员留任率** | >90% | <80% | HR系统 | 月度 |
| **新客户获取率** | 同比增长 | 同比下降>10% | CRM系统 | 季度 |
| **业务系统打通进度** | 按计划推进 | 延期>30天 | IT项目进度 | 月度 |
| **供应商整合率** | 集中采购占比>50% | <30% | 采购系统 | 季度 |
| **产能利用率** | >85% | <70% | 生产系统 | 月度 |
| **员工满意度** | 同比不下降 | 下降>10% | HR调查 | 半年 |

### 3.3 对赌执行指标

| 指标 | 计算方法 | 警戒标准 | 数据来源 |
|:---|:---|:---|:---|
| **业绩承诺完成率** | 当年实际 ÷ 当年承诺 | <80%（单年）或 <100%（累计） | 年度审计报告 |
| **累计业绩完成率** | 累计实际 ÷ 累计承诺 | <100% | 年度审计报告 |
| **补偿覆盖率** | 承诺方质押股份市值 ÷ 潜在补偿金额 | <150%（须关注） | 证券市场/财务分析 |
| **商誉减值风险** | 含商誉资产组可回收金额 vs 账面价值 | 可回收金额 < 105%账面价值 | 商誉减值测试 |

## 四、对赌协议执行跟踪

### 4.1 业绩承诺基本框架

| 要素 | 说明 |
|-----|------|
| 承诺期限 | 通常3年（部分项目4-5年） |
| 承诺口径 | 扣非归母净利润（最常见） |
| 承诺金额 | 逐年增长型为主（部分阶梯型） |
| 补偿方式 | 股份补偿（优先）→现金补偿（补充） |
| 补偿计算 | （累计承诺净利润 - 累计实现净利润）÷ 累计承诺净利润 × 交易对价 |
| 减值测试 | 业绩承诺期满后，对标的资产进行减值测试，减值额 > 已补偿额时追加补偿 |

### 4.2 年度跟踪流程

```
第一步：获取标的公司年度审计报告
第二步：计算当年度实际实现的承诺口径净利润
第三步：对比承诺金额，计算完成率
第四步：判断是否触发补偿义务
第五步：出具专项核查意见（独立财务顾问+会计师）
第六步：如触发补偿 → 督促承诺方履行补偿 → 上市公司公告
第七步：更新业绩承诺跟踪档案
```

### 4.3 业绩承诺完成率判断

| 完成率 | 状态 | 处理 |
|--------|------|------|
| ≥ 100% | ✅ 达标 | 正常推进 |
| 90% ~ 100% | ⚠️ 接近达标 | 关注下一期完成情况 |
| 80% ~ 90% | 🟡 关注 | 分析原因，评估全年和累计完成预期 |
| < 80% | 🔴 风险 | 立即核查原因，评估对商誉减值的影响 |

**累计口径优先**：如当期完成率<100%但累计完成率≥100%，通常不触发当期补偿，但需密切关注剩余承诺期。

### 4.4 业绩承诺未达标时的补偿计算

```
补偿方式确定流程：

步骤1：判断承诺方是否持有足够股份
  ├── 持有足够股份 → 优先以股份补偿
  └── 股份不足 → 不足部分以现金补偿

步骤2：股份补偿计算
  应补偿股份数 = (累计承诺净利润 - 累计实现净利润) ÷ 累计承诺净利润 × 交易对价 ÷ 发行价
  · 精确到股，不足一股的向上取整
  · 上市公司以1元总价回购应补偿股份并注销

步骤3：现金补偿计算
  应补偿现金 = (累计承诺净利润 - 累计实现净利润) ÷ 累计承诺净利润 × 交易对价 - 已补偿股份价值
  · 承诺方在收到通知后30日内支付

步骤4：减值测试追加补偿（业绩承诺期满后）
  标的资产期末减值额 > 已补偿总金额 → 承诺方追加补偿差额
```

### 4.5 业绩承诺实现情况专项核查报告

**报告结构**：

```
一、本次重组基本情况
二、业绩承诺及补偿安排
三、标的资产当年业绩实现情况
  - 承诺净利润 vs 实际实现净利润
  - 完成率及差异原因分析
四、标的资产累计业绩实现情况
五、关于是否需要补偿的结论
六、保荐人/独立财务顾问核查意见
```

### 4.6 业绩承诺未达标的常见原因

| 原因 | 应对 |
|-----|------|
| 行业下行（外部系统性风险） | 区分系统性vs标的管理层责任 |
| 客户流失 | 核查流失原因，评估持续性 |
| 成本上升超预期 | 分析成本结构，评估可优化空间 |
| 新项目/新产品不及预期 | 核查研发进展和市场反馈 |
| 整合效果不佳 | 进入整合风险分析流程 |

### 4.7 仲裁/诉讼预案

```
业绩承诺纠纷的常见焦点：

1. 业绩认定口径争议
   预案：协议中明确约定计算口径（包括非经常性损益的定义、会计政策变更的处理等）
         → 如发生争议，优先以审计师意见为准（协议约定）

2. 承诺方无力履行补偿义务
   预案：要求质押承诺方足够数量的上市公司股份（≥预计补偿金额的150%）
         → 要求承诺方在协议签订时即提供担保（质押股份/资产抵押）
         → 如股份不足又不支付现金 → 准备启动法律程序（冻结资产）

3. 承诺方主张"不可抗力/情势变更"
   预案：协议中明确约定不可抗力的范围和排除事项
         → 行业正常波动不构成不可抗力
         → 发生争议时，立即委托律师出具法律意见

4. 发起仲裁/诉讼的决策标准：
   ① 补偿金额>交易对价20% → 启动法律程序
   ② 承诺方拒绝按协议约定履行（限定期限届满）→ 启动法律程序
   ③ 承诺方财务状况恶化导致履约能力丧失 → 紧急启动保全措施
```

## 五、商誉减值测试跟踪

### 5.1 商誉减值测试时间节点

| 时间 | 事项 |
|-----|------|
| 每年年末（12月31日） | 无论是否有减值迹象，必须进行年度商誉减值测试 |
| 业绩大幅低于承诺时 | 可能需要开展中期减值测试 |
| 行业/市场重大变化时 | 可能需要开展临时减值测试 |
| 并购后每季度 | 评估商誉减值迹象，判断是否需要启动临时减值测试 |

### 5.2 减值测试关注要点

- 测试所采用的资产组/资产组组合是否合理
- 未来现金流预测假设是否合理（与评估时、上年测试时对比）
- 折现率是否合理（与评估时使用的折现率对比）
- 可比公司选择是否恰当
- 测试结论是否与标的业绩表现一致

### 5.3 商誉减值迹象识别

| 减值迹象 | 触发条件 | 核查建议 |
|---------|---------|---------|
| 业绩大幅低于承诺 | 实际完成率 < 80% | 必须进行减值测试并分析 |
| 行业政策重大不利变化 | 影响持续经营能力 | 立即开展减值评估 |
| 核心客户/供应商丢失 | 流失占比 > 30% | 分析现金流预测需下调幅度 |
| 核心团队流失 | 关键人员离职 | 评估经营持续性影响 |
| 核心技术被替代 | 技术路线重大变革 | 分析技术贬值影响 |
| 产品价格大幅下滑 | >20%或毛利率下降>10pct | 下调盈利预测 |
| 可比公司市值大幅下跌 | 可比公司PE<标的估值时的PE | 下调估值倍数 |
| 汇率/地缘政治重大变化 | （海外收购）经营环境恶化 | 调整现金流预测假设 |

### 5.4 商誉减值测试的独立财务顾问核查

```
核查步骤：

1. 确认资产组/资产组组合划分的合理性
   · 是否包含了与商誉相关的协同效益
   · 是否与企业合并时的划分一致

2. 核查现金流预测的关键假设
   · 收入增长率（是否高于行业预期增长率）
   · 毛利率（是否与历史水平和行业趋势一致）
   · 资本支出（是否足以支撑预测的增长）

3. 核查折现率（WACC）
   · 无风险利率、贝塔系数、市场风险溢价的取值是否合理
   · 与上年测试时使用的折现率差异及原因

4. 核查减值测试结论
   · 如可回收金额 > 账面价值但仅略高（<110%）→ 关注
   · 如触发减值 → 核查减值金额的计算是否合理

5. 出具核查意见
```

## 七、重大不利变化监测

### 7.1 监测触发事项

以下事项一旦发生，需立即评估对标的资产持续经营能力和上市公司的影响：

| 触发事项 | 行动 |
|---------|------|
| 标的公司主要经营资质被吊销/到期未续 | 评估经营合法性风险 |
| 标的公司重大诉讼/仲裁/行政处罚 | 评估财务影响和声誉影响 |
| 标的公司核心客户/供应商终止合作 | 评估对收入和供应链的影响 |
| 标的公司核心技术/专利被无效/侵权 | 评估技术壁垒持续性 |
| 承诺方（业绩补偿义务人）财务状况恶化 | 评估补偿能力 |
| 上市公司对标的公司管理失控 | 录为值得特别关注风险 |

### 7.2 重大不利变化处置

1. 初步评估影响程度
2. 与上市公司、承诺方沟通
3. 督促上市公司进行信息披露
4. 出具独立财务顾问核查意见
5. 评估是否触发重组前置条款的MAC条款
6. 记录入跟踪档案

## 八、对赌期/或有对价管理

### 8.1 或有对价（Earnout）机制

部分重组可能约定，除基础对价外，根据标的公司未来业绩表现追加支付额外对价（earnout）：

- Earnout条件的逐条跟踪
- Earnout期间标的公司的行为约束（防止操纵业绩）
- Earnout支付金额的计算和确认

### 8.2 Earnout常见纠纷及预防

| 纠纷类型 | 预防措施 |
|---------|---------|
| 业绩认定口径争议 | 协议明确约定计算口径 |
| 会计政策变更影响 | 约定按原政策调整 |
| 承诺方对期间行为的争议 | 协议约定标的独立运营约束条款 |
| 支付时间的争议 | 约定审计报告出具后XX天内支付 |

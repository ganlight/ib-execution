## 二、核心原则

1. **三线独立路由** — bond/abs/equity 三线各有独立 SKILL.md 子入口，阶段文件各自独立
2. **七阶段统一骨架** — 三线共享相同的七阶段结构（启动→尽调→方案→撰写→申报→发行→存续）
3. **META通用叠加** — 16个META元技能三线通用，可叠加任何产品线使用
4. **共享参考分层** — references 按 bond/abs(shared)/equity/shared/compliance 五类管理
5. **阶段间数据契约** — 每阶段产出有明确格式，下一阶段以此为输入
6. **质量内建** — 每阶段有必过质量门槛，不通过不得进入下一阶段
7. **迭代收敛** — 文件从P0初稿到PN定稿，用 META-02 迭代工作流驱动

## 三、四层递进模型（IB版）

```
        ┌─────────────────────────────────────────────┐
  4层   │  方案语义 — 交易方案、产品选择、时间表         │  03 方案设计
        │  为什么选科创板？为什么定增不配股？           │  适用于三线
        ├─────────────────────────────────────────────┤
  3层   │  文件语义 — 招股书/募集书/重组报告/计划书     │  04 文件撰写  05 申报
        │  各章节约束、数据一致性、披露完整性            │  各线有专属文件类型
        ├─────────────────────────────────────────────┤
  2层   │  事实语义 — 尽调发现、财务数据、法律事实       │  02 尽职调查
        │  收入真实性？关联交易公允？诉讼风险？          │  三线DD各有侧重
        ├─────────────────────────────────────────────┤
  1层   │  准入语义 — 客户资质、利益冲突、基础资产筛查   │  01 项目启动
        │  这个客户能不能做？资产能不能证券化？          │  债券/ABS红线各异
        └─────────────────────────────────────────────┘
```

每一层是下一层的前提：准入→事实→文件→方案。上层犯错导致下层全部返工。

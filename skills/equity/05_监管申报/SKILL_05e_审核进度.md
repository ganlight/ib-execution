# SKILL_05f：审核进度追踪

## 一、概述

审核进度追踪是项目管理的核心工具，用于实时掌握项目在审核各阶段的推进状态，识别异常延迟，及时采取应对措施。本技能定义追踪维度、状态指标体系和分析方法，并提供 filing_tracker.py 的使用指引。

## 二、追踪维度

### 2.1 项目基本信息维度

| 维度 | 字段 | 说明 |
|-----|------|------|
| 项目编号 | project_id | 内部项目编号 |
| 发行人名称 | issuer_name | 发行人/上市公司全称 |
| 板块 | board | 主板/创业板/科创版/北交所 |
| 项目类型 | project_type | IPO/再融资/重组/债券 |
| 当前阶段 | current_stage | 见阶段状态定义 |

### 2.2 关键日期维度

| 里程碑 | 字段名 | 记录内容 |
|--------|--------|---------|
| 申报日 | filing_date | 向交易所提交申请文件日期 |
| 受理日 | acceptance_date | 交易所发出受理通知日期 |
| 首次问询发出日 | inquiry1_sent | 首轮问询函日期 |
| 首轮问询截止日 | inquiry1_deadline | 回复截止日期（法定时限） |
| 首轮回复实际日 | inquiry1_reply_actual | 实际提交回复日期 |
| 第二轮问询发出日 | inquiry2_sent | 如有 |
| 第二轮问询截止日 | inquiry2_deadline | 如有 |
| 第二轮回复实际日 | inquiry2_reply_actual | 如有 |
| 第三轮问询发出日 | inquiry3_sent | 如有 |
| 上会日 | hearing_date | 上市委/重组委会议日期 |
| 上市委通过日 | hearing_pass_date | 审议通过日期 |
| 提交注册日 | registration_submit | 报送证监会注册日期 |
| 注册决定日 | registration_approval | 获得注册批文日期 |
| 批文到期日 | approval_expiry | 批文有效期届满日 |
| 发行日 | offering_date | 发行上市日期（如已完成） |

### 2.3 审核轮次统计维度

| 维度 | 说明 |
|-----|------|
| 首轮问询问题数 | 反馈意见函总题数 |
| 第二轮问询问题数 | 如有 |
| 第三轮问询问题数 | 如有 |
| 首轮回复总页数 | 回复文件页数 |
| 问询重大问题类别 | 标注核心争议问题类型 |

## 三、状态指标

### 3.1 三色状态灯

| 状态 | 灯号 | 判断标准 | 处理动作 |
|-----|------|---------|---------|
| 🟢 正常 | 绿色 | 当前节点耗时 ≤ 同类项目平均耗时 × 1.2 | 保持关注，正常推进 |
| 🟡 关注 | 黄色 | 平均 × 1.2 < 当前节点耗时 ≤ 平均 × 1.5 | 核查原因，适度催促 |
| 🔴 风险 | 红色 | 当前节点耗时 > 平均 × 1.5 | 立即排查，升级汇报 |

### 3.2 状态灯触发逻辑（各节点分别计算）

| 节点 | 触发条件示例 |
|-----|------------|
| 受理→首轮问询 | 超30个工作日（主板）/ 20个工作日（科创/创业）未收到反馈 |
| 首轮回复已超时限 | 提交日期 > 截止日期（已申请延期除外） |
| 回复提交后久未出下轮 | 超30个工作日未收到下一轮问询 |
| 过会后久未提交注册 | 超3个月未提交注册 |
| 注册后久未获批文 | 提交注册后3个月+未获批 |

### 3.3 特殊状态

| 状态 | 标识 | 说明 |
|-----|------|------|
| 中止审查 | SUSPENDED | 因更换中介、补充审计、发行人配合等原因中止 |
| 终止审查 | TERMINATED | 撤回申请或被决定终止 |

## 四、历史对标分析

### 4.1 对标维度

对比同一板块、相近时间申报的同类项目的审核进度：

| 对标指标 | 对标口径 | 分析目的 |
|---------|---------|---------|
| 受理→上会天数 | 同类项目平均/中位 | 判断本项目的效率 |
| 问询轮次 | 同类项目模式 | 预期还需几轮 |
| 首轮回复后→二轮发出 | 同类项目间隔 | 判断审核节奏 |
| 总审核时长 | 同板块平均 | 判断是否正常 |

### 4.2 偏差分析模板

当本项目进度与对标数据出现偏差时：
```
偏差分析：
  - 节点：首轮问询→首轮回复
  - 本项目耗时：XX天
  - 同类平均耗时：XX天
  - 偏差：+XX天（+XX%）
  - 偏差原因：
    1. ...（内部因素）
    2. ...（外部因素）
  - 对总时间表的影响：预计总审核延长XX天
  - 应对建议：...
```

## 五、里程碑偏离预警

### 5.1 预警时间窗口

| 距期限时间 | 预警级别 | 行动 |
|-----------|---------|------|
| >30天 | — | 正常推进 |
| 15-30天 | 温和提醒 | 通知项目组注意时限 |
| 7-15天 | 黄色预警 | 项目负责人重点关注 |
| 3-7天 | 红色预警 | 每日跟踪进度，准备延期申请方案 |
| <3天 | 紧急 | 提交回答方案或延期申请 |

### 5.2 延期申请条件

- 反馈问题涉及重大事项需额外时间核查
- 涉及补充审计/评估
- 发行人重大事项变化需更新披露
- 更换中介机构（特殊情形）
- 每轮通常仅允许延期1次，最长1个月

## 六、filing_tracker.py 使用指南

### 6.1 工具功能

`filing_tracker.py` 是一个审核进度管理命令行工具，支持：
- 创建/更新项目审核进度记录
- 自动计算状态灯（绿/黄/红）
- 生成里程碑偏离预警
- 历史对标分析
- 导出进度看板

### 6.2 安装和初始化

```bash
# 安装依赖
pip install pandas openpyxl

# 初始化项目追踪
python filing_tracker.py init --project "XX科技IPO" --board "科创版"
```

### 6.3 更新进度

```bash
# 更新某个里程碑
python filing_tracker.py update --project "XX科技IPO" \
  --milestone acceptance_date \
  --date "2026-08-01"

# 更新状态
python filing_tracker.py update --project "XX科技IPO" \
  --current_stage "首轮问询回复中"
```

### 6.4 查看进度

```bash
# 查看项目进度摘要
python filing_tracker.py show --project "XX科技IPO"

# 查看所有项目进度看板
python filing_tracker.py dashboard

# 查看需要关注的项目（黄色/红色灯）
python filing_tracker.py alerts
```

### 6.5 历史对标

```bash
# 与同类项目对标
python filing_tracker.py benchmark \
  --project "XX科技IPO" \
  --board "科创版" \
  --period "2025-2026"

# 输出：对标分析报告（含本项目耗时、平均耗时、排名、状态灯）
```

### 6.6 数据导出

```bash
# 导出为Excel进度看板
python filing_tracker.py export --format xlsx --output "审核进度看板.xlsx"

# 导出为CSV
python filing_tracker.py export --format csv --output "进度数据.csv"
```

### 6.7 数据存储

追踪数据默认存储在 `filing_tracker_data/` 目录下的 SQLite 数据库中：

```
filing_tracker_data/
├── tracker.db           # SQLite数据库
├── projects/            # 各项目JSON文件
│   └── XX科技IPO.json
└── benchmarks/          # 历史对标数据
    └── 科创版_2024-2026.csv
```

## 七、进度追踪最佳实践

1. **及时更新**：每个里程碑达成后24小时内更新系统
2. **每周巡检**：每周一检查所有项目状态，更新三色灯
3. **月度报告**：每月生成审核进度月报
4. **提前预判**：在进入下一阶段前2周开始准备
5. **双人核对**：关键日期由两人独立确认
6. **对标学习**：每季度更新历史对标数据

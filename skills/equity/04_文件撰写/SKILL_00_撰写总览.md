# SKILL_04：文件撰写总览

## 一、概述

文件撰写是投资银行项目的核心产出环节。本阶段将前期的尽职调查成果转化为符合监管要求、逻辑严谨、数据准确的正式申报文件。文件质量直接影响审核进度和通过率。

## 二、各产品线文件体系

### 2.1 IPO 项目文件体系

| 文件类型 | 撰写方 | 审核方 | 关键要点 |
|---------|--------|--------|---------|
| 招股说明书（含申报稿、上会稿、注册稿） | 券商（主办） | 发行人、律师、会计师 | 全文核心文件，券商统筹 |
| 保荐人推荐书 | 券商 | 内核部门 | 保荐人执业质量声明 |
| 保荐工作报告 | 券商 | 内核部门 | 详述尽调过程和结论 |
| 审计报告（三年一期） | 会计师 | 券商、律师 | 财务数据基础 |
| 内部控制鉴证报告 | 会计师 | 券商 | 内控有效性 |
| 经注册会计师核验的非经常性损益明细表 | 会计师 | 券商 | 扣非净利润计算 |
| 法律意见书 | 律师 | 券商 | 合法合规性结论 |
| 律师工作报告 | 律师 | 券商 | 详细核查过程 |
| 发行人全体董事、监事和高级管理人员对发行申请文件的承诺 | 发行人 | 律师 | 真实性承诺 |
| 盈利预测报告（如需） | 会计师 | 券商 | 预测合理性 |

> **券商为核心统筹方**：券商主要负责招股说明书、保荐人推荐书、保荐工作报告。其他机构文件由券商审核把关。

### 2.2 并购重组项目文件体系

| 文件类型 | 撰写方 | 审核方 | 关键要点 |
|---------|--------|--------|---------|

| 备考财务报表及审阅报告 | 会计师 | 券商 | 合并后财务影响 |

## 三、中介机构分工与协作

### 3.1 券商职责

- **主办撰写**：招股说明书/重组报告书、保荐人推荐书、保荐工作报告/独立财务顾问报告
- **审核把关**：审核律师、会计师出具的所有专业报告
- **内部流程**：立项→内核→质控→申报

### 3.2 律师职责

- **独立核查**：发行人主体资格、历史沿革、资产权属、重大合同、诉讼仲裁
- **出具文件**：法律意见书、律师工作报告
- **协助券商**：审核招股书法律相关章节

### 3.3 会计师职责

- **独立审计**：三年/两年一期财务报表审计
- **出具文件**：审计报告、内控鉴证报告、非经常性损益核验、纳税情况鉴证等
- **协助券商**：审核招股书财务章节、提供盈利预测（如需）

### 3.4 评估师职责（并购项目）

- **独立评估**：标的资产评估/估值
- **出具文件**：评估报告/估值报告
- **协助券商**：审核重组报告书中评估相关信息

### 3.5 协调机制

- **券商为总协调人**：统一项目进度，管理文件接口
- **交叉审阅**：各机构间相互审阅，确保数据一致性
- **定期碰头会**：每周文件进度同步会，关键节点每日站会

## 四、起草原则

### 4.1 核心起草原则（按优先级排列）

1. **数据一致性第一**：所有文件中同一数据必须完全一致。招股说明书数据 → 审计报告数据 → 法律意见书引用数据，三线比对。
2. **定量先于定性**：能量化的内容必须量化。如"产能利用率85%"优于"产能利用率较高"。
3. **事实先于分析**：先陈述客观事实，再进行分析评论。事实引用需注明来源。
4. **逐章推进**：一次撰写一个完整章节，避免多章并行造成的逻辑断裂。
5. **TBD标注规范**：待补充内容统一用【TBD：内容描述-责任人-预计完成日期】标注。严禁出现无标注的空白数据。

### 4.2 TBD 标注标准

```
格式：【TBD：缺失内容简要描述-责任机构/人-预计补充日期】
示例：【TBD：2024年产能利用率数据-发行人-2026.06.15】
禁止：留空白、写"待补充"而无责任人和日期、用占位符"XXX"
```

### 4.3 文字风格统一

- 使用客观、中性语言
- 避免夸张、营销性表述
- 数据保留适当精度（金额：万元/亿元，比例：两位小数）
- 全文用语规范统一（注意"报告期"vs"最近三年"，"发行人"vs"公司"的一致性）

## 五、版本管理体系

### 5.1 版本编号规则

| 阶段 | 版本号 | 说明 |
|-----|--------|------|
| 初稿撰写 | v0.1 ~ v0.9 | 券商内部起草和修改 |
| 内部审核 | v1.0 ~ v1.9 | 内核部门审核修改 |
| 申报提交 | v2.0 | 正式申报稿 |
| 反馈回复 | v3.0 ~ v3.9 | 反馈意见回复修改 |
| 注册申请 | v4.0 | 注册稿 |
| 发行阶段 | v5.0 | 发行稿/最终稿 |

### 5.2 文件命名规范

```
格式：[文件类型]_[项目简称]_[阶段]_v[版本号].[日期].docx
示例：招股说明书_XX科技_申报稿_v2.0.20260601.docx
     反馈回复_XX科技_第一轮回复_v3.0.20260815.docx
```

## 七、常见问题与风险防范

1. **数据口径不一致**：财务数据在招股书、审计报告、法律意见书中出现不同版本。→ 建立单一数据源（以审计报告为准），所有引用交叉比对。
2. **事实性错误**：公司成立日期、注册资本、股权比例等基础信息错误。→ 与工商档案、营业执照逐一比对。
3. **描述性矛盾**：同一事实在不同章节描述相反或不同。→ 全文检索关键字，消除矛盾。
4. **时间节点错乱**：历史沿革时间线前后矛盾。→ 建立时间轴验证。

# META-18: AI辅助投行工作伦理准则

> 定位：在AI能力持续扩展的背景下，为投行工作确立AI使用的清晰边界和伦理准则。本准则需定期审视更新。

## 一、核心原则

> **AI是投行从业者的助手，不是替代者。所有最终决策和签字责任必须由具备相应资质的专业人员承担。**

### 1.1 四原则体系

```
            ┌────────────────────────────┐
            │      AI辅助投行四原则       │
            │                            │
            │  1️⃣ 辅助而不替代          │
            │  2️⃣ 安全而非冒险          │
            │  3️⃣ 透明而非隐蔽          │
            │  4️⃣ 迭代而非一次          │
            │                            │
            └────────────────────────────┘
```

| 原则 | 内涵 | 底线 |
|------|------|------|
| **辅助而不替代** | AI提供信息、整理、初稿，人做决策 | AI产出不得直接替代专业判断 |
| **安全而非冒险** | 不以效率为代价牺牲安全性 | 涉及客户信息时从严控制 |
| **透明而非隐蔽** | 对客户披露AI使用 | 不隐瞒AI的参与 |
| **迭代而非一次** | AI产出必须经人工审核和迭代 | 不直接使用AI生成的最终文件 |

## 二、AI使用边界

### 2.1 三区分类法

| 区域 | 定义 | 特征 | 审批要求 |
|------|------|------|----------|
| 🟢 **绿灯区** | 明确可使用AI | 信息检索、格式化、基础整理 | 无需特别审批 |
| 🟡 **黄灯区** | 需人工审核 | 涉及判断的内容 | 必须人工复核 |
| 🔴 **红灯区** | 严禁使用AI | 涉及法律责任/未公开信息 | 绝对禁止 |

### 2.2 分类详解

#### 🟢 绿灯区 — 明确可使用

| AI使用场景 | 示例 | 注意事项 |
|------------|------|----------|
| **信息检索** | 查找法规案例 | 需交叉验证关键信息 |
| **数据整理** | 表格生成、数据汇总 | 核对计算逻辑 |
| **格式规范** | 统一格式、生成模板 | 确认模板适用性 |
| **草案生成** | 写初稿框架 | 所有法律和财务内容需核实 |
| **案例检索** | 查找同类案例 | 确保案例背景相似 |
| **语言润色** | 优化表达 | 不得改变实质内容 |
| **多语言翻译** | 中英互译 | 专业术语复核 |

#### 🟡 黄灯区 — 需人工审核（须经签字人复核）

| AI使用场景 | 原因 | 审核要求 |
|------------|------|----------|
| **实质性判断建议** | 涉及专业判断 | 必须经承做人员复核确认 |
| **估值分析辅助** | 涉及定价决策 | 必须经估值团队审核 |
| **风险评级辅助** | 涉及风险判断 | 必须经风控确认 |
| **回复策略建议** | 涉及监管沟通 | 必须经项目负责人审核 |
| **方案对比分析** | 涉及业务判断 | 必须经团队讨论确认 |
| **行业分析辅助** | 数据和判断混合 | 必须交叉验证资料来源 |

#### 🔴 红灯区 — 绝对禁止

| 禁止场景 | 原因 |
|----------|------|
| ❌ **替代专业签字** | 法律责任须由持牌人员承担 |
| ❌ **直接用于申报文件最终版本** | 申报文件须经专业人员核实 |
| ❌ **涉及未公开信息推断** | 内幕信息边界不可逾越 |
| ❌ **未经审核的对外沟通** | 投行对客户/监管的沟通须经授权 |
| ❌ **生成具有法律效力的文本并直接使用** | 法律文本须经律师审核 |
| ❌ **进行具有法律后果的决策** | 决策责任清晰不可转嫁 |

## 三、保密要求

### 3.1 信息分级保护

| 信息级别 | 定义 | 处理规则 | 使用AI |
|----------|------|----------|--------|
| 🔐 **绝密** | 未公开重大信息 | 不得进入任何AI系统 | ❌ |
| 🔒 **机密** | 客户非公开信息 | 仅限脱敏后可用 | ⚠️（脱敏后） |
| 🔓 **内部** | 项目内部信息 | 脱敏处理后方可AI辅助 | ✅（脱敏后） |
| 📢 **公开** | 已公开信息 | 可直接使用AI辅助 | ✅ |

### 3.2 保密操作规则

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  客户信息 → AI系统 → ❌ 绝对禁止
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

规则：
1. 客户敏感信息不得输入公共AI系统
2. 企业内部使用的AI系统（如本技能体系）
   仅限于脱敏后的信息
3. 以下信息必须脱敏（替换为占位符）：
   · 公司名称 → [A公司]
   · 个人姓名 → [XX]
   · 财务数据 → [XX万元]
   · 交易金额 → [XX亿元]
   · 客户唯一识别信息 → [XX]
4. 脱敏后的信息不可逆向识别客户
```

### 3.3 保密违规处理

| 违规级别 | 违规描述 | 处理措施 |
|----------|----------|----------|
| **一级** | 脱敏不充分，可逆向识别 | 立即撤回、修正脱敏 |
| **二级** | 将客户非公开信息输入公共AI | 内部通报、终止AI权限 |
| **三级** | 将内幕信息泄露 | 启动保密事故处理程序 |

## 四、披露义务

### 4.1 对客户的AI使用披露

| AI使用程度 | 披露要求 | 示例披露语言 |
|------------|----------|--------------|
| **基础辅助**（绿灯区） | 合同约定即可 | "本项目将使用信息技术工具辅助开展工作" |
| **深度辅助**（黄灯区） | 专项书面告知 | "本项目将使用AI辅助工具进行信息分析和初稿处理，所有AI产出均经过专业人员审核" |
| **禁止场景**（红灯区） | AI参与为零 | 无须披露 |

### 4.2 披露的基本原则

```
1. 事前披露 — AI参与程度在合同/协议中预先约定
2. 适度披露 — 不制造AI恐慌，但需诚实告知
3. 专业保证 — 明确"人类专业判断是最终依据"
4. 可追溯 — AI辅助的部分留有记录备查
```

## 五、质量控制

### 5.1 AI产出的"三道关卡"

```
AI产出
    │
    ▼
┌─────────────────────┐
│  第一关：事实核查    │  ← 所有AI产出的数据、引用、时间
│  - 数据来源核实     │     必须人工核实
│  - 法律引用核对     │
│  - 时效性检查       │
└─────────┬───────────┘
          │ ✅ 通过
          ▼
┌─────────────────────┐
│  第二关：专业审核    │  ← 涉及实质性判断的内容
│  - 逻辑一致性       │     必须由专业人员审核
│  - 判断合理性       │
│  - 方案可行性       │
└─────────┬───────────┘
          │ ✅ 通过
          ▼
┌─────────────────────┐
│  第三关：最终确认    │  ← 最终版本必须经过
│  - 全面核对         │     项目负责人确认
│  - 责任归属明确     │
│  - 归档记录完整     │
└─────────────────────┘
```

### 5.2 AI辅助比例的合理区间

| 文件类型 | AI辅助上限 | 原因 |
|----------|-----------|------|
| 行业分析初稿 | 60% | 数据驱动，AI效率高 |
| 财务数据汇总 | 40% | AI适合整理，需人核对 |
| 法律分析初稿 | 30% | 法律判断复杂 |
| 估值模型辅助 | 25% | 估值涉及大量假设和判断 |
| 风险分析 | 20% | 风险判断高度依赖经验 |
| 申报文件最终版 | **0%** | 必须由人完成 |

## 六、持续改进

### 6.1 定期审查机制

| 审查频率 | 审查内容 | 责任人 |
|----------|----------|--------|
| **季度** | AI辅助的实际效果评估 | 项目负责人 |
| **半年度** | AI使用边界的审视和更新 | 投行部门 |
| **年度** | 本伦理准则的全面修订 | 合规+IB |

### 6.2 伦理准则演进原则

随着AI技术的演进，未来可能扩展AI辅助范围：
- AI由辅助工具演进为协同工具 → 绿灯区可能扩大
- 法规/监管对AI使用的态度演变 → 边界相应调整
- 新的专业判断任务类型出现 → 分类表持续更新

**始终不变的原则**：
> 最终责任由人类专业人士承担，签字权不可让渡。

## 八、使用指南

### 8.1 如何使用本伦理准则

1. **新入职承做人员**：通读全文，理解AI辅助边界
2. **项目启动**：确认AI使用策略并向客户披露
3. **使用AI辅助后**：按"三道关卡"进行质量审核
4. **遇到边界模糊**：先归入🟡黄灯区，经项目负责人确认

# META-11: 投行项目生命周期管理模型

> 定位：投行项目全生命周期的结构化框架，覆盖从业务承揽到项目结项的完整闭环。

## 一、模型总览

投行项目生命周期分为七个阶段，每个阶段有明确的产出物、决策门和时间约束：

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 01 机会  │──▶│ 02 尽调  │──▶│ 03 方案  │──▶│ 04 文件  │
│  发现    │   │  阶段    │   │  设计    │   │  撰写    │
└──────────┘   └──────────┘   └──────────┘   └──────────┘
    │              │              │              │
  3个月         3-6个月        1-2个月        2-4个月

┌──────────┐   ┌──────────┐   ┌──────────┐
│ 05 监管  │──▶│ 06 发行  │──▶│ 07 存续  │
│  审核    │   │  执行    │   │  期管理  │
└──────────┘   └──────────┘   └──────────┘
    │              │              │
  3-6个月        1-3个月        持续
```

## 二、阶段一：机会发现（3个月）

### 2.1 核心流程

```
市场扫描 → 客户接触 → 初步尽调 → 立项决策
```

### 2.2 核心产出物

| 产出物 | 内容要求 | 责任人 |
|--------|----------|--------|
| 客户名单 | 按行业/区域/规模筛选的潜在客户清单 | BD团队 |
| 初步尽调报告 | 基本情况、财务概览、潜在问题 | 项目组 |
| 立项建议书 | 项目可行性初步评估 | 项目负责人 |
| 立项审批表 | 质控/风控/IB负责人签字 | 内核 |

### 2.3 关键决策门（Gate Check）

**Gate 1: 立项审批**

| 决策要素 | 通过标准 | 不通过信号 |
|----------|----------|------------|
| 客户资质 | 符合基本准入条件 | 存在硬伤（如重大违法、财务造假嫌疑） |
| 业务价值 | 预计承销费≥X万 | 项目规模过小 |
| 风险评估 | 风险可控 | 存在无法整改的实质性问题 |
| 利益冲突 | 无冲突或可隔离 | 存在不可隔离的利益冲突 |

**决策产出**：
- ✅ 通过 → 进入尽调阶段
- ⏸️ 有条件通过 → 补充尽调后复议
- ❌ 不通过 → 归档，12个月内不再接触

### 2.4 风险控制要点

1. **红线筛查**（10项硬伤）
   - 是否存在重大违法违规
   - 是否涉嫌财务造假
   - 实控人是否有重大负面记录
   - 是否存在未决重大诉讼
   - 是否符合环保要求
   - 是否存在关联交易非关联化
   - 是否存在隐性债务
   - 是否符合所在行业政策导向
   - 是否存在隐性担保
   - 是否存在股权纠纷

2. **软性风险提示**
   - 行业周期判断
   - 公司治理结构
   - 管理层稳定性
   - 客户集中度

### 2.5 时间窗口约束

| 节点 | 标准时间 | 最长时限 |
|------|----------|----------|
| 首次接触 → 尽调启动 | 2周 | 4周 |
| 尽调启动 → 立项报告 | 4周 | 6周 |
| 立项报告 → 立项审批 | 1周 | 2周 |
| **阶段总时长** | **7周** | **12周** |

### 2.6 跨产品线差异

| 产品线 | 机会发现特点 | 关键差异点 |
|--------|--------------|------------|
| **IPO** | 行业研究驱动，早介入培养 | 培育期长（1-3年），需长期跟踪 |
| **债券** | 融资需求驱动，响应快 | 季节性强，需把握窗口期 |
| **并购** | 战略咨询驱动，定制化 | 涉及多方博弈，时机敏感 |

## 三、阶段二：尽职调查（3-6个月）

### 3.1 核心流程

```
┌─────────────────────────────────────────────────────────────┐
│                     尽职调查阶段                              │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐    │
│  │ 法律DD  │──▶│ 财务DD  │──▶│ 业务DD  │──▶│ 问题整改 │    │
│  └─────────┘   └─────────┘   └─────────┘   └─────────┘    │
│       │             │             │             │          │
│       └─────────────┴─────────────┴─────────────┘          │
│                           │                                │
│                    ┌──────▼──────┐                        │
│                    │  内核初审   │                        │
│                    └─────────────┘                        │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 核心产出物

| 产出物 | 内容要求 | 责任人 |
|--------|----------|--------|
| 法律尽调报告 | 主体资格、历史沿革、资产权属、重大合同、诉讼仲裁 | 律师 |
| 财务尽调报告 | 财务真实性、盈利能力、内控有效性 | 会计师 |
| 业务尽调报告 | 行业地位、商业模式、竞争分析 | 投行 |
| 问题清单（RAG） | 按风险等级分类的问题汇总 | 项目组 |
| 整改完成确认 | 问题闭环证明材料 | 客户/中介 |

### 3.3 关键决策门（Gate Check）

**Gate 2: 内核初审**

| 决策要素 | 通过标准 | 关注信号 |
|----------|----------|----------|
| 尽调完整性 | 三大尽调覆盖所有重大事项 | 存在尽调盲区 |
| 问题整改 | Red问题清零，Amber问题有方案 | 存在未整改的Red问题 |
| 底稿完备性 | 底稿支持尽调结论 | 底稿缺失或不完整 |
| 中介意见一致 | 各中介对关键问题达成一致 | 存在重大分歧 |

### 3.4 风险控制要点

**尽调核心关注点矩阵**

| 尽调类型 | 核心关注 | 风险信号 | 整改要求 |
|----------|----------|----------|----------|
| 法律DD | 主体资格、股权清晰、资产完整 | 股权代持、权属瑕疵 | Red必须清零 |
| 财务DD | 收入真实性、成本完整性、利润质量 | 银行流水异常、收入确认争议 | 需充分披露+风险缓释 |
| 业务DD | 商业模式可持续性、核心竞争力 | 客户集中度高、技术依赖 | 需合理解释+风险提示 |

### 3.5 时间窗口约束

| 尽调类型 | 标准时间 | 复杂项目 |
|----------|----------|----------|
| 法律DD | 4-6周 | 8-10周 |
| 财务DD | 6-8周 | 10-12周 |
| 业务DD | 3-4周 | 6-8周 |
| 问题整改 | 2-4周 | 6-8周 |
| 内核初审 | 1-2周 | 2-3周 |
| **阶段总时长** | **16-24周** | **32-40周** |

### 3.6 跨产品线差异

| 产品线 | 尽调重点差异 | 特殊要求 |
|--------|--------------|----------|
| **IPO** | 全面、深入，历史沿革核查 | 三年一期，全覆盖 |
| **债券** | 侧重偿债能力、信用分析 | 近三年财务，信用评级配合 |
| **并购** | 侧重估值、协同效应 | 尽调范围可能扩展至交易对方 |

## 四、阶段三：方案设计（1-2个月）

### 4.1 核心流程

```
交易方案设计 → 估值定价 → 条款谈判 → 方案定稿
```

### 4.2 核心产出物

| 产出物 | 内容要求 | 责任人 |
|--------|----------|--------|
| 交易方案 | 产品选择、规模、期限、结构设计 | 投行 |
| 估值报告 | 估值方法、假设、结论 | 评估师/投行 |
| 条款清单 | 核心条款及谈判记录 | 投行 |
| 方案确认函 | 客户确认签字 | 客户 |

### 4.3 关键决策门（Gate Check）

**Gate 3: 方案审定**

| 决策要素 | 通过标准 | 关注信号 |
|----------|----------|----------|
| 方案可行性 | 符合监管要求、市场可接受 | 存在监管障碍 |
| 定价合理性 | 在市场合理区间 | 定价过高/过低 |
| 风险收益匹配 | 收益覆盖风险成本 | 风险敞口过大 |

### 4.4 时间窗口约束

| 节点 | 标准时间 |
|------|----------|
| 方案设计 | 2-3周 |
| 估值定价 | 2-3周 |
| 条款谈判 | 1-2周 |
| 方案定稿 | 1周 |
| **阶段总时长** | **6-9周** |

### 4.5 跨产品线差异

| 产品线 | 方案设计特点 |
|--------|--------------|
| **IPO** | 板块选择、发行规模、募投项目、发行定价 |
| **债券** | 品种选择、期限结构、增信安排、利率区间 |
| **并购** | 交易结构、对价支付、业绩承诺、整合方案 |

## 五、阶段四：文件撰写（2-4个月）

### 5.1 核心流程

```
┌─────────────────────────────────────────────────────────────┐
│                     文件撰写阶段                              │
│                                                             │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐    │
│  │P0 初稿  │──▶│中介审阅 │──▶│P1 修改稿│──▶│P2 内核稿│    │
│  └─────────┘   └─────────┘   └─────────┘   └─────────┘    │
│       │             │             │             │          │
│    投行撰写      中介意见      投行修改      内核审核       │
│                                                             │
│                         │                                   │
│                  ┌──────▼──────┐                           │
│                  │P3 申报稿    │                           │
│                  └─────────────┘                           │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 核心产出物

| 产出物 | 内容要求 | 版本要求 |
|--------|----------|----------|
| 申报文件全套 | 主报告+附件+底稿 | P3版本 |
| 各中介专业报告 | 法律意见书、审计报告等 | 签字版本 |
| 内核意见回复 | 对内核问题的逐项回复 | 正式回复函 |
| 申报材料清单 | 全套材料checklist | 打勾确认版 |

### 5.3 关键决策门（Gate Check）

**Gate 4: 内核终审**

| 决策要素 | 通过标准 | 关注信号 |
|----------|----------|----------|
| 文件完整性 | 符合申报要求清单 | 有遗漏项 |
| 信息披露 | 充分、准确、完整 | 存在重大遗漏 |
| 合规性 | 符合法律法规要求 | 存在合规障碍 |
| 风险揭示 | 风险因素全面揭示 | 风险揭示不充分 |

### 5.4 文件版本迭代规范

```
P0 (初稿) → P1 (中介审阅后) → P2 (内核稿) → P3 (申报稿) → PN (反馈修改稿)
```

| 版本 | 定义 | 审核重点 |
|------|------|----------|
| P0 | 投行初稿完成 | 结构完整、逻辑清晰 |
| P1 | 中介交叉审阅后 | 事实准确、数据一致 |
| P2 | 提交内核版本 | 合规性、风险揭示 |
| P3 | 申报版本 | 最终核对、签字盖章 |
| PN | 反馈回复修改 | 逐项回复、修改留痕 |

### 5.5 时间窗口约束

| 节点 | 标准时间 | 复杂项目 |
|------|----------|----------|
| P0初稿 | 3-4周 | 6-8周 |
| 中介审阅 | 2周 | 3周 |
| P1修改 | 1-2周 | 2-3周 |
| 内核审核 | 1-2周 | 2-3周 |
| P3定稿 | 1周 | 2周 |
| **阶段总时长** | **8-11周** | **14-19周** |

## 六、阶段五：监管审核（3-6个月）

### 6.1 核心流程

```
材料申报 → 受理/补正 → 问询轮次 → 上会/注册 → 批文
```

### 6.2 核心产出物

| 产出物 | 内容要求 | 时间要求 |
|--------|----------|----------|
| 申报材料 | 全套文件+清单 | 申报日 |
| 反馈回复 | 逐项回复监管问询 | 收到反馈后规定时间内 |
| 上会稿 | 上会/注册申请版本 | 上会前 |
| 批文 | 监管批准文件 | 核准日 |

### 6.3 关键决策门（Gate Check）

**Gate 5: 申报前确认**

| 决策要素 | 通过标准 | 风险信号 |
|----------|----------|----------|
| 材料完备性 | 符合受理要求 | 存在补正风险 |
| 时效性 | 财务数据在有效期内 | 接近有效期截止 |
| 重大事项 | 无重大变化需补充披露 | 近期有重大事项 |

### 6.4 审核周期参考

| 产品线 | 标准周期 | 审核机构 |
|--------|----------|----------|
| IPO（注册制） | 6-12个月 | 交易所+证监会 |
| 企业债 | 3-6个月 | 发改委 |
| 公司债 | 2-4个月 | 交易所 |
| 重大重组 | 3-6个月 | 证监会 |

### 6.5 风险信号识别

| 风险等级 | 信号特征 | 应对策略 |
|----------|----------|----------|
| 🔴 Red | 问询>3轮、涉及持续经营能力 | 准备撤回或重大调整方案 |
| 🟡 Amber | 问询涉及财务/同业竞争 | 重点准备，补充材料 |
| 🟢 Green | 问询涉及信息披露规范性 | 常规回复即可 |

## 七、阶段六：发行执行（1-3个月）

### 7.1 核心流程

```
发行准备 → 路演推介 → 询价/簿记 → 定价配售 → 挂牌/交割
```

### 7.2 核心产出物

| 产出物 | 内容要求 |
|--------|----------|
| 发行方案 | 发行时间表、配售原则 |
| 路演材料 | 投资故事、Q&A |
| 询价记录 | 询价对象、报价区间 |
| 配售结果 | 配售明细、缴款安排 |
| 上市/交割文件 | 挂牌申请、交割确认 |

### 7.3 关键决策门（Gate Check）

**Gate 6: 发行定价确认**

| 决策要素 | 通过标准 |
|----------|----------|
| 定价合理性 | 在询价区间内、市场可接受 |
| 配售合规性 | 符合配售规则、无利益输送 |
| 募集资金 | 达到预期规模 |

### 7.4 时间窗口约束

| 节点 | 标准时间 |
|------|----------|
| 发行准备 | 2周 |
| 路演推介 | 1-2周 |
| 询价/簿记 | 3-5天 |
| 缴款交割 | 1周 |
| 挂牌上市 | 1-2周 |
| **阶段总时长** | **4-7周** |

## 十一、使用指南

### 11.1 如何使用本模型

1. **项目启动时**：对照本模型制定项目时间表
2. **阶段推进时**：检查决策门是否具备通过条件
3. **风险管理时**：识别当前阶段的关键风险点
4. **跨产品线工作时**：参考产品线差异调整策略

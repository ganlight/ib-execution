## 一、各板块标准IPO全周期

### 1.1 总览

| 板块 | 标准周期 | 最紧凑周期 | 关键加速因素 | 主要瓶颈 |
|------|:--------:|:--------:|-------------|----------|
| **主板**（沪深） | 12~18个月 | 10个月 | 辅导验收绿色通道 | 审核排队时间长 |
| **科创板** | 9~12个月 | 7个月 | 科创属性明确、无现场检查 | 科创属性论证要求高 |
| **创业板** | 10~14个月 | 8个月 | 三创四新清晰、行业不敏感 | 负面清单行业限制 |
| **北交所** | 6~9个月 | 5个月 | 新三板挂牌满12个月自动触发 | 注册制磨合期 |

### 1.2 各板块阶段拆解（单位：月）

```
板块      改制重组  辅导备案    文件制作    审核/问询   注册/发行  总计
                    规范运行

主板       2~3      4~6        3~4        4~6        1~2      12~18
  ├─ 国企改制通常更慢（国资审批+职工安置+土地处置）
  └─ 大型企业改制复杂（业务分拆+关联方清理）

科创板     1~2      3~4        2~3        3~4        1~2       9~12
  ├─ 核心技术人员稳定（IPO前2年无重大变化）
  └─ 研发投入/专利量化指标达标

创业板     1~3      3~5        2~3        3~5        1~2      10~14
  ├─ 行业属性审查趋严（部分行业劝退）
  └─ 成长性论证（营收/利润复合增长率）

北交所     1~2      2~3        1~2        2~3        0.5~1     6~9
  ├─ 前提：新三板挂牌满12个月（含创新层）
  └─ 规模较小，改制复杂度低
```

## 二、六大阶段详细拆解

### 阶段一：改制重组（1~3个月）

```
主要工作：
  ├── 确定上市主体/架构
  ├── 业务重组（剥离非核心资产）
  ├── 股权结构调整（清理代持、搭建员工持股平台）
  ├── 有限公司→股份公司变更
  └── 工商登记完成

里程碑：取得股份公司营业执照

常见延期原因：
  - 国资审批流程长（+1~2个月）
  - 土地/房产权属不清需补办（+2~3个月）
  - 历史沿革中的股权纠纷/诉讼（不可预估）
```

### 阶段二：辅导备案与规范运行（3~6个月）

```
主要工作：
  ├── 辅导备案申请（属地证监局）
  ├── 辅导期内规范运作
  │   ├── 完善公司治理结构
  │   ├── 规范关联交易/对外担保
  │   ├── 建立内控制度
  │   └── 培训董监高
  ├── 辅导验收申请
  └── 取得辅导验收报告

里程碑：取得证监局辅导验收通过函

常见延期原因：
  - 辅导期内发现重大不规范需整改（+1~3个月）
  - 证监局要求补充辅导（+1~2个月）
  - 现场检查发现问题需整改（+1个月+）
  - 辅导验收排队（部分地区）
```

### 阶段三：文件制作与内核（2~3个月）

```
主要工作：
  ├── [会计师] 出具三年一期审计报告
  ├── [会计师] 出具内控鉴证报告
  ├── [律师] 出具法律意见书+律师工作报告
  ├── [评估师] 出具评估报告（如适用）
  ├── [券商] 编制招股说明书
  ├── [券商] 全套申报文件汇编
  └── [券商] 内核（立项→质控→内核会）

里程碑：券商内核通过

常见延期原因：
  - 审计报告延迟（见 META-06 问题一）
  - 内核反馈问题多需要重大修改（+2~4周）
  - 法律问题需补充核查（+2~6周）
```

### 阶段四：监管审核/问询（3~6个月）

```
主要工作：
  ├── 正式向交易所提交申报材料（受理）
  ├── 交易所首轮问询 → 回复
  ├── 交易所二轮问询（如有）→ 回复
  ├── 交易所审核中心会议
  └── 上市委会议审议

里程碑：上市委审议通过

关键时间节点：
  - 受理→首轮问询：约20个工作日
  - 每轮问询回复：15个工作日（可延期）
  - 二轮问询发出：首轮回复后约10个工作日
  - 上市委会议通知：会前5个工作日公告

常见延期原因：
  - 问题复杂需多轮问询（3轮以上 +1~2个月）
  - 需要专项核查（现场督导/检查 +2~3个月）
  - 补充最新一期财务数据（年报/半年报更新节点）
  - 行业政策变化需更新论述（+1~2个月）
```

### 阶段五：证监会注册（1~2个月）

```
主要工作：
  ├── 向证监会提交注册申请
  ├── 证监会反馈（如有）→ 回复
  └── 取得注册批文

里程碑：取得证监会注册批文

注意：
  - 主板/创业板/科创板均需注册（注册制全覆盖）
  - 注册批文有效期：12个月
  - 注册环节可能有补充问询（不是必经环节）
```

### 阶段六：发行上市（1~2个月）

```
主要工作：
  ├── 路演推介（分析师路演+管理层路演）
  ├── 询价/定价（网下询价→确定发行价）
  ├── 网上/网下申购
  ├── 中签/缴款
  └── 挂牌上市

里程碑：交易所敲钟上市 🎉

关键节点：
  - 取得批文→启动发行：1~2周
  - 路演周期：约1周
  - 申购→上市：约2~3周

常见延期原因：
  - 市场环境不佳推迟发行
  - 会后事项（利润大幅下滑等）
  - 发行失败风险（极端情况）
```

## 三、关键依赖关系

### 3.1 硬依赖（不可逾越）

```
依赖关系链：

[审计报告定稿]
    │
    ├──→ 内控鉴证报告定稿
    │
    ├──→ 法律意见书定稿（关联方/担保数据）
    │
    ├──→ 招股书财务章节定稿
    │
    ├──→ 招股书MD&A定稿
    │
    └──→ 全套申报文件 → [提交受理]

[上市委通过] → [注册批文] → [发行上市]

关键约束：
  若审计报告延期2周 → 所有下游节点顺延2周
  注册批文12个月内有效 → 发行人必须在有效期内完成发行
```

### 3.2 软依赖（可提前/并行工作）

```
中介机构选择 → 改制重组启动 （可提前在项目论证期完成）
招股书业务章节 → 审计报告 （可用发行人内部数据先行）
路演材料准备 → 注册批文 （注册期间可同步准备）
封卷稿材料 → 上市委审议 （审议通过后立即提交）
网下投资者沟通 → 注册批文 （可提前建立投资者关系）
```

## 四、风险预警系统

### 4.1 三级预警信号

```
🔴 红线（Red Line）— 直接影响关键路径

触发条件：
  ├── 审计报告交付延迟超过2周
  │   → 所有下游节点整体平移
  │   → 可能错过申报窗口（如年报更新截止前）
  │
  ├── 注册批文有效期剩余 < 3个月 仍未启动发行
  │   → 存在批文过期风险
  │   → 必须立即决定发行或延期
  │
  ├── 报告期最后一年利润大幅下滑（>30%）
  │   → 不符合上市条件
  │   → 可能需要更换报告期或终止
  │
  └── 现场检查/督导发现问题严重
      → 可能劝退/撤回
      → 个别情况下引发处罚

处理级别：立即报告项目负责人+部门管理层，24小时内制定应对方案
```

```
🟡 黄色预警（Yellow Warning）— 需密切关注，可能升级

触发条件：
  ├── 内核反馈问题超过20条或多轮内核
  │   → 可能在审核阶段收到更多问询
  │   → 需评估是否提前准备二轮回复模板
  │   → 参考 META-03 内核高频问题库
  │
  ├── 招股书篇幅超标（>800页，不含附件）
  │   → 交易所可能要求精简
  │
  ├── 单一中介机构人员变动（签字人离职等）
  │   → 影响文件出具节奏
  │
  ├── 行业处于下行周期
  │   → 招股书需加强可持续发展论证
  │   → 可能被要求补充盈利预测
  │
  └── 可比公司出现重大负面事件
      → 披露时需针对性说明

处理级别：纳入每周进度会议议题，持续跟踪
```

```
🔵 蓝色提示（Blue Note）— 优化建议，非紧迫

触发条件：
  ├── 可比公司估值下行 → 考虑是否调整定价策略预期
  ├── 新一轮再融资政策出台 → 评估是否影响审核节奏
  ├── 行业季节性 → 合理安排路演时间窗口
  └── 国会/两会等重大事件 → 注意政治敏感期审核进度

处理级别：项目负责人自行判断是否需要采取行动
```

### 4.2 预警响应流程

```
发现预警信号
  │
  ├── 判断级别：
  │     🔴 → 立即响应（2小时内）
  │     🟡 → 当天内响应
  │     🔵 → 周会中讨论
  │
  ├── 制定方案：
  │     🔴 → 书面应对方案 + 管理层审批
  │     🟡 → 项目组长审批
  │     🔵 → 个人判断
  │
  └── 执行 + 持续监控
```

```

│ ├─ 中介机构尽调（机构方预备工作）    │ 并行！
│ └─ 董监高培训（券商+律师）           │

[文件制作期间] ────────────────────────
│ ├─ 审计报告（会计师）                │
│ ├─ 法律意见书-无争议部分（律师）     │ 并行！
│ ├─ 招股书-业务章节（券商）           │
│ ├─ 评估现场勘查（评估师）            │
│ └─ 申报材料汇编框架（券商）          │
└──────────────────────────────────────┘

[审核期间] ────────────────────────────
│ ├─ 问询回复（全组）                  │
│ ├─ 路演材料初稿（券商）              │ 并行！
│ ├─ 投资者沟通预路演（券商）          │
│ └─ 年报/半年报更新准备（会计师）     │
└──────────────────────────────────────┘
```

### 5.2 时间压缩策略

| 策略 | 适用场景 | 可压缩时间 | 代价/风险 |
|------|----------|:--------:|----------|
| 多中介并行尽调 | 改制与尽调重叠 | 1~2个月 | 信息同步成本高 |
| 问询回复预填模板 | 常规问题按 META-03 模板提前准备 | 2~3周 | 部分模板需调整 |
| 滚动文件更新 | 边写边改边审而非串行 | 2~4周 | 版本管理复杂 |
| 绿色通道 | 符合国家战略的企业 | 1~3个月 | 门槛高（需部委推荐等） |

## 六、timeline_gen.py 脚本逻辑说明

### 6.1 脚本概述

```
名称：timeline_gen.py
功能：根据项目参数自动生成 IPO 全周期甘特图
输入：板块、企业规模、行业、特殊事项
输出：甘特图（HTML可交互）、Excel里程碑表、iCal文件
```

### 6.2 核心模块

```python
# 伪代码
class TimelineEngine:
    """时间线生成引擎"""

    def __init__(self):
        # 加载各板块标准周期基准
        self.board_baseline = load_baseline()  # 来自本节1.1表
        # 加载阶段依赖关系
        self.dependency_graph = load_dependencies()  # 来自本节3.1
        # 加载并行策略
        self.parallel_strategies = load_parallel()  # 来自本节5.1

    def generate(self, params: ProjectParams) -> Timeline:
        """主生成流程"""
        # Step 1: 确定基准周期（按板块）
        baseline = self.board_baseline[params.board]

        # Step 2: 调整因子
        adjustments = []
        adjustments += self.size_adj(params.revenue_scale)     # 规模
        adjustments += self.industry_adj(params.industry)      # 行业
        adjustments += self.soe_adj(params.is_state_owned)     # 国企
        adjustments += self.complexity_adj(params.complexity)  # 复杂度

        # Step 3: 计算各阶段时长
        phases = self.calc_phases(baseline, adjustments)

        # Step 4: 按依赖关系排序，考虑并行
        timeline = self.schedule(phases, self.dependency_graph,
                                 self.parallel_strategies)

        # Step 5: 注入风险预警节点
        timeline = self.inject_risk_markers(timeline)

        # Step 6: 生成关键里程碑检查点
        timeline = self.inject_checkpoints(timeline)

        return timeline


class RiskMarker:
    """风险标记生成"""
    def inject(self, timeline, params):
        # 自动插入风险缓冲
        # 关键节点前后自动添加缓冲期
        timeline.add_buffer("审计报告完成", "+2周")
        timeline.add_buffer("反馈回复提交", "+1周")
        # 标记可能的加速窗口
        timeline.mark_acceleration("改制重组与尽调重叠")
```

### 6.3 使用示例

```bash
# 生成创业板项目时间表
python timeline_gen.py \
  --board 创业板 \
  --revenue 5000000000 \   # 50亿营收
  --industry 半导体 \
  --start-date 2026-06-01 \
  --output-html timeline.html \
  --output-excel milestones.xlsx

# 对比多个板块
python timeline_gen.py \
  --board 主板 \
  --compare 科创板,创业板 \
  --output-html comparison.html
```

## 七、快速决策矩阵

| 场景 | 如果延迟≤1周 | 如果延迟1~3周 | 如果延迟>3周 |
|------|------------|-------------|------------|
| 审计报告 | 通过压缩后续文件制作时间消化 | 启动紧急并行策略，提前开始部分工作 | 需要正式延期计划，重排全部下游节点 |
| 问询回复 | 内部加班消化 | 申请1次延期（15个工作日可用） | 可能触发二轮问询，需调整预期 |
| 发行窗口 | 无影响 | 市场判断是否推迟 | 考虑更换发行方式（战略配售等） |

> **维护提示**: 时间表是动态文件，不是一次性产物。每次项目复盘时，将实际与计划之间的偏差原因记录到 META-08（SKILL优化与演化）Step 3 中，用于修正未来的基准时长和风险预警规则。

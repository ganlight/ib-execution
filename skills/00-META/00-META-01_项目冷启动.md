# 00-META-01 项目冷启动方法论

> **适用场景**：接手全新IPO/再融资/重组项目时，从零到一的启动流程标准化操作指南。
> **核心目标**：48小时内完成项目画像、可比分析、监管预判、风险摸底、时间规划五步，输出《项目冷启动报告》。

---

## 一、冷启动五步法总览

```
Day 1上午   Step 1 → 项目画像（2h）
Day 1下午   Step 2 → 可比分析（3h）
Day 1晚上   Step 3 → 监管检查（1h）
Day 2上午   Step 4 → 风险识别（3h）
Day 2下午   Step 5 → 时间规划（2h）
Day 2晚上   汇总输出 → 冷启动报告（1h）
```

### 前置条件检查清单

- [ ] 已获取公司基础资料（营业执照、工商底档、近三年审计报告）
- [ ] 已确认申报板块（主板/科创板/创业板/北交所/境外）
- [ ] 已获取客户/供应商清单及关联方清单
- [ ] 已获取历史股权变动文件（工商变更记录、增资协议）
- [ ] 已确认项目负责人及团队分工

## 二、Step 1：项目画像（Project Profile）

### 2.1 信息收集维度

| 维度 | 核心指标 | 数据来源 |
|------|---------|---------|
| **财务规模** | 最近一年营收、净利润、总资产、净资产 | 审计报告 |
| **财务质量** | 毛利率、净利率趋势（近三年）、经营性现金流 | 审计报告 |
| **行业定位** | 所属证监会行业分类、产业链位置 | 公司业务描述 |
| **股权结构** | 实际控制人及持股比例、前五大股东 | 工商底档 |
| **客户集中度** | 前五大客户占比、是否存在单一重大依赖 | 公司提供 |

### 2.2 板块偏好判定

使用 `ipo_board_check.py` 的判定逻辑：

```python
# 板块选择决策树
def determine_board(revenue, net_profit, total_assets, industry, rnd_ratio):
    if net_profit >= 100_000_000:  # 净利润≥1亿
        if industry in MAIN_BOARD_RESTRICTED:
            return "科创板" if is_tech_innovative(industry, rnd_ratio) else "创业板"
        return "主板（沪深）"
    elif net_profit >= 50_000_000:
        if is_tech_innovative(industry, rnd_ratio):
            return "科创板"
        return "创业板"
    else:
        return "北交所"  # 或境外
```

## 三、Step 2：可比分析（Comparable Analysis）

### 3.1 搜索范围与方法

**时间范围**：最近12个月（滚动窗口，覆盖最新审核动态）

**搜索源**：
1. 交易所审核项目公示页面（上交所/深交所/北交所）
2. Wind金融终端 → 一级市场 → IPO审核
3. 证监会/交易所问询函公开披露

**搜索策略**：
```
关键词组合 = [行业关键词] + [IPO] + [审核状态]
行业关键词 = 同行前3-5家上市公司的主营业务词云
```

### 3.2 可比分析框架

| 分析维度 | 对比内容 | 输出格式 |
|---------|---------|---------|
| **估值对标** | 发行市盈率、市净率、募资规模 | 表格对比 |
| **审核周期** | 受理→问询→上会→注册 各环节天数 | 甘特图 |
| **被问询概率排名** | 哪些问题被反复提及（统计词频） | 排序表 |
| **被否/撤回案例分析** | 最近6个月同类行业被否案例及原因 | 摘要 |
| **财务指标参考区间** | 可比过会企业的收入/利润/毛利率区间 | 分布图 |

### 3.3 输出：可比分析摘要模板

```markdown
## 可比分析（最近12个月同类IPO）

### 板块平均水平
```

## 四、Step 3：监管检查（Regulatory Check）

### 4.1 板块准入条件对照

使用 `ipo_board_check.py` 的完整检查逻辑：

```python
# 文件位置: scripts/ipo_board_check.py
# 功能: 根据公司数据自动检查各板块准入条件

class IPOBoardChecker:
    """
    检查项包括:
    1. 财务指标门槛（各板块差异化）
    2. 行业负面清单
    3. 特殊政策要求（如科创板"硬科技"属性）
    4. 环保/安全等特殊行业额外要求
    """
    def check_all(self, company_data):
        results = {}
        for board in ['main_sh', 'main_sz', 'star', 'chinext', 'bse']:
            results[board] = self._check_single(board, company_data)
        return results
```

### 4.2 行业准入限制清单

| 板块 | 限制/禁止行业 | 鼓励行业 |
|------|-------------|---------|
| **科创板** | 金融科技、模式创新类（商业模式创新不支持）、房地产 | 新一代信息技术、高端装备、新材料、新能源、生物医药 |
| **创业板** | 农林牧渔、酒/饮料、纺织、黑色金属冶炼、火电等12个传统行业 | "三创四新"企业 |
| **主板** | 无明确行业限制，但"大盘蓝筹"定位 | 规模大、模式成熟的行业龙头 |
| **北交所** | 金融业、房地产 | "专精特新"中小企业 |

### 4.3 特殊行业附加要求

- **医药/医疗器械**：需取得注册证/生产许可证，临床试验数据完整性
- **军工**：保密资质核查，涉密信息脱敏处理
- **教育**："双减"政策合规性（K12学科培训禁止上市）
- **数据相关行业**：网络安全审查、数据出境安全评估
- **环保行业**：排污许可、环保处罚记录排查

## 五、Step 4：关键风险识别

### 4.1 四大风险维度核查清单

#### 维度一：权属清晰度（Ownership Clarity）

| 核查点 | 风险信号 | 风险等级 |
|--------|---------|---------|
| 实际控制人认定 | 近三年变更、多人共同控制且无一致行动协议 | 🔴 高 |
| 股权代持 | 历史存在代持且未完全解除 | 🔴 高 |
| 历次出资 | 非货币出资未评估、出资瑕疵 | 🟡 中 |
| 股权质押/冻结 | 控股股东股权质押比例>50% | 🟡 中 |
| VIEs架构 | 红筹/VIE结构需回归或解释 | 🟡 中 |

#### 维度二：关联交易（Related Party Transactions）

| 核查点 | 风险信号 | 风险等级 |
|--------|---------|---------|
| 关联交易必要性 | 可通过第三方替代的关联采购/销售 | 🔴 高 |
| 关联交易比例 | 关联销售占比>30%或关联采购>30% | 🔴 高 |
| 关联交易定价 | 无第三方比价、明显偏离市场 | 🔴 高 |
| 关联方资金往来 | 历史存在非经营性资金占用 | 🔴 高 |
| 关联担保 | 对关联方大额担保 | 🟡 中 |

#### 维度三：历史合规（Historical Compliance）

```markdown
□ 社保公积金缴纳比例及基数是否合规？
□ 历史是否存在税务处罚（增值税、企业所得税）？
□ 环保处罚/安全生产事故记录？
□ 土地/房产使用是否存在违规（无证/农用地转用）？
□ 知识产权侵权诉讼或潜在纠纷？
□ 商业贿赂/不正当竞争相关调查或处罚？
□ 劳动用工（劳务派遣比例>10%）？
```

#### 维度四：财务异常（Financial Anomalies）

```python
# 财务异常自动检测脚本（fina_check.py）
anomaly_checks = [
    ("收入增长 vs 应收增长", 
     lambda: rev_growth > 30% and ar_growth > 50%, 
     "可能存在放宽信用政策刺激收入"),
    ("毛利率 vs 行业均值",
     lambda: abs(margin - industry_avg) > 15,
     "毛利率显著偏离行业均值，需深入解释"),
    ("存货周转天数变化",
     lambda: inventory_days_yoy > 60,
     "存货积压风险，可能关联跌价准备不足"),
    ("经营活动现金流 vs 净利润",
     lambda: ocf < net_profit * 0.5 for 3 consecutive years,
     "盈利质量存疑，利润缺乏现金流支撑"),
    ("大额异常交易",
     lambda: near_period_end_large_sales > 30%,
     "临近资产负债表日大额外销，收入截止性问题"),
]
```

### 4.2 风险矩阵输出格式

```markdown
## 关键风险矩阵

| 风险项 | 风险等级 | 发生概率 | 影响程度 | 应对策略 |
|--------|---------|---------|---------|---------|
| 关联交易占比过高 | 🔴 高 | 确认存在 | 重大 | 引入第三方、承诺下降计划 |
| 社保未足额缴纳 | 🟡 中 | 确认存在 | 中等 | 测算补缴金额、控股股东兜底承诺 |
| 收入确认跨期风险 | 🟡 中 | 中等 | 重大 | 截止性测试、谨慎性原则 |
| ... | | | | |
```

### 6.1 标准IPO时间线生成

使用 `timeline_gen.py` 自动生成：

```python

# 用法: python timeline_gen.py --base_date 2024-12-31 --board star --type ipo

class TimelineGenerator:
    """
    根据基准日、板块、项目类型自动生成标准时间线
    输出: Gantt图 + Excel里程碑表 + CSV
    """
    STANDARD_PHASES = {
        "股份制改造": {"duration_days": 60, "before_filing": True},
        "辅导验收": {"duration_days": 90, "before_filing": True},
        "材料制作": {"duration_days": 120, "before_filing": False},

        "发行上市": {"duration_days": 45, "before_filing": False},
    }
```

### 6.2 时间线调整因子

根据项目特征调整各阶段时间：

| 调整因素 | 影响阶段 | 调整幅度 |
|---------|---------|---------|
| 行业敏感度高（如医药/军工） | 审核问询 | +30～+60天 |
| 存在重大无先例问题 | 整个审核周期 | +90天以上 |
| 前期已经过辅导 | 辅导验收 | -60天 |
| 财务年度衔接（基准日后） | 材料制作 | +30天（加期审计） |
| 境外核查（海外业务占比>30%） | 材料制作 | +30天 |
| 需要取得主管部门意见 | 材料制作/审核 | +30～+90天 |

## 七、质量门（Quality Gates）

### 冷启动报告通过标准

```markdown
□ Step 1 通过: 项目画像信息完整度 ≥ 90%（20个核心字段至少填18个）
□ Step 2 通过: 可比案例 ≥ 5个（至少3个过会、2个被否/撤回）
□ Step 3 通过: 所有准入条件条目已逐项打勾、行业限制已交叉确认
□ Step 4 通过: 风险矩阵覆盖率 ≥ 80%（4大维度*5个核心项 = 20项中至少16项已覆盖）
□ Step 5 通过: 时间线已有3轮以上（乐观/基准/悲观）且关键依赖关系已标注
□ 汇总通过: 冷启动报告已由项目负责人审阅确认
```

## 八、常见坑与避坑指南

| 易犯错误 | 后果 | 正确做法 |
|---------|------|---------|
| 跳过可比分析直接开工 | 材料方向偏差、反复修改 | 务必先做12个月可比扫描 |
| 只做一轮时间规划 | 执行中频繁修改时间线 | 至少乐观/基准/悲观三轮 |
| 风险识别浮于表面 | 审核阶段暴雷 | 按四大维度逐项深挖 |
| 板块选择凭感觉 | 申报后发现不达标需换板 | 用 `ipo_board_check.py` 跑一遍 |
| 忽视行业特殊要求 | 材料漏项被打回 | 对照行业清单逐项确认 |
| 未与发行人确认信息 | 后续推翻重来 | 重要判断必须有发行人书面确认 |
| 冷启动报告未留存 | 团队交接信息丢失 | 纳入项目知识库 `knowledge/` 目录 |

# SKILL_00 ABS方案总览

> ABS方案设计阶段全景与决策框架

## 一、ABS方案设计五大决策

### 1.1 决策树

```
基础资产筛选
    │
    ▼
方案设计五大决策
    │
    ├── ① 基础资产池构建
    │      └── 资产池规模 / 资产筛选标准 / 封包日设定
    │
    ├── ② 交易结构设计（核心决策）
    │      ├── SPV形式选择（信托/专项计划）
    │      ├── 分层结构（优先A/优先B/次级比例）
    │      ├── 现金流分配（Waterfall规则）
    │      ├── 循环购买（适用时）
    │      └── 信用触发机制（加速清偿/违约事件定义）
    │
    ├── ③ 信用增级设计
    │      ├── 内部增信（分层/超额抵押/超额利差/现金储备/触发机制）
    │      └── 外部增信（差额支付承诺/第三方担保/流动性支持/维好协议）
    │
    ├── ④ 现金流分配机制
    │      └── 税费→中介费用→优先A利息→优先A本金→优先B利息→优先B本金→次级收益
    │
    └── ⑤ 定价策略
           └── 各档票面利率确定方法（基准+利差）
```

### 1.2 决策优先级

| 决策维度 | 优先级 | 理由 |
|---------|-------|------|
| 基础资产池构建 | P0（最高） | 资产质量是ABS的信用基础，决定方案可行性上限 |
| 分层结构设计 | P0（最高） | ABS信用增级的核心机制，直接决定评级结果 |
| 现金流分配机制 | P1（高） | 影响各档证券的本息偿付确定性和时间 |
| 信用增级搭配 | P1（高） | 弥补分层信用支持的不足 |
| 定价策略 | P2（中） | 在结构和增信确定后，定价是发行前最后一步 |

## 二、三大市场方案差异

| 维度 | 信贷ABS方案 | 企业ABS方案 | ABN方案 |
|------|------------|------------|---------|
| **SPV形式** | 特定目的信托 | 资产支持专项计划 | 特定目的信托/公司/专项计划均可 |
| **分层结构** | 优先档+次级档（通常2-3档） | 优先A/优先B/次级（通常3-4档） | 介于信贷与企业之间 |
| **循环购买** | 较少使用 | 消费金融类常用 | 融资租赁/消费金融类使用 |
| **外部增信** | 较少使用（内部增信为主） | 常见（原始权益人提供差额支付承诺） | 常见（核心企业/母公司提供支持） |
| **现金流分配** | 按月/按季分配 | 按月/按季分配 | 按月/按季分配 |
| **主文件** | 信托合同+发行说明书 | 计划说明书+标准条款 | 募集说明书+信托合同 |
| **评级目标** | 优先档争取AAA | 优先A档争取AAA | 优先档争取AAA |

## 三、方案比选Checklist（8维度评分卡）

| 维度 | 评分标准（1-5分） | 权重 | 评分方法 |
|------|------------------|------|---------|
| 1. 合规完备性 | 5=完全合规，1=存在合规瑕疵 | {15}% | 对照法规清单逐项核查 |
| 2. 信用增级强度 | 5=AAA可达成，1=难以达到投资级 | {20}% | 评级机构预沟通反馈 |
| 3. 现金流匹配度 | 5=完全匹配，1=存在显著缺口 | {20}% | 现金流模型测试结果 |
| 4. 投资者接受度 | 5=主流结构，1=创新结构需大量解释 | {15}% | 投资者预沟通反馈 |
| 5. 发行成本 | 5=成本低，1=成本高 | {10}% | 费用预算 vs 同类产品 |
| 6. 操作复杂度 | 5=简单，1=高度复杂 | {10}% | 结构复杂度评分 |
| 7. 监管接受度 | 5=成熟案例多，1=无先例 | {5}% | 监管预沟通反馈 |
| 8. 灵活性 | 5=结构可调整，1=刚性 | {5}% | 结构可调整性评估 |

**总分计算**：`∑(各维度得分 × 权重)`

**决策规则**：
- 总分 ≥ {4.0}：方案可行，可推进
- 总分 {3.0-4.0}：须优化后重新评分
- 总分 < {3.0}：须重新设计方案

## 四、各基础资产类型典型交易结构（文字描述）

### 4.1 供应链ABS交易结构

```
核心企业
    │
    │ 确认应付账款
    ├──→ 供应商（原始债权人）
    │              │
    │              │ 转让应收账款
    │              ▼
    │    保理公司（原始权益人）
    │              │
    │              │ 转让给SPV
    │              ▼
    │    资产支持专项计划（SPV）
    │              │
    │              │ 发行ABS
    │              ▼
    │        投资者（购买ABS）
    │
    │ 到期按应付账款付款
    └──────────────────→ 核心企业
```

**结构要点**：
- 反向保理模式：核心企业确认应付账款，供应商将应收账款转让给保理公司
- 核心企业信用为核心：ABS评级取决于核心企业主体信用
- 通常设置为一次性（非循环），期限12-18个月

### 4.2 融资租赁ABS交易结构

```
融资租赁公司（原始权益人）
    │
    │ 拥有租赁合同债权
    ▼
资产支持专项计划/特定目的信托（SPV）
    │
    │ 发行ABS
    ▼
投资者
    │
    │ 本息偿付
    ▼
SPV（由托管行保管资金）
    │
    │ 收款指令
    ▼
承租人（按期支付租金）
```

**结构要点**：
- 融资租赁公司通常同时担任资产服务机构
- 租赁物抵押权随租赁债权一并转让
- 通常设置优先/次级分层+超额利差

### 4.3 消费金融ABS交易结构（含循环购买）

```
消费金融公司（原始权益人+资产服务机构）
    │
    │ 拥有的消费贷款债权
    ▼
特定目的信托（SPV）
    │
    ├── 循环购买期内：回收款 → 购买新贷款
    │
    └── 摊还期内：回收款 → 按Waterfall分配
            │
            ▼
        投资者（ABS持有人）
```

**结构要点**：
- 循环期（如18个月）+ 摊还期（如12个月）
- 循环购买设置合格标准（新入池贷款须符合预设条件）
- 统计模型驱动（历史数据丰富，逐笔核查不现实）

## 五、Gate Check：方案设计阶段拦停规则

| # | 拦停情形 | 触发级别 | 恢复条件 |
|---|---------|---------|---------|
| GC-S01 | 基础资产合规性问题未解决（如属负面清单） | 🔴终止 | 不适用 |
| GC-S02 | 分层方案无法达到目标评级（AAA/AA+） | 🟡中止 | 增加增信强度或调整分层比例 |
| GC-S03 | 现金流模型显示存在缺口（覆盖倍数<1.0x） | 🔴终止 | 重新构建资产池（增加规模或提高资产质量） |
| GC-S04 | 交易结构存在法律瑕疵（真实出售/破产隔离论证不足） | 🔴终止 | 重构SPV结构或补充法律论证 |
| GC-S05 | 增信方资质不足（如主体评级低于AA） | 🟡中止 | 更换增信方或增加其他增信措施 |

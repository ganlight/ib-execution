# SKILL_01 ABS交易结构设计

> ABS交易结构核心设计规范——分层、现金流分配、循环购买与信用触发机制

---

## 一、交易结构核心要素

### 1.1 交易结构五步链

```
① SPV设立
    │  特定目的信托 / 资产支持专项计划
    │
    ▼
② 资产购买
    │  原始权益人将基础资产转让给SPV
    │  签署《资产买卖协议》
    │
    ▼
③ 证券发行
    │  SPV向投资者发行优先级证券
    │  优先A / 优先B / 次级（如需要，可设更多档）
    │
    ▼
④ 现金流归集与分配
    │  收款→监管账户→信托账户→按Waterfall分配
    │
    ▼
⑤ 增信机制
    │  优先/次级分层 + 超额抵押/利差 + 外部增信
    │
    ▼
  完成交易结构闭环
```

### 1.2 各交易方角色

| 交易方 | 角色 | 核心权利义务 |
|--------|------|------------|
| 原始权益人 | 资产出售方 + 资产服务机构 | 出售资产、持续管理回收、提供基础资产管理报告 |
| 计划管理人/受托机构 | SPV管理人 | 设立和管理SPV、组织发行、执行分配指令 |
| 投资者 | 证券持有人 | 认购支付、获得本息偿付、行使持有人大会表决权 |
| 托管银行 | 资金保管方 | 保管信托财产/专项计划财产中的货币资金 |
| 监管银行 | 资金监管方 | 监管原始权益人收款账户的资金归集 |
| 评级机构 | 信用质量独立评估方 | 初始评级、跟踪评级 |
| 增信方（如适用） | 信用支持提供方 | 按约定条件提供差额支付/担保/流动性支持 |

## 二、优先/次级分层设计

### 2.1 标准三层结构

```
┌────────────────────────────────────┐
│         优先A档（Senior A）          │  ← 占比 {70-80}%
│         目标评级：AAA                │
├────────────────────────────────────┤
│         优先B档（Senior B）          │  ← 占比 {10-20}%
│         目标评级：AA+ ~ AA           │
├────────────────────────────────────┤
│         次级档（Subordinate）        │  ← 占比 {5-10}%
│         无评级（通常由原始权益人持有） │
└────────────────────────────────────┘

信用支持顺序：次级档 → 优先B档 → 优先A档
分配顺序：优先A档 → 优先B档 → 次级档
```

### 2.2 分层比例确定方法

**Step 1：确定目标评级**

| 目标评级 | 所需信用支持量（参考范围） |
|---------|----------------------|
| AAA | {15-25}% |
| AA+ | {10-18}% |
| AA | {7-12}% |
| AA- | {5-8}% |

**Step 2：信用支持量 = 次级档比例 + 超额抵押/利差**

```
信用支持量（%） = （次级档本金规模 / 总资产池规模）× 100% + 超额抵押覆盖率 + 超额利差折现覆盖
```

**Step 3：反向计算分层比例**

```
次级档规模 = 优先A档所需信用支持 - 超额抵押 - 超额利差折现贡献

优先B档规模 = 总资产池 - 次级档 - 优先A档（优先B档本身也对优先A档形成信用支持）

优先A档规模 = 总资产池 - 优先B档 - 次级档
```

### 2.3 分层设计常见问题

| 问题 | 解决方案 |
|------|---------|
| 信用支持量不足（无法达到目标AAA） | 增加次级档厚度 / 增加超额抵押 / 引入外部增信 |
| 次级档过薄（原始权益人无力持有） | 引入第三方投资者认购次级 / 采用超额抵押替代部分次级 |
| 分层过多（超过4档） | 简化分层（过多分层增加结构复杂度和投资者决策成本） |

## 三、现金流分配机制（Waterfall）

### 3.1 标准Waterfall分配顺序

```
每期可供分配资金
    │
    ▼
① 支付各项税费（印花税、增值税等）
    │
    ▼
② 支付中介机构费用（管理费、托管费、评级费、律师费等）
    │
    ▼
③ 支付优先A档当期利息（当期应付利息 = 优先A档本金余额 × 票面利率 × 计息天数/365）
    │
    ▼
④ 支付优先A档本金（直至全部清偿）
    │
    ▼
⑤ 支付优先B档当期利息
    │
    ▼
⑥ 支付优先B档本金（直至全部清偿）
    │
    ▼
⑦ 如有差额支付承诺/担保等外部增信义务，履行相关偿付
    │
    ▼
⑧ 剩余收益分配至次级档持有人
```

### 3.2 Waterfall关键参数

| 参数 | 说明 | 典型设置 |
|------|------|---------|
| 分配频率 | 现金分配的时间间隔 | 按月 / 按季 / 按半年 |
| 利息计算方式 | 固定利率 / 浮动利率 | 按天计息 / 按实际天数/360 / 按实际天数/365 |
| 本金偿还方式 | 过手型 / 计划摊还型 / 到期一次偿还 | 过手型（Pass-through）最常用 |
| 超额利差处理 | 是否作为信用支持、是否返还原始权益人 | 初期锁定为信用支持，优先档还清后方返还 |

### 3.3 过手型 vs 计划摊还型

| 偿还方式 | 特征 | 适用资产 | 优缺点 |
|---------|------|---------|--------|
| 过手型 | 每期可供分配资金全部用于偿还证券（扣除费用后），不设计划偿还金额 | 多数资产类型 | 优点：自然匹配；缺点：每期偿还金额不确定 |
| 计划摊还型 | 每期按预定计划偿还，不足部分由信用支持补足 | CMBS/类REITs | 优点：投资者可预期现金流；缺点：需强信用支持保障 |
| 到期一次还本 | 存续期内仅支付利息，到期一次性偿还本金 | 短期供应链ABS | 优点：结构简单；缺点：可能与资产端分期回款不匹配 |

## 四、循环购买结构

### 4.1 适用场景

循环购买结构适用于**基础资产期限短于证券期限**的情形，典型如：

- 消费金融ABS（消费贷款期限3-12个月，证券期限2-3年）
- 信用卡分期ABS
- 小额贷款ABS

### 4.2 循环购买机制

```
     循环期（Revolving Period）           摊还期（Amortization Period）
    ┌─────────────────────────────┐    ┌─────────────────────────┐
    │                             │    │                         │
    │  基础资产到期回款            │    │  基础资产到期回款         │
    │       │                     │    │       │                  │
    │       ▼                     │    │       ▼                  │
    │  购买新基础资产              │    │  按Waterfall分配         │
    │  （满足合格标准）            │    │  （不再购买新资产）       │
    │                             │    │                         │
    │  仅付各档利息               │    │  同时偿还本金+利息        │
    │                             │    │                         │
    └─────────────────────────────┘    └─────────────────────────┘
     通常12-18个月                      通常12个月
```

### 4.3 循环购买额合格标准（Eligibility Criteria）

新购入的每笔资产须同时满足以下条件：

| 标准 | 内容 | 阈值 |
|------|------|------|
| 债务人信用 | 在原始权益人内部授信体系内达合格等级 | ≥{合格级} |
| 无逾期状态 | 截至购买日不存在逾期 | 逾期天数=0 |
| 贷款期限 | 新贷款到期日不应超出摊还期结束日 | ≤{摊还期结束日-X个月} |
| 利率 | 新贷款加权平均利率须达标 | ≥{X}% |
| 分散度 | 不因新增资产导致集中度超标 | 符合原始入池标准 |

### 4.4 循环购买终止触发事件

| 触发事件 | 后果 | 触发条件 |
|---------|------|---------|
| 加速清偿事件 | 立即停止循环购买，转为加速摊还 | 基础资产累计违约率超限 |
| 违约事件 | 立即停止循环购买，按Waterfall加速清偿 | SPV违约/原始权益人破产等 |
| 合格资产不足 | 暂停或终止循环购买 | 连续{X}个分配期合格资产增量不足 |
| 服务机构更换事件 | 暂停或终止循环购买 | 原始权益人信用恶化至不再满足服务机构条件 |

## 五、信用触发机制

### 5.1 加速清偿事件（Accelerated Amortization Events）

**触发条件**：

| # | 触发条件 | 量化阈值 |
|---|---------|---------|
| 1 | 资产池累计违约率超过阈值 | >{X}%（通常与评级机构协商确定） |
| 2 | 原始权益人主体评级下调至某档以下 | 从{A}级下调至{A-}级或以下 |
| 3 | 服务机构未能及时转付回收款 | 连续{3}个分配期延迟 |
| 4 | 基础资产回款现金流持续低于预期 | 连续{3}个月低于预测值{80}% |
| 5 | 原始权益人/增信方发生重大不利变化 | 破产/接管/核心牌照被吊销 |

**法律后果**：
- 立即停止循环购买（如适用）
- 所有回收款不再用于购买新资产，全部按Waterfall顺序加速偿还
- 分配频率可从按季提升至按月

### 5.2 违约事件（Events of Default）

**触发条件**（通常比加速清偿事件更严重）：

| # | 触发条件 |
|---|---------|
| 1 | 任何一档优先证券在到期日未能按时足额偿付本息 |
| 2 | SPV违反信托/专项计划合同实质性约定 |
| 3 | 计划管理人/受托机构丧失资格或被吊销牌照 |
| 4 | 法院宣告SPV进入破产程序 |

**法律后果**：
- 宣布全部未偿还证券立即到期应付
- 变更Waterfall分配顺序：先偿付优先A档全部未清偿本金+利息，再偿付优先B档全部未清偿本金+利息
- 授权受托人/持有人大会采取加速清偿行动

## 六、回收款转付频率

### 6.1 转付机制

原始权益人作为资产服务机构将归集的回收款从监管账户划转至信托/托管账户。

### 6.2 转付频率选择依据

| 转付频率 | 优点 | 缺点 | 适用情形 |
|---------|------|------|---------|
| 按月转付 | 资金停留短，混同风险低 | 操作频率高 | 原始权益人信用偏弱 |
| 按季转付 | 操作成本适中 | 混同风险适中 | 标准配置 |
| 按半年转付 | 操作简便 | 混同风险高 | 原始权益人信用很高（如AAA级核心企业） |

### 6.3 转付频率调整触发

当原始权益人/服务机构信用等级下调时，自动缩短转付频率：

| 服务机构评级 | 转付频率 |
|------------|---------|
| AAA ~ AA+ | 按半年 → 按季 |
| AA ~ AA- | 按季 → 按月 |
| A+及以下 | 按月 → 按周，甚至按日 |

## 七、典型结构案例

### 7.1 案例一：供应链ABS

**基本参数**：
- 原始权益人：XX商业保理有限公司
- 核心债务人：XX大型房地产/建筑集团（主体评级{AAA}）
- 基础资产：保理公司持有的对核心企业的应收账款
- 资产池规模：{10}亿元
- 证券期限：{1.5}年

**分层结构**：

| 证券档级 | 规模（亿元） | 占比 | 预期评级 | 期限 |
|---------|------------|------|---------|------|
| 优先A档 | {9.2} | 92% | AAA | 1.5年 |
| 次级档 | {0.8} | 8% | 无评级 | 1.5年 |

**结构特点**：
- 依赖核心企业主体信用，无需更多内部分层
- 无循环购买（应收账款期限≥证券期限）
- 加速清偿触发条件：核心企业主体评级下调至AA以下
- 原始权益人同时提供差额支付承诺

### 7.2 案例二：融资租赁ABS

**基本参数**：
- 原始权益人：XX融资租赁有限公司（主体评级{AA}）
- 基础资产：融资租赁合同项下租金债权（含租赁物所有权）
- 资产池规模：{20}亿元
- 入池资产：{150}笔租赁合同，平均单笔{1,333}万元
- 证券期限：优先A档{3}年，优先B档{3}年，次级档{4}年

**分层结构**：

| 证券档级 | 规模（亿元） | 占比 | 预期评级 | 票面利率（参考） |
|---------|------------|------|---------|---------------|
| 优先A档 | {15.0} | 75% | AAA | {3.0}%~{3.5}% |
| 优先B档 | {3.0} | 15% | AA+ | {3.8}%~{4.5}% |
| 次级档 | {2.0} | 10% | 无评级 | — |

**信用支持量**：优先A档获得优先B+次级共{25}%的信用支持（= {3.0+2.0}/{20.0}）

**结构特点**：
- 超额利差约{2}%锁定为信用支持
- 按月转付回收款（原始权益人AA评级背景下降低混同风险）
- 加速清偿触发：基础资产累计违约率>{5}%
- 原始权益人持有全部次级档（风险自留要求）

# SKILL_02 ABS信用增信设计

> ABS信用增信机制设计规范——内部增信与外部增信的体系化方案

---

## 一、信用增信总览

### 1.1 信用增信在ABS中的定位

```
基础资产池信用质量（初始）
        │
        ▼
    ┌──────────────┐
    │  信用增信     │ ← 提升信用质量至目标评级
    │  Enhancement  │
    └──────────────┘
        │
        ▼
  分层证券信用质量（增信后）
        │
   ┌────┼────┐
   ▼    ▼    ▼
 优先A 优先B 次级
```

### 1.2 增信措施全景

```
信用增信体系
    │
    ├── 内部增信（Internal Credit Enhancement）
    │      ├── ① 优先/次级分层（Senior/Subordinated Structure）
    │      ├── ② 超额抵押（Over-Collateralization, OC）
    │      ├── ③ 超额利差（Excess Spread）
    │      ├── ④ 现金储备账户（Cash Reserve Account）
    │      └── ⑤ 信用触发机制（Credit Triggers）
    │
    └── 外部增信（External Credit Enhancement）
           ├── ① 差额支付承诺（Deficiency Payment Undertaking）
           ├── ② 第三方担保（Third Party Guarantee）
           ├── ③ 流动性支持（Liquidity Facility）
           └── ④ 维好协议（Keepwell Deed）
```

## 二、内部增信措施

### 2.1 优先/次级分层（最核心内部增信）

**机制**：次级档本金先行承担资产池的违约损失，为优先级证券提供保护。

**量化设计**：

| 目标评级 | 所需信用支持量 | 相当于次级比例（纯分层） |
|---------|-------------|---------------------|
| AAA | {15-25}% | 次级档占总资产池 {15-25}% |
| AA+ | {10-18}% | 次级档占总资产池 {10-18}% |
| AA | {7-12}% | 次级档占总资产池 {7-12}% |
| AA- | {5-8}% | 次级档占总资产池 {5-8}% |

**设计要点**：
- 信用支持量须覆盖评级压力测试的极端违约损失
- AAA评级的压力测试标准通常是违约率×回收率缺口的最差情况（相当于经济危机情景）
- 分层不宜过少（优先A/次级仅2档结构保持简单但信用支持量可能浪费），不宜过多（超过4档增加复杂性）

### 2.2 超额抵押（OC）

**机制**：基础资产池本金规模 > 证券发行规模，差额部分形成超额抵押，提供额外信用保护。

**量化设计**：

```
超额覆盖率 = (资产池总本金 - 证券发行总规模) / 证券发行总规模 × 100%
```

| 资产类型 | 典型超额覆盖率 |
|---------|-------------|
| 融资租赁 | ≥{5}% |
| 消费金融 | ≥{5-10}% |
| 供应链 | 0~{5}%（风险较小） |
| CMBS | ≥{5}% |

**设计要点**：
- OC比例过高会增加原始权益人资金成本（资产出表效益下降）
- OC由原始权益人提供，相当于原始权益人承担"第一损失"风险
- 在优先级偿付后，OC对应的超额回收款可返还原始权益人

### 2.3 超额利差（Excess Spread）

**机制**：基础资产加权平均利率 > 证券端加权平均利率 + 各项费用率，形成利差保护。

**计算公式**：

```
超额利差 = 资产池加权平均利率 - 证券端加权平均利率 - 年化费用率
```

**量化阈值**：

| 超额利差水平 | 信用保护强度 | 是否可独立用于增信 |
|------------|------------|-----------------|
| ≥{2}% | 强 | 可作为主要增信手段之一 |
| {1-2}% | 适中 | 可作为辅助增信手段 |
| <{1}% | 弱 | 不可独立用于增信，须配合其他手段 |

**设计要点**：
- 超额利差是每期产生的"流量"，而非"存量"（与OC不同）
- 通常将超额利差首先用于补充OC下降或覆盖当期违约损失
- 延期支付模式下：超额利差累积在现金储备账户，形成额外保护

### 2.4 现金储备账户（Cash Reserve Account）

**机制**：从发行收入中提取一定比例或固定金额存入独立现金账户，用于弥补未来现金流缺口。

**设计要点**：

| 参数 | 典型设置 |
|------|---------|
| 初始提取比例 | 发行总规模的 {1-3}% |
| 目标余额 | 等于下一期应付利息（或本金+利息）的 {1-3} 倍 |
| 补足机制 | 每期分配前从回收款中提取差额补至目标余额 |
| 动用条件 | 当期可供分配资金不足以支付优先级利息 |
| 释放条件 | 优先级证券全部清偿后释放 |

### 2.5 信用触发机制（已在交易结构设计中详述）

此处补充增信视角：信用触发机制通过动态调整Waterfall分配顺序或加速机制，在资产质量恶化时提前"切换至保护模式"，本质上是一种**条件性增信**。

| 触发机制类型 | 增信效果机制 | 实施方式 |
|------------|------------|---------|
| 加速清偿事件 | 停止循环购买→加速还本 | 变更Waterfall：跳过循环购买，直接偿还优先级 |
| 违约事件 | 宣布提前到期→变更分配顺序 | Waterfall变为：优先A全额 → 优先B全额 |
| 资产质量恶化触发 | 增加现金储备余额要求 | 超额利差不再释放，全部锁定 |

## 三、外部增信措施

### 3.1 差额支付承诺

**定义**：承偌人（通常为原始权益人或其母公司）承诺在可供分配资金不足以支付优先级证券本息时，补足差额。

**量化设计**：

- 承诺额度：通常为优先级证券未偿还本金总额的{100}%（全额承诺）
- 承诺期限：覆盖至优先级证券全部清偿为止
- 承诺人的信用能力：主体评级≥{AA}

**设计要点**：
- 差额支付承诺不是担保，而是单方承诺，法律执行力弱于担保
- 差额支付承诺的信用效力等同于承诺人的主体信用（AAA承诺人→AAA信用保护）
- 若承诺人信用降级，须触发备选增信措施（如更换承诺人）

### 3.2 第三方担保

**定义**：第三方（非原始权益人的独立主体，如专业担保公司/母公司）向SPV提供保证担保，承诺在特定条件下代为清偿。

**量化设计**：

| 担保参数 | 典型设置 |
|---------|---------|
| 担保覆盖范围 | 全部优先级证券本息 / 仅优先级利息 |
| 担保额度 | 优先级证券总额的{100}% |
| 担保期限 | 覆盖至优先级证券全部清偿后{6}个月 |
| 担保人评级要求 | 不低于{AAA}（如目标优先级AAA） |

**法律形式**：保证合同（受《民法典》担保制度约束）+ 担保人董事会/股东会授权

### 3.3 流动性支持

**定义**：流动性支持方承诺在SPV出现临时性流动性缺口（非信用损失）时提供过桥资金。

**与差额支付承诺的区别**：

| 维度 | 差额支付承诺 | 流动性支持 |
|------|-----------|----------|
| 触发条件 | 可供分配资金不足以支付优先级本息 | 当期现金流暂时性不足（但资产质量尚未恶化） |
| 信用损失覆盖 | 覆盖 | 不覆盖（信用损失须由其他增信措施覆盖） |
| 偿付来源 | 承诺人自有资金 | 支持方垫付，后续由基础资产回款优先偿还 |
| 法律性质 | 单方承诺 | 借款合同（垫付+归还） |

### 3.4 维好协议（Keepwell Deed）

**定义**：维好协议出具人承诺将促使被支持方（原始权益人/SPV）保持良好财务状况和履约能力，但不构成直接清偿义务。

**法律效力**：
- ❌ 不构成担保
- ❌ 不构成直接支付义务
- ⚠️ 仅具有道义约束力
- ⚠️ 法律强制执行难度大

**使用场景**：
- 适用于跨境交易中国境外主体无法提供境内担保
- 通常不单独作为信用增信手段，须配合其他实质性增信

## 四、增信措施的组合策略

### 4.1 不同基础资产类型的增信组合推荐

| 基础资产类型 | 内部增信组合 | 外部增信补充 | 评级目标 |
|------------|------------|------------|---------|
| 融资租赁 | 优先/次级分层（信用支持量{15-20%}）+ 超额利差 | 差额支付承诺（原始权益人） | 优先A AAA |
| 供应链（反向保理） | 优先/次级分层（信用支持量{5-10%}） | 差额支付承诺（核心企业/母公司） | 优先A AAA |
| 消费金融 | 优先/次级分层（信用支持量{15-25%}）+ 超额利差 + 现金储备账户 | 流动性支持 | 优先A AAA |
| CMBS | 优先A/优先B/次级三层（信用支持量{20-30%}） | 差额支付承诺 + 物业抵押担保 | 优先A AAA, 优先B AA+ |
| 类REITs | 优先A/优先B/次级三层 + 超额抵押 | 维好协议+优先收购权 | 优先A AAA |
| 收费收益权 | 优先/次级分层 + 超额利差 | 差额支付承诺 | 优先A AA+ |

### 4.2 增信强度评估矩阵

| 增信措施 | 保护强度 | 评级贡献 | 法律确定性 | 原始权益人成本 |
|---------|---------|---------|-----------|-------------|
| 优先/次级分层 | ⭐⭐⭐⭐⭐ | 极高 | 高 | 中等（次级自留） |
| 超额抵押 | ⭐⭐⭐⭐ | 高 | 高 | 高（锁定资金） |
| 超额利差 | ⭐⭐⭐ | 中 | 中 | 低 |
| 现金储备账户 | ⭐⭐⭐ | 中 | 高 | 中等 |
| 差额支付承诺 | ⭐⭐⭐⭐ | 高 | 中 | 低（名义）→高（触发时） |
| 第三方担保 | ⭐⭐⭐⭐⭐ | 极高 | 高 | 高（担保费） |
| 流动性支持 | ⭐⭐ | 较低 | 中 | 低 |
| 维好协议 | ⭐ | 极低 | 低 | 低 |

## 五、增信措施的法律效力与可执行性评估

### 5.1 日常监管

| 增信措施 | 法律依据 | 可执行性 | 时间效率 |
|---------|---------|---------|---------|
| 优先/次级分层 | 合同约定+信托/专项计划文件 | ✅ 自动执行（合同安排的固有机制） | 即时 |
| 超额抵押 | 合同约定 | ✅ 自动（OC账户由托管行控制） | 即时 |
| 差额支付承诺 | 单方承诺函 | ⚠️ 须通过诉讼/仲裁确认后执行 | 3-12个月 |
| 第三方担保 | 《民法典》保证合同 | ✅ 可直接申请实现担保权利 | 1-6个月 |
| 流动性支持 | 借款合同 | ✅ 按合同条件执行 | 1-4周 |

### 5.2 可执行性风险缓释

| 风险 | 缓释措施 |
|------|---------|
| 增信方拒不履行 | 在相关协议中预先授权SPV/受托人直接操作资金划转 |
| 增信方信用恶化 | 设置增信方评级下调触发机制（如更换增信方或增加担保物） |
| 法律效力争议 | 在交易文件中明确约定仲裁条款，缩短争议解决时间 |

## 六、监管对增信的要求

### 6.1 风险自留规则

原始权益人原则上须持有不低于总发行规模{5}%的次级档证券（风险自留），以保持利益一致性。豁免情形：
- 基础资产风险极低（如核心企业AAA级供应链ABS，由监管认可豁免）
- 已存在其他强信用增信（如第三方AAA担保）

### 6.2 增信信息披露要求

- 增信方主体信用状况及最近一年财务摘要须在计划说明书中列示
- 差额支付承诺/担保合同中所有实质性条款须完整披露
- 增信措施的局限性和风险须在风险因素章节揭示

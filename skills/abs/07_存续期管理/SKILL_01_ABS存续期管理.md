# SKILL_01 ABS存续期管理

> ABS存续期管理详细执行规范——从报告审核到风险应对的全流程操作手册

---

## 一、资产服务机构报告审核

### 1.1 月度资产服务机构报告

报告由资产服务机构（通常为原始权益人）按月度编制并提交。

**报告内容核查清单**：

| # | 核查项 | 核查内容 | 关注点 |
|---|--------|---------|--------|
| 1 | 资产池概况 | 期末资产笔数、本金余额、加权平均利率 | 与上期对比，是否合理变动 |
| 2 | 还款明细 | 当期本金回收、利息回收、提前还款金额 | 与预期现金流预测对比 |
| 3 | 逾期和违约情况 | 各逾期天数档（30+/60+/90+）的资产金额和笔数 | 环比变化趋势 |
| 4 | 违约资产处置和回收 | 当期已处置违约资产的回收金额和回收率 | 回收率是否偏离历史水平 |
| 5 | 资产池分散度变化 | 债务人/行业/地域集中度更新 | 是否触发集中度变化预警 |
| 6 | 合格资产标准 | 循环购买期内新购入资产的合格率 | 合格率≥{100}% |
| 7 | 触发机制监控 | 是否触发加速清偿/违约事件的条件 | 实时监控 |
| 8 | 服务机构信用变化 | 服务机构最新主体评级和经营概况 | 如降级须启动评估 |

**审核流程**：

```
资产服务机构提交月度报告
    │
    ▼
计划管理人初审（{5}个工作日）
    │  ├── 数据逻辑审核（与上月报告/原始预期对比）
    │  ├── 异常数据标注（如某指标大幅偏离预期）
    │  └── 编制《月度审核备忘录》
    │
    ▼
如发现异常：
    │  ├── 通知资产服务机构解释异常
    │  ├── 评估是否需要触发临时重大事项披露
    │  └── 如触发信用事件，启动相应程序
    │
    ▼
审核通过后发布（计划管理人网站/指定渠道）
```

### 1.2 季度托管报告

托管银行每季度出具托管报告。

| # | 核查项 | 核查内容 |
|---|--------|---------|
| 1 | 信托/专项计划账户余额 | 期初、期末余额及变动 |
| 2 | 当期资金流入明细 | 基础资产回收款、投资收益等 |
| 3 | 当期资金流出明细 | 费用支付、证券本息偿付、循环购买支出 |
| 4 | 证券本金余额 | 各档证券的未偿还本金变化 |
| 5 | 现金储备账户余额 | 如设有现金储备账户，报告期末余额 |
| 6 | 托管行执行的分配指令 | 是否均按Waterfall规定执行 |
| 7 | 托管行声明 | 托管行对报告期间履职的声明 |

## 三、现金流分配执行监督

### 3.1 分配执行复核步骤

每期分配日前后，计划管理人须按以下步骤复核托管银行的分配执行：

```
Step 1: 核算当期可供分配资金总额
   可供分配资金 = 当期回收款 + 现金储备账户动用（如有）+ 差额支付承诺动用（如有）

Step 2: 按Waterfall模拟分配
   ① 税费
   ② 中介费用
   ③ 优先A档当期利息
   ④ 优先A档本金（过手）
   ⑤ 优先B档当期利息
   ⑥ 优先B档本金（过手）
   ⑦ 增信义务履行（如有）
   ⑧ 次级收益分配

Step 3: 对比模拟分配结果与托管银行的实际分配结果
   - 各档利息金额是否一致
   - 各档本金偿付金额是否一致
   - 如有现金储备账户补充/回流，金额是否正确

Step 4: 输出《分配复核报告》
   - 记录复核结果
   - 如有差异，说明原因和处理方案
```

### 3.2 常见分配异常及处理

| 异常情形 | 原因诊断 | 处理方案 |
|---------|---------|---------|
| 可供分配资金低于预期 | 回收款延迟/违约率上升 | 必要时启动差额支付承诺 |
| 分配结果与Waterfall不一致 | 托管行误操作 | 要求托管行更正并补回差额 |
| 某档利息不足支付 | 资产池表现恶化 | 启动增信措施（差额支付/现金储备动用） |
| 现金流提前耗尽 | 违约大幅上升 | 评估是否已触发信用事件 |

## 四、信用触发事件监控

### 4.1 监控指标体系（含阈值）

| 监控指标 | 警告阈值 | 触发阈值 | 监控频率 |
|---------|---------|---------|---------|
| 累计违约率 | {X}% 的 {80}%（逼近警告线） | 达到{X}% | 月度 |
| 连续X个月违约率上升 | 连续{2}个月 | 连续{3}个月 | 月度 |
| 原始权益人/服务机构主体评级下调 | 下列至{AA-} | 下降至{A+}及以下 | 实时（评级变动日） |
| 转付延迟 | 延迟{1}个工作日 | 延迟{3}个工作日或连续{2}期延迟 | 每期分配日 |
| 回款低于预期 | 低于预测值{90}% | 连续{3}个月低于{80}% | 月度 |

### 4.2 信用触发事件启动程序

```
触发事件发生
    │
    ▼
计划管理人{24}小时内确认触发事实
    │
    ▼
通知各方（{2}个工作日内）
    │  ├── 投资者（持有人）→ 重大事项临时公告
    │  ├── 评级机构 → 启动降级审查
    │  ├── 原始权益人/增信方
    │  ├── 监管机构（交易所/NAFMII/银保监）
    │  └── 托管银行
    │
    ▼
启动触发机制
    │  ├── 加速清偿：停止循环购买，转为加速摊还
    │  └── 违约事件：变更Waterfall分配顺序
    │
    ▼
召开持有人会议（如必要）
    │  └── 讨论后续处置方案
    │
    ▼
执行后续行动
```

## 五、持有人会议召开程序

### 5.1 五步流程

```
① 召集（{10}个工作日）
    ├── 召集人：计划管理人（或≥{10}%份额持有人提议）
    ├── 召集条件：须修改交易合同核心条款/更换服务机构/宣布违约/其他重大事项
    └── 召集人拟定会议议题

② 通知（{10}个工作日）
    ├── 召集人向全体持有人发出书面通知
    ├── 通知内容：会议时间/地点/议题/参会方式/表决权计算
    └── 通知方式：公告+直接通知（如可获持有人联系信息）

③ 议案（{5}个工作日）
    ├── 持有人或管理人可在会议前提出修改/补充议案
    └── 最终议案在会议前{3}个工作日内确定

④ 表决（会议当日 ~ 会后{5}个工作日）
    ├── 出席份额须达：{50}%（一般决议）/ {66.67}%（特殊决议）
    ├── 同意比例须达：出席份额的{50}%以上（一般）/ {66.67}%以上（特殊）
    └── 表决方式：现场+书面投票

⑤ 执行（决议通过后）
    ├── 决议通过后{10}个工作日内执行
    ├── 如涉及变更服务机构：过渡期合理安排
    └── 将决议和执行情况向监管报告
```

### 5.2 需持有人会议决议的典型事项

| 事项 | 决议类型 | 最低出席线 | 最低同意线 |
|------|---------|-----------|-----------|
| 变更计划管理人 | 特殊决议 | ≥{66.67}% | ≥出席份额的{66.67}% |
| 变更资产服务机构 | 特殊决议 | ≥{66.67}% | ≥出席份额的{66.67}% |
| 提前终止专项计划 | 特殊决议 | ≥{66.67}% | ≥出席份额的{66.67}% |
| 修改Waterfall分配顺序 | 特殊决议 | ≥{66.67}% | ≥出席份额的{66.67}% |
| 变更投资者保护条款 | 特殊决议 | ≥{66.67}% | ≥出席份额的{66.67}% |
| 修改信息披露安排（非实质性） | 一般决议 | ≥{50}% | ≥出席份额的{50}% |
| 审议年度资产管理报告 | 一般决议 | ≥{50}% | ≥出席份额的{50}% |

## 七、资产置换/赎回/清算的特殊处理

### 7.1 资产置换

原始权益人按合同约定，以合格资产置换不合格/问题资产。

| 环节 | 操作要求 |
|------|---------|
| 触发条件 | 入池资产被认定为不合格（如权属瑕瘫、超过集中度限制） |
| 置换资产标准 | 须满足原始入池标准，且加权平均质量不低于被置换资产 |
| 置换执行 | 被置换资产从资产池中移除，置换资产纳入 |
| 置换对价 | 若置换资产价值低于被置换资产，原始权益人须补付差价 |
| 信息披露 | 置换执行前{10}个工作日公告 |

### 7.2 资产赎回

原始权益人按合同约定赎回基础资产（全额或部分）。

| 环节 | 操作要求 |
|------|---------|
| 赎回方式 | 全额赎回（清仓赎回，Clean-up Call） / 部分赎回 |
| 全额赎回条件 | 通常为资产池余额降至初始规模{10}%以下时触发 |
| 赎回对价 | 足以支付剩余全部证券未偿本息（含已到期未付） |
| 执行程序 | 持有人会议表决 → 计划管理人执行 |

### 7.3 专项计划清算

专项计划到期或宣布提前到期后，进入清算流程。

```
清算启动
    │
    ▼
公告专项计划终止/清算
    │
    ▼
基础资产处置（如尚有未变现资产）
    │  └── 以合理方式变现（拍卖/转让）
    │
    ▼
清算分配（按Waterfall最终分配顺序）
    │  ├── ① 清算费用
    │  ├── ② 应交税费
    │  ├── ③ 未付中介费用
    │  ├── ④ 优先A档未偿本金+利息
    │  ├── ⑤ 优先B档未偿本金+利息
    │  └── ⑥ 剩余归次级持有人
    │
    ▼
清算报告公布
    │
    ▼
专项计划完成注销
```

## 八、年度压力测试更新

### 8.1 更新要求

- **更新频率**：每年至少重做一次
- **更新时点**：结合年度审计报告和年度资产管理报告的编制期
- **更新内容**：
  - 以最新资产池数据替换初始发行时的假设
  - 以最新历史表现数据（违约率、早偿率、回收率）更新参数
  - 以最新市场/利率环境调整压力情景参数
- **更新目标**：验证在当前资产池表现下，信用增信是否仍充足

### 8.2 更新步骤

```
Step 1：数据更新
    ├── 更新资产池概况（最新期末数据）
    ├── 更新历史表现指标（更新至最新一期）
    └── 更新资产池影子评级

Step 2：参数校准
    ├── 以历史表现重新校准违约率
    ├── 以历史表现重新校准早偿率
    └── 以历史表现重新校准回收率

Step 3：压力测试
    ├── 保留原有压力情景结构
    ├── 以校准后参数重新运行9种情景
    └── 关注最早触发信用支持的时点和幅度

Step 4：结论输出
    ├── 各档证券覆盖倍数是否仍达标
    ├── 信用支持是否仍充足
    ├── 是否需要调整增信措施
    └── 出具《年度压力测试更新报告》
```

### 8.3 压力测试恶化的应对

| 恶化程度 | 应对措施 |
|---------|---------|
| 轻度恶化（覆盖倍数下降<{10}%） | 记录+跟踪，暂不行动 |
| 中度恶化（覆盖倍数下降{10-30}%） | 向评级机构报告+评估是否需要增加增信 |
| 重度恶化（覆盖倍数下降≥{30}%或跌破{1.0}倍） | 启动信用触发机制评估+召开持有人会议讨论 |

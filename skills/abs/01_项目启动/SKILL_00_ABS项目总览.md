# SKILL_00 ABS项目总览

> 资产支持证券（ABS）产品线全景概览

---

## 一、ABS定义与核心理念

### 1.1 定义

资产支持证券（Asset-Backed Securities，ABS）是指以基础资产所产生的未来现金流为偿付支持，通过结构化设计进行信用增级，在此基础上发行的资产支持证券。

核心逻辑：**资产信用 > 主体信用**——投资者依赖基础资产池产生的现金流而非原始权益人的主体信用来获取本息偿付。

### 1.2 核心理念：结构化融资三要素

| 要素 | 说明 |
|------|------|
| **资产真实出售** | 原始权益人将基础资产合法、有效地转让给SPV，实现破产隔离 |
| **信用增级** | 通过内部/外部增信将基础资产信用质量提升至目标评级 |
| **现金流匹配** | 基础资产现金流入与证券端本息偿付在金额和时点上匹配 |

## 二、中国市场三大ABS体系

### 2.1 体系对比

| 维度 | 信贷ABS | 企业ABS | ABN（资产支持票据） |
|------|---------|---------|---------------------|
| **监管机构** | 银保监会+央行 | 证监会+交易所 | NAFMII（交易商协会） |
| **审核模式** | 备案制（首次注册+分期发行） | 交易所挂牌条件确认（类审核制） | 注册制（NAFMII注册） |
| **SPV形式** | 特定目的信托 | 资产支持专项计划 | 特定目的信托/公司/专项计划均可 |
| **管理人** | 信托公司 | 证券公司/基金子公司 | 信托公司/证券公司/基金子公司 |
| **典型资产** | 个人住房贷/车贷/消费贷/不良资产 | 供应链/融资租赁/收费收益权/CMBS | 融资租赁/供应链/应收账款 |
| **投资者** | 银行间市场合格机构 | 交易所合格投资者 | 银行间+交易所（双市场） |
| **挂牌场所** | 银行间债券市场 | 沪深交易所 | 银行间债券市场 |
| **上位法** | 《信托法》+《信贷资产证券化试点管理办法》 | 《证券法》+《证券公司及基金管理公司子公司资产证券化业务管理规定》 | 《银行间债券市场非金融企业债务融资工具管理办法》 |

## 三、基础资产类型（7大类）

### 3.1 应收账款类

- **商业保理ABS**：以保理公司持有的应收账款债权为基础
- **供应链ABS**：反向保理模式，核心企业确认应付、供应商转让应收
- 关键特征：单笔金额适中、期限较短（通常3-12个月）、债务人信用为核心

### 3.2 租赁债权类

- **融资租赁ABS**：以租赁合同项下租金请求权为基础
- 关键特征：单笔金额较大、期限较长（通常3-7年）、分散度需重点关注

### 3.3 供应链类

- 以核心企业对上游供应商的应付账款为基础
- 典型结构：1+N反向保理——1个核心债务人+N个原始债权人
- 关键特征：债务人集中度极高、信用取决于核心企业

### 3.4 收费收益权类

- **高速公路收费权ABS**、**供水收费权ABS**、**供热收费权ABS**
- 关键特征：未来现金流预测是关键、无明确债务人、运营风险需考量

### 3.5 消费金融类

- **消费贷ABS**、**信用卡分期ABS**、**花呗/借呗ABS**
- 关键特征：小额分散、历史数据丰富、循环购买结构常见、统计模型驱动

### 3.6 CMBS（商业房地产抵押贷款支持证券）

- 以商业物业抵押贷款为基础
- 关键特征：单笔规模大、物业运营收入为第一还款来源、抵押权保障

### 3.7 类REITs

- 以持有商业物业的项目公司股权+债权为基础
- 关键特征：结构化安排、增信依赖主体、通常设置优先收购权

## 四、ABS与普通债券的8大差异

| # | 维度 | 普通债券 | ABS |
|---|------|---------|-----|
| 1 | **发行主体** | 法人企业直接发行 | SPV作为发行载体，原始权益人不直接作为发行人 |
| 2 | **信用逻辑** | 依赖发行人主体信用（资产负债表实力） | 依赖基础资产池信用（资产质量+结构设计） |
| 3 | **现金流匹配** | 无特殊要求，发行人自由调度资金 | 必须实现基础资产现金流与证券偿付的匹配 |
| 4 | **增信方式** | 外部担保为主（保证人/抵押/质押） | 内部增信为主（分层/超额抵押）+外部补充 |
| 5 | **破产隔离** | 无破产隔离安排（发行人在己方破产时，债券为普通债权） | 通过真实出售+SPV实现破产隔离 |
| 6 | **评级逻辑** | 主体评级决定债项评级上限（通常债项≤主体） | 债项评级可高于原始权益人主体评级（资产信用独立） |
| 7 | **投资人保护** | 交叉违约、财务指标维持等 | 资产表现监控指标、信用触发机制、加速清偿 |
| 8 | **信息披露** | 以发行人财务信息为主 | 以基础资产池表现数据为主（月度/季度资产服务报告） |

## 五、关键参与方

### 5.1 参与方清单与职责

| 参与方 | 核心职责 |
|--------|---------|
| **原始权益人** | 提供基础资产、担任资产服务机构（通常）、进行资产持续管理和回收 |
| **计划管理人/受托机构** | 设立并管理SPV（专项计划/信托）、组织发行、存续期管理 |
| **托管银行** | 开立信托账户、资金保管、按指令划付、出具托管报告 |
| **监管银行（如适用）** | 基础资产回收款归集账户的监管 |
| **信用评级机构** | 对分层证券进行初始评级和跟踪评级 |
| **会计师事务所** | 现金流模型验证、税务意见、审计服务 |
| **律师事务所** | 法律尽职调查、交易文件起草、法律意见书出具 |
| **资产评估机构（如适用）** | 基础资产评估、现金流预测审核 |
| **主承销商/推广机构** | 证券销售、簿记建档、投资者沟通 |

### 5.2 各参与方相互独立性要求

- 托管银行与计划管理人不得存在关联关系
- 评级机构、律师事务所、会计师事务所须独立于原始权益人和计划管理人
- 资产服务机构通常由原始权益人担任（存在利益一致性，但须有后备服务机构安排）

## 六、ABS项目执行生命周期

```
01_项目启动 → 02_尽职调查 → 03_方案设计 → 04_文件撰写 → 05_监管申报 → 06_发行执行 → 07_存续期管理
```

各阶段核心工作：

| 阶段 | 核心工作 | 周期（估） |
|------|---------|-----------|
| 01_项目启动 | 资产筛选、可行性论证、监管预沟通、中介选聘、立项内核 | 4-8周 |
| 02_尽职调查 | 资产池尽调、法律尽调、原始权益人尽调 | 6-12周 |
| 03_方案设计 | 交易结构设计、增信方案设计、现金流分配机制 | 4-6周 |
| 04_文件撰写 | 计划说明书/募集说明书、法律意见书、评级报告 | 4-8周 |
| 05_监管申报 | 交易所/NAFMII/银保监申报与反馈回复 | 4-12周 |
| 06_发行执行 | 定价、簿记建档、缴款、起息、挂牌 | 2-4周 |
| 07_存续期管理 | 资产服务报告审核、现金流分配执行、信息披露 | 贯穿存续期 |

## 七、核心术语表

| 术语 | 英文 | 释义 |
|------|------|------|
| 基础资产 | Underlying Assets | 作为证券偿付来源的原始资产组合 |
| 破产隔离 | Bankruptcy Remote | 原始权益人破产时基础资产不受追偿 |
| 真实出售 | True Sale | 资产从原始权益人法律上完全转移至SPV |
| 信用增级 | Credit Enhancement | 提升证券信用质量的结构化安排 |
| 优先/次级 | Senior/Subordinated | 通过分配顺序实现内部信用增级的结构 |
| 超额抵押 | Over-Collateralization (OC) | 基础资产池规模大于证券发行规模 |
| 瀑布分配 | Waterfall | 按优先级从高到低的现金流分配顺序 |
| 循环购买 | Revolving Purchase | 在循环期内持续以回收款购买新资产 |
| 加速清偿 | Accelerated Amortization | 触发后停止循环购买，开始加速还本 |
| 影子评级 | Shadow Rating | 管理人对基础资产债务人信用的内部评估 |
| 封包日 | Cut-off Date | 基础资产池的锁定基准日 |

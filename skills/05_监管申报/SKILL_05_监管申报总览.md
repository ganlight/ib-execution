# SKILL_05：监管申报总览

## 一、概述

监管申报是将尽调成果和撰写文件转化为正式申报材料，并向监管机构报送的环节。不同产品的申报流程、审核机制和监管机构存在显著差异。本技能提供申报流程的总览性框架和各产品线共性方法论。

## 二、各产品线申报流程对比

### 2.1 总体对比表

| 维度 | IPO | 债券 | 并购重组 |
|-----|-----|------|---------|
| **监管机构** | 证监会 + 交易所（双重审核） | NAFMII/证监会/交易所（视品种） | 证监会并购重组委/交易所 |
| **审核制度** | 注册制（交易所审核+证监会注册） | 注册制（银行间/交易所各有规则） | 核准制（部分转为注册制） |
| **核心审核文件** | 招股说明书 | 募集说明书 | 重组报告书 |
| **审核时限** | 3-6个月（受理至注册） | 视品种而定 | 3-6个月 |
| **反馈次数** | 2-3轮（平均） | 1-2轮 | 1-2轮 |
| **上市委/重组委审议** | 有（上市委会议） | 无 | 有（重组委会议） |
| **注册有效期** | 1年（批文有效期） | 2年（批文有效期） | 1年（批文有效期） |

### 2.2 IPO申报路径

```
改制设立 → 辅导备案 → 辅导验收 → 申报受理 → 
交易所审核（含多轮问询）→ 上市委审议 → 证监会注册 → 
发行上市
```

主板/创业板/科创版的具体差异见 **SKILL_05a** 和 **SKILL_05b**。

### 2.3 债券申报路径

> **→ 委托给 ib-bond-executor 技能处理**。

不同债券品种对应的审核路径：银行间（NAFMII注册）→ 交易所（证监会注册或交易所审核）→ 发改委（企业债审批）。具体流程见 ib-bond-executor SKILL.md。

### 2.4 并购重组申报路径

```
董事会决议 → 股东大会决议 → 申报受理 → 
交易所审核 → 重组委审议（发股购买资产）→ 证监会核准/注册 → 
实施交割
```

详细流程见 **SKILL_05d**。

## 三、关键里程碑

### 3.1 IPO关键里程碑及时间参考

| 里程碑 | 预计时间（自申报起） | 主板参考 | 创业板参考 | 科创版参考 |
|--------|-------------------|---------|-----------|----------|
| 申报受理 | T+0 | 5个工作日 | 5个工作日 | 5个工作日 |
| 首轮问询发出 | T+20~40工作日 | 约1个月 | 约1个月 | 约1个月 |
| 首轮回复提交 | 首轮问询后约1~2个月 | 2个月 | 1-2个月 | 1-2个月 |
| 第二轮问询（如有） | 回复提交后2~4周 | 约2周 | 约2周 | 约2周 |
| 上市委会议（上会） | 全部问询回复完毕 | 约1-2个月 | 约1个月 | 约1个月 |
| 提交注册 | 过会后尽快 | 约1-2周 | 约1周 | 约1周 |
| 注册批文下发 | 提交注册后 | 1-3个月 | 1-2个月 | 1-2个月 |
| **总周期** | — | **8-18个月** | **6-12个月** | **5-9个月** |

### 3.2 批文有效期

| 产品类型 | 批文有效期 | 起算日 |
|---------|-----------|--------|
| IPO（注册批文） | 12个月 | 注册决定日 |
| 上市公司再融资 | 12个月 | 注册决定日 |
| 债券（注册通知书） | 24个月 | 注册通知书出具日 |
| 并购重组（核准/注册批文） | 12个月 | 核准/注册决定日 |

## 四、时效追踪方法论

### 4.1 追踪维度

详见 **SKILL_05f_审核进度追踪.md**。

### 4.2 关键节点预警逻辑

```
绿色（正常）：当前节点耗时 ≤ 同类项目平均耗时 × 1.2
黄色（关注）：平均 × 1.2 < 当前节点耗时 ≤ 平均 × 1.5
红色（风险）：当前节点耗时 > 平均 × 1.5
```

### 4.3 历史数据对标

建立同类项目（同板块、同行业、同规模）的平均审核时间数据库，用于：
- 预估本项目审核进度
- 识别异常延迟并及时介入
- 为发行人管理预期

## 五、常见延误因素及应对

### 5.1 内部可控制因素

| 延误因素 | 风险等级 | 应对措施 |
|---------|---------|---------|
| 反馈回复准备耗时过长 | 🔴高 | 收到反馈前即预判回复方向，提前准备 |
| 补充审计/尽调耗时 | 🟡中 | 申报前充分预估，先补充后回复 |
| 中介沟通不顺畅 | 🟡中 | 建立书面确认机制，减少电话沟通 |
| 发行人配合度低 | 🔴高 | 申报前签订配合协议，明确时间表 |

### 5.2 外部不可控因素

| 延误因素 | 应对措施 |
|---------|---------|
| 监管审核负荷重（排队） | 跟踪审核员进度，适时适当催促 |
| 政策变化 | 密切关注政策动向，及时调整方案 |
| 行业舆情/媒体质疑 | 建立危机应对预案，及时回应 |

## 七、申报前最终核查

无论哪个产品线，申报前必须完成以下共同核查：

- [ ] 所有申报文件均已完成内部审批（质控+内核）
- [ ] 签字页全部签署完毕
- [ ] 电子版和纸质版文件内容一致
- [ ] 关键数据在所有文件中保持一致
- [ ] 所有文件均在有效期内（审计报告6个月、法律意见书无固定有效期但需刷新）
- [ ] 发行人/中介机构签章完备
- [ ] 申报文件格式符合监管机构要求

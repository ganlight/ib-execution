# SKILL_02d：底稿工作体系

## 概述

DD底稿是投行尽职调查最基础也是最核心的工作档案。它不仅支撑招股说明书/申报文件的每一项陈述，更是日后监管检查、内部质控复核、法律纠纷中证明"勤勉尽责"的关键证据。

## 二、文件命名规范

### 2.1 标准命名格式

```
{DD编号}_{文件内容描述}_{日期}_{版本}.{扩展名}

示例：
DD-01-01_营业执照_20240115_v3.pdf
DD-05-04_大额交易核查底稿_20240220_v2.xlsx
DD-06-08_CEO访谈纪要_20240128_v1.docx
```

### 2.2 版本规范

| 版本标识 | 含义 |
|----------|------|
| v1 | 初稿 |
| v2, v3... | 修订稿 |
| final | 定稿（不可修改） |
| annex | 附件 |

## 三、交叉引用体系

### 3.1 底稿与招股书章节对照

| 招股书章节 | 底稿索引 |
|------------|----------|
| 释义 | DD-00 |
| 概览 | DD-00 |
| 风险因素 | DD-05/06/90 |
| 发行人基本情况 | DD-01/02 |
| 业务与技术 | DD-06 |
| 财务会计信息 | DD-05 |
| 管理层讨论与分析(MD&A) | DD-05/06 |
| 募集资金运用 | DD-07 |
| 公司治理 | DD-02/04 |
| 关联交易与同业竞争 | DD-04/05 |
| 重大合同 | DD-04 |
| 诉讼与仲裁 | DD-90-01 |

### 3.2 交叉引用在底稿中的标记

每份底稿文件应在页眉/注释中包含以下信息：

```
底稿编号: DD-05-04
交叉引用: 招股书"第八节 财务会计信息"、"第九节 MD&A"
核查人: XXX
核查日期: 2024-01-20
核查结论: [通过 / 发现异常 / 待补充]
```

## 四、电子底稿管理

### 4.1 文件夹权限

| 角色 | 权限 |
|------|------|
| 项目负责人 | 全部读写 |
| 执行负责人 | 全部读写 |
| 项目组成员 | 本模块读写、其他只读 |
| 质控/内核 | 只读 |
| 外部审计/律师 | 指定文件夹读写（按工作范围） |

### 4.2 版本控制

- 每次修改必须保存为新版本（用v1/v2等标识）
- 旧版本不得删除（移至 /archive/ 目录）
- 定稿（final）版本设为只读
- 使用云盘/内部系统同步，确保版本一致性

### 4.3 备份策略

- 每日增量备份
- 每周全量备份
- 异地备份（至少一个异地副本）
- 备份保留至项目归档后2年

## 五、纸质底稿管理

### 5.1 原件管理

| 注意事项 | 要求 |
|----------|------|
| 核实与保管 | 原件经核实后立即归还客户或放入档案柜 |
| 留存方式 | 原件扫描彩色PDF + 关键文件保留复印件 |
| 签收确认 | 接收和归还原件均需签收单 |

### 5.2 纸质文件索引标贴

- 使用统一的索引标签（彩色分页纸）
- DD-00 用白色、DD-01 用蓝色、DD-02 用绿色……以此类推
- 每个文件夹封面标注：项目名称、DD编号、创建日期、负责人

## 六、底稿完整性检查

### 6.1 申报前自检清单

- [ ] 底稿目录完整（DD-00 至 DD-90 无缺失模块）
- [ ] 每份底稿有明确的编号、日期、核查人
- [ ] 所有交叉引用的招股书陈述均有对应底稿支撑
- [ ] 关键文件（执照/资质/合同/审计报告）均为最新版本
- [ ] 原件扫描件分辨率≥300dpi、内容清晰可读
- [ ] 电子版与纸质版底稿编号一致
- [ ] 底稿中没有夹带无关文件/废弃版本
- [ ] 底稿清单已编制完成（含文件编号/名称/日期/核查人）

### 6.2 底稿清单（Index List）模板

| DD编号 | 文件名称 | 文件日期 | 版本 | 核查人 | 核查日期 | 备注 |
|--------|----------|----------|------|--------|----------|------|
| DD-01-01 | 营业执照 | 2024-01-15 | final | 张三 | 2024-01-16 | 有效期至2030年 |
| DD-01-02 | 公司章程 | 2023-06-01 | final | 张三 | 2024-01-16 | 最新修正版 |

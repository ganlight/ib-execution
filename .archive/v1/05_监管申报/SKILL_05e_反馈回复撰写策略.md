# SKILL_05e：反馈回复撰写策略

## 一、概述

反馈回复是IPO/重组审核过程中最关键的交互环节。一次高质量的回复可以大幅推进审核进程，一次质量不高的回复可能导致多轮追加问询甚至项目终止。本技能定义回复的格式标准、撰写原则和策略决策框架。

## 二、回复文件格式标准

### 2.1 标准回复结构

```
第一部分：核查过程概述
  - 收到反馈的时间和函件编号
  - 中介机构核查工作的组织安排
  - 核查方法和范围（审阅文件、函证、访谈、走访、分析性复核等）
  - 核查总体结论

第二部分：逐题回复
  （每道反馈问题按以下格式逐一回复）

第三部分：招股说明书/重组报告书修改说明
  - 修改明细表（逐一列出修改位置和内容）
  - 附红黑修订对比稿
```

### 2.2 逐题回复格式标准

```
【反馈问题原文】（引自反馈意见函第X题）

【回复】

一、核查程序
  针对上述问题，保荐人/独立财务顾问执行了以下核查程序：
  1. ...（具体程序）
  2. ...

二、核查事实
  （陈述经核查查明的客观事实，含数据、图表等）

三、分析论证
  （基于事实进行分析和论证，引用法规或准则依据）

四、核查结论
  经核查，保荐人/独立财务顾问认为：......（明确结论）

【招股说明书/重组报告书修改情况】
  已在招股说明书"第X节 XX"之"X.XX"中补充/修改如下：
  "......（新增/修改内容原文）"
```

### 2.3 涉及其他中介的问题格式

如果问题涉及会计师或律师的核查结论：
- 会计师意见：（单独段落，由会计师出具）
- 律师意见：（单独段落，由律师出具）
- 保荐人核查意见：（保荐人综合判断）

## 三、回复四大原则

### 3.1 数据准确——与申报文件一致

- 回复中引用的所有数据必须与招股书/审计报告/法律意见书完全一致
- 不因为回复需要而"创造"或"调整"数据
- 如需引用未经申报确认的新数据，必须先更新申报文件

### 3.2 回复完整——不留空白和遗漏

- 反馈问题中每一个子问题都必须逐一回答
- 不能选择性地回答"容易的问题"而回避"难的问题"
- 隐含关切也要回应（如表面问"关联交易定价"，实质关切"利益输送"）
- **宁可多答不可漏答**

### 3.3 谨慎承诺——不"过度承诺"

- 不使用"确保""一定""绝对"等绝对化用语
- 对未来的预测需说明假设前提和不确定性
- 承诺事项须有可行方案支持
- 不为了过关而做出超出能力的承诺

### 3.4 可验证——答复有据可查

- 每项结论须有充分的核查依据支撑
- 依据来源清晰可追溯（合同编号、函证编号、走访记录编号等）
- 如涉及发行人出具的说明，需注明并评估其证明力
- 回复正文本身构成可独立验证的论证链

## 四、回复策略决策树

### 4.1 策略决策框架

根据问题的性质，选择以下策略：

```
┌── 问题性质判断 ──┐
│                   │
├─ 事实性错误？ ───→ 策略A：承认+纠正
├─ 理解偏差？ ────→ 策略B：解释+补充
├─ 披露不充分？ ──→ 策略C：补充披露（不辩解）
├─ 政策灰色地带？ → 策略D：引用法规+可比案例
├─ 边界问题？ ────→ 策略E：论证"不构成实质性障碍"
└─ 数据需更新？ ──→ 策略F：补充审计/更新数据
```

### 4.2 策略A：事实性错误

**触发条件**：招股书/申报文件中确实存在数据错误、事实表述错误

**应对方法**：
1. 坦承错误，不回避、不辩解
2. 修正数据/表述
3. 说明错误产生的原因（不可以引导至他人）
4. 修正后数据嵌入招股书并在修改说明中标注
5. 保证类似错误已经全面排查

**禁忌**：推卸责任给其他中介或发行人

### 4.3 策略B：理解偏差

**触发条件**：审核员的疑问源自对业务模式/行业特点/交易结构的不完全理解

**应对方法**：
1. 正面解释，用对比、举例、图表等直观方式
2. 提供补充信息帮助理解
3. 避免暗示审核员"不懂"，尊重专业问询
4. 解释后应使任何具备财务/法律背景的人都能理解

**示例结构**："审核员关注到XX现象，该现象是由于以下行业惯例造成的：（1）...（2）...。以下举例说明：..."

### 4.4 策略C：披露不充分

**触发条件**：申报文件中确实遗漏了应披露的信息

**应对方法**：
1. **不辩解**为何未披露（理由只是借口）
2. **直接补充**完整的披露内容
3. 补充内容充实具体，不打折扣
4. 说明补充披露的招股书具体位置
5. 保证相关方面的信息披露已全面检查

**黄金法则**：补充的信息应比审核员预期的更充分，给其超出预期的满足感。

### 4.5 策略D：政策灰色地带

**触发条件**：问题涉及法规未明确规定或存在多种理解的情形

**应对方法**：
1. 引用相关法律/法规/规范性文件的具体条文
2. 分析不同理解路径及其适用情境
3. 说明发行人采取的路径及其合理性
4. 列举同行业可比公司的类似处理方式
5. 结论：所采取的处理方式合法合规

**引证优先级**：法律 > 行政法规 > 部门规章 > 规范性文件 > 监管问答/窗口指导 > 可比案例

### 4.6 策略E：边界问题

**触发条件**：问题涉及某事项是否构成实质性障碍但实际影响有限

**应对方法**：
1. 承认该事项的存在
2. 用定性和定量数据说明影响程度
3. 论证为什么"不构成实质性法律障碍/不构成重大不利影响"
4. 提供解决方案（如有）
5. 列举已获审核通过的具有类似事项的可比案例

### 4.7 策略F：数据需更新

**触发条件**：反馈时点已超过申报基准日6个月，或审核员要求更新财务数据

**应对方法**：
1. 安排补充审计（增加最近一期数据）
2. 更新招股书中所有受影响的财务数据和分析
3. 补充期后事项核查
4. 同时更新行业数据、经营数据等非财务的时效数据

## 五、回复语言风格指南

### 5.1 用语规范

| 推荐用语 | 禁止用语 |
|---------|---------|
| 经核查，…… | 我们认为……（除非有依据）|
| 该数据来源于……审计报告第X页 | 据了解/通常/一般 |
| 上述情形符合……规定 | 没问题/不存在问题（过于随意）|
| 具有合理性（即分析合理性的原因） | 是合理的（无分析）|
| 请参见本回复第X题之回复 | 详见上文（不够精确）|

### 5.2 定量分析的标准

凡是涉及"是否存在差异""是否有影响"的问题，必须定量化：
- ❌ "影响较小"
- ✅ "上述因素对报告期各期净利润的影响金额分别为XX万元、XX万元、XX万元，占各期净利润的比例分别为X.XX%、X.XX%、X.XX%，影响较小"

## 六、提交前自查清单

### 6.1 内容完整性

- [ ] 所有反馈问题（含子问题）均已逐一回答
- [ ] 每个问题的回复包含核查程序、核查事实、分析论证、核查结论
- [ ] 反馈中提及的修改已在招股书中定位
- [ ] 招股书修改说明表完整
- [ ] 如涉及会计师/律师意见，已分别获取并纳入

### 6.2 数据准确性

- [ ] 回复中所有数据与招股书/审计报告一致
- [ ] 新引用的数据有可靠来源
- [ ] 百分比、同比、环比计算准确
- [ ] 表格数字汇总与明细一致

### 6.3 论证充分性

- [ ] 每项结论有充分的依据支撑
- [ ] 定性结论有定量数据支持
- [ ] 引用的法规条文准确有效
- [ ] 逻辑链完整，无跳跃
- [ ] 边界条件说明清晰

### 6.4 文件格式

- [ ] 问题编号与反馈意见函完全一致
- [ ] 排版格式统一规范
- [ ] 签章完整
- [ ] 电子版PDF书签完整
- [ ] 招股书红黑稿正确生成

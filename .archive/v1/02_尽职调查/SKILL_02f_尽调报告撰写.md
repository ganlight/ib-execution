# SKILL_02f：尽调报告撰写

## 概述

尽调报告是DD阶段的最终产出，汇总法律、财务、业务三大维度的发现，形成对发行人/标的公司的全面风险评估。报告质量直接关系到后续方案设计、内核审批和申报文件撰写。

---

## 一、报告结构

### 1.1 标准结构（7部分）

```
                            尽调报告
                               │
        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
  第一部分              第二部分                第三部分
  执行摘要              法律DD发现             财务DD发现
  (2-3页)              (8-15页)               (10-20页)
        │                      │                      │
        └──────────────────────┼──────────────────────┘
                               │
                    ┌──────────┴──────────┐
                    │                     │
              第四部分               第五部分
              业务DD发现             风险评估
              (8-15页)             (3-5页)
                    │                     │
                    └──────────┬──────────┘
                               │

                    │                     │
              第六部分               第七部分
              结论与建议             附录
              (1-2页)             (不限)
```

## 二、各部分编写指南

### 2.1 执行摘要（Executive Summary）

**目标读者**：没有时间读全文的高管/内核委员

**必须包含**（按此顺序）：

1. **一句话定性**：本次DD的整体结论（如"截至本报告日，发行人不存在构成实质性障碍的Red问题，Amber问题已制定完整的披露方案，基本具备申报条件"）

2. **关键发现Top 3**：最重要的3条发现（无论好坏）
   - 正面发现示例："公司自主研发的核心技术在行业内处于领先地位，近3年毛利率稳定在45%以上"
   - 负面发现示例："公司2022年关联方资金占用2,000万元，截至DD日已全额归还但需完整披露"

3. **Red问题清单**（如有）：枚举所有Red问题及其当前状态

4. **核心风险总结**：3-5条最重要的风险因素

5. **下一步建议**：是否建议进入方案设计阶段

### 2.2 法律DD发现

按五大板块逐一报告（与 SKILL_02a 对应）：

| 板块 | 报告内容 | 每板块必须包含 |
|------|----------|---------------|
| 1.主体资格 | 设立/变更/存续合法性 | 事实描述→发现问题→影响评估→建议 |
| 2.股权结构 | 实际控制人/股权穿透/代持 | 同上 |
| 3.主要资产 | 土地/房产/知识产权 | 同上 |
| 4.重大合同 | 借款/担保/采购/销售 | 同上 |
| 5.诉讼处罚 | 未决诉讼/行政处罚 | 同上 |

**每板块报告格式**：
```
【事实描述】
（客观陈述核查发现的情况，引用具体数据/时间/文件）

【发现问题】
（按RAG等级列出问题，每个问题注明ID和等级）

【影响评估】
（问题对公司持续经营/发行上市条件的具体影响）

【建议】
（整改方案/披露建议）
```

### 2.3 财务DD发现

按八大核查领域逐一报告（与 SKILL_02b 对应）：

1. 银行流水核查
2. 应收账款
3. 存货
4. 关联交易与资金占用
5. 收入确认
6. 毛利率分析
7. 负债结构
8. 对外担保与或有负债

**关键数据呈现**：
- 使用表格而非段落文字呈现财务数据
- 标注同比/环比变化
- 标注与同行业的比较
- 异常数据用红色/加粗标识

### 2.4 业务DD发现

按六大板块逐一报告（与 SKILL_02c 对应）：

1. 行业地位与前景
2. 技术与研发
3. 采购与供应商
4. 销售与客户
5. 生产能力
6. 商业模式

**业务DD报告特色**：
- 要有判断和观点（而非简单的数据罗列）
- 比较分析（与竞争对手、行业平均水平）
- 趋势研判（未来1-3年的发展预判）

### 2.5 风险评估

| 风险类别 | 风险描述 | 可能性 | 影响程度 | 缓释措施 |
|----------|----------|--------|----------|----------|
| 法律风险 | | 高/中/低 | 高/中/低 | |
| 财务风险 | | 高/中/低 | 高/中/低 | |
| 业务/市场风险 | | 高/中/低 | 高/中/低 | |
| 合规风险 | | 高/中/低 | 高/中/低 | |
| 整合风险(M&A专用) | | 高/中/低 | 高/中/低 | |

### 2.6 结论与建议

**结论模板**：

```
基于本次尽职调查，我们认为：

1. [公司名称]是依法设立且合法存续的[股份有限公司/有限责任公司]，
   股权结构清晰，实际控制人认定为[XXX]。

2. 公司主营业务为[XXX]，财务状况良好，具备持续经营能力。
   [如有重大保留意见，在此表述]

3. 本次DD发现Red问题[X]项（[已解决X项/在整改X项]），
   Amber问题[X]项（均已制定披露方案），Green问题[X]项（已记录归档）。

4. 结论：[项目名称]基本具备[申报条件/进一步推进条件]/
   尚不具备，因[核心障碍]。
```

### 2.7 附录

- 问题清单（完整版）
- 关键数据汇总表
- 文件清单
- 核查方法说明
- 术语表

## 三、撰写原则

### 3.1 三大原则

| 原则 | 含义 | 反面教材 |
|------|------|----------|
| **事实性 (Factual)** | 每一条陈述必须有底稿支撑。不推论、不猜测。 | "我们认为公司治理良好" → 应改为"公司建立了三会一层治理结构，报告期内召开董事会XX次、股东大会XX次，会议程序合规" |
| **引用性 (Referenced)** | 每一条关键陈述必须注明底稿索引(DD编号)。 | 无引用的判断性陈述 = 无意义 |
| **平衡性 (Balanced)** | 正面和负面均如实反映。不掩盖问题、不夸大风险。 | "公司存在XXX问题" → 应同时说明"已采取/拟采取XXX措施" |

### 3.2 写作语言规范

- 使用客观中性语言（不出现"非常优秀""行业翘楚"等正面评价）
- 使用精确语言（"约500万元" → "4,856,320元"）
- 避免模糊表述（"可能""大概""基本上"→用具体事实代替）
- 专业术语首次出现时标注定义
- 英文术语首次出现时标注中文对应

## 四、常见撰写错误

| 错误类型 | 示例 | 正确做法 |
|----------|------|----------|
| 缺乏底稿支撑 | "经核查，公司不存在关联方资金占用" | 必须注明核查方法/范围/底稿编号 |
| 结论过度 | "公司财务情况良好" | 陈述具体财务指标，让读者自己判断 |
| 避重就轻 | Red问题写得轻描淡写 | Red问题必须正面、完整呈现 |
| 啰嗦冗长 | 一大段无结构文本 | 使用表格/要点式呈现 |
| 信息遗漏 | 跳过某个应核查但未核查的领域 | 写明"因XXX原因，该领域本次未纳入DD范围" |
| 引用过时数据 | 使用3年前的行业报告 | 数据引用标注日期，确保时效性 |
| 相同问题的版本不统一 | 同一事项在不同章节出现不一致描述 | 全文交叉校验 |

## 五、与招股说明书/申报文件的对应

| DD报告章节 | 招股说明书对应章节 |
|------------|-------------------|
| 法律DD-主体资格 | 第五节 发行人基本情况 |
| 法律DD-股权结构 | 第五节 发行人基本情况 |
| 法律DD-主要资产 | 第六节 业务与技术 |
| 法律DD-诉讼处罚 | 第十五节 其他重要事项 |
| 财务DD-收入/毛利/资产/负债 | 第八节 财务会计信息/管理层讨论与分析 |
| 财务DD-关联交易 | 第十节 关联交易与同业竞争 |
| 业务DD-行业/技术/销售/采购 | 第六节 业务与技术 |
| 风险评估 | 第三节 风险因素 |

**DD报告是招股说明书的"前身"**：DD报告中的事实描述和风险分析是招股书初稿的核心素材来源。撰写DD报告时应注意：
- 对于要在招股书中披露的内容，语言尽量接近招股书要求（正式、规范）
- 对于不应公开的内部讨论（如整改策略、内部判断），明确标注"内部使用/不对外"

## 七、报告审阅流程

```
初稿撰写（执行负责人/项目组成员）
        ↓
同行审阅（项目组内部交叉审阅）
        ↓
修改完善
        ↓
项目负责人审阅
        ↓
质控审阅（质量+合规）
        ↓
修改完善
        ↓
内核委员会审阅
        ↓
定稿 + 分发
```

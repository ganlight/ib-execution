# SKILL_02：尽职调查总览

## 概述

尽职调查（Due Diligence, DD）是投行项目的核心阶段，耗时通常占项目总周期的 **40%-60%**。DD的核心目标是：**发现风险、评估风险、管理风险**——而非简单地描述公司情况。

本文件为DD阶段总览，具体操作指引由子文件展开。

---

## 一、DD差异矩阵：不同产品的尽调重点

| DD维度 | IPO（A股） | 再融资 | M&A |
|--------|-----------|--------|-----|
| **法律DD深度** | 最深：历史沿革全部核查 | 中等：新增变化核查 | 最深：标的公司全面核查 |
| **财务DD周期** | 3年+1期（科创板2年+1期） | 最近1-2年 | 2-3年+1期 |
| **业务DD重点** | 独立性+持续经营能力 | 募投项目可行性 | 协同效应+整合 |
| **行业DD** | 全面 | 侧重募投行业 | 标的行业+买方行业 |
| **关联交易关注度** | 极高（独立性要求） | 中 | 高（交易定价公允性） |
| **同业竞争关注度** | 极高（IPO红线） | 低 | 中 |
| **内部控制要求** | 内控审计报告 | 一般关注 | 标的公司内控 |
| **知识产权** | 核心技术来源+权属 | 涉及技术升级 | 核心技术估值 |

### IPO专项：不同板块DD差异

| 维度 | 主板 | 科创板 | 创业板 | 北交所 |
|------|------|--------|--------|--------|
| 行业定位 | 无特殊限制 | 硬科技（科创属性） | 三创四新 | 创新型中小企业 |
| 研发投入要求 | 无 | 近3年≥15%（标准2）或≥5% | 无特殊指标 | 无特殊指标 |
| 核心技术DD | 一般 | 深度（专利+人员+来源） | 较深 | 较一般 |
| 盈利要求 | 有（3套标准） | 允许未盈利（5套标准） | 有盈利要求 | 有盈利要求 |

## 二、三大尽调维度

```
                  ┌──────────┐
                  │ 法律尽调   │
                  │ 合法性/权属 │
                  └─────┬────┘
                        │
              ┌─────────┴─────────┐
              │                   │
        ┌─────┴─────┐     ┌─────┴─────┐
        │ 财务尽调    │     │ 业务尽调    │
        │ 真实性/规范  │     │ 持续性/成长  │
        └───────────┘     └───────────┘
```

### 2.1 法律DD维度（详见 SKILL_02a）

| 核查领域 | 核心问题 | 关键风险 |
|----------|----------|----------|
| 发行主体资格 | 依法设立且合法存续 | 出资不实/历史沿革瑕疵 |
| 股权结构 | 股权清晰、控制权稳定 | 代持/纠纷/频繁变更 |
| 主要资产 | 权属清晰、无重大限制 | 权属瑕疵/抵押/查封 |
| 重大合同 | 合法有效、无重大风险 | 违约风险/不公平条款 |
| 诉讼仲裁 | 无重大未决争议 | 核心专利诉讼/大额赔偿 |

### 2.2 财务DD维度（详见 SKILL_02b）

| 核查领域 | 核心问题 | 关键风险 |
|----------|----------|----------|
| 收入真实性 | 收入确认合规、无虚增 | 提前确认/虚构交易 |
| 资产质量 | 应收/存货/商誉无重大减值 | 坏账/滞销/商誉暴雷 |
| 负债完整性 | 所有负债已完整记录 | 表外负债/隐性担保 |
| 关联交易 | 定价公允、决策合规 | 利益输送/资金占用 |
| 盈利能力 | 利润来源可持续 | 依赖补贴/非经常性损益 |

### 2.3 业务DD维度（详见 SKILL_02c）

| 核查领域 | 核心问题 | 关键风险 |
|----------|----------|----------|
| 行业前景 | 行业处于上升期、政策友好 | 行业萎缩/政策打压 |
| 技术竞争力 | 核心技术自主可控 | 技术依赖/侵权风险 |
| 客户/供应商 | 不依赖单一客户/供应商 | 过度集中/关联方 |
| 生产能力 | 产能充足、利用率合理 | 产能过剩/受限 |
| 商业模式 | 模式可持续、可复制 | 烧钱模式/补贴驱动 |

## 三、问题分级体系（RAG）

### 3.1 分级标准

| 等级 | 图标 | 定义 | 处理要求 | 时限 |
|------|------|------|----------|------|
| 🔴 **Red** | 🔴 | 构成实质性障碍，可能影响发行条件/上市条件 | 必须在申报前解决 | 发现后1周内启动整改 |
| 🟡 **Amber** | 🟡 | 需要充分披露和解释的风险或瑕疵 | 在招股书中如实充分披露 | 发现后2周内确定披露方案 |
| 🟢 **Green** | 🟢 | 可通过常规措施管理的小问题 | 在底稿中记录并存档 | 正常处理 |

### 3.2 RAG等级判断标准

| 问题性质 | 等级 |
|----------|------|
| 不符合发行条件/上市条件中的硬性指标 | 🔴 Red |
| 存在重大违法违规（刑事/重大行政处罚） | 🔴 Red |
| 实际控制人认定不清、控制权不稳定 | 🔴 Red |
| 对持续经营能力构成重大影响 | 🔴 Red |
| 财务数据存在重大差错需追溯调整 | 🔴 Red |
| 存在同业竞争（拟IPO） | 🔴 Red |
| 关联交易占比较高但定价公允 | 🟡 Amber |
| 历史沿革存在小瑕疵但已完成补正 | 🟡 Amber |
| 单一客户/供应商依赖度>50% | 🟡 Amber |
| 报告期内行政处罚（非重大） | 🟡 Amber |
| 次要资产权属瑕疵（不影响主业） | 🟢 Green |
| 小额诉讼（<净资产的1%） | 🟢 Green |

## 四、行业DD重点提示

| 行业 | 法律重点 | 财务重点 | 业务重点 |
|------|----------|----------|----------|
| **TMT/半导体** | IP权属+来源+境外技术限制 | 研发资本化政策+政府补助依赖 | 技术路线+人才流失+流片依赖 |
| **医药/医疗** | 临床试验+产品注册+GMP合规 | 研发费用+销售费用合规+两票制影响 | 在研管线+审批风险+集采冲击 |
| **制造业** | 环保+土地+安全生产 | 存货+产能利用率+毛利率 | 原材料价格+产能消化+下游周期 |
| **金融** | 牌照+监管指标+合规体系 | 拨备+风险资产分类+流动性 | 监管政策+牌照壁垒+净资本约束 |
| **消费/零售** | 食品安全+特许经营+加盟 | 收入确认+库存+现金交易 | 品牌+渠道+复购率 |

## 五、DD规划与排期

### 5.1 时间规划参考

| 产品类型 | DD总周期 | 法律DD | 财务DD | 业务DD |
|----------|----------|--------|--------|--------|
| A股IPO | 8-16周 | 4-6周 | 6-10周 | 4-6周 |
| 再融资 | 4-8周 | 2-3周 | 3-5周 | 2-3周 |
| M&A | 6-12周 | 3-5周 | 4-6周 | 3-5周 |

### 5.2 DD资料收集阶段划分

```
Week 1-2: 资料清单发出 + 客户数据室搭建
Week 2-4: 第一轮资料收集 + 管理层访谈（CEO/CFO/CTO/业务负责人）
Week 3-6: 第二轮补充资料 + 职能部门访谈（财务/法务/人事/销售/采购）
Week 4-8: 专题DD（知识产权/环保/关联交易/重大合同等）
Week 6-10: 问题追踪与整改
Week 8-12: DD报告初稿
Week 10-16: DD报告定稿 + 问题清零
```

## 六、进入Stage 03的门槛条件

以下条件**全部满足**后方可进入方案设计阶段：

| 编号 | 门槛条件 | 验证方式 | 责任人 |
|------|----------|----------|--------|
| DD01 | 法律DD已完成，Red问题已清零 | 法律DD报告定稿 | 法律团队 |
| DD02 | 财务DD已完成，Red问题已清零 | 财务DD报告定稿 | 审计团队 |
| DD03 | 业务DD已完成，Red问题已清零 | 业务DD报告定稿 | 投行团队 |
| DD04 | 所有Amber问题已确定披露方案 | 问题追踪表 | 项目负责人 |
| DD05 | DD底稿已整理归档（≥90%完成） | 底稿目录 | 执行负责人 |
| DD06 | DD总报告已撰写完成并通过内部审核 | DD总报告 | 执行负责人 |
| DD07 | 中介协调会总结已完成 | 会议纪要 | 执行负责人 |
| DD08 | 关键假设（收入/利润/估值）已与管理层确认一致 | 管理层确认函 | 项目负责人 |

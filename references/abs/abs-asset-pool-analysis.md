# ABS 资产池分析方法论

## 一、资产池分析基本框架

资产池分析是 ABS 信用评估的核心环节，其目标在于通过对基础资产组合的系统性分析，科学量化资产池的预期损失分布，为证券分层评级提供数据支撑。方法论的核心要旨在于：**从单笔资产的信用特征出发，结合资产间的相关性与组合效应，推导资产池整体的损失概率分布，最终映射至各档证券的预期损失与信用等级**。

### 1.1 分析流程总览

```
资产数据收集与清洗 → 合格标准筛选 → 资产池特征画像 →
静态/动态池分析 → 影子评级 → 压力情景与敏感性分析 →
现金流瀑布模拟 → 分层评级结论
```

### 1.2 关键假设前提

| 假设类别 | 具体内容 | 偏差风险 |
|---------|---------|---------|
| 历史数据代表性 | 未来表现与历史数据呈现的统计规律一致 | 经济周期转折点时失效 |
| 资产独立性（基础） | 单笔资产违约间不存在显著正相关 | 系统性风险低估 |
| 服务商能力稳定 | 资产服务机构催收回收能力保持不变 | 经济下行期催收难度上升 |
| 早偿率稳定性 | 早偿行为模式不发生结构性变化 | 利率大幅变动时早偿行为突变 |
| 提前摊销触发 | 满足触发条件时的资产处置假设合理 | 极端场景下的回收率偏差 |

## 二、静态池与动态池分析方法

### 2.1 静态池分析（Static Pool Analysis）

**定义：** 以某一固定时点的资产组合为分析对象，不假定后续有新资产加入或替换，分析该封闭资产池在未来各期的预期表现。

**适用场景：**

| 资产类型 | 适用性 | 典型示例 |
|---------|:----:|---------|
| 个人住房抵押贷款 | ★★★★★ | RMBS |
| 个人汽车抵押贷款 | ★★★★★ | Auto ABS |
| 企业贷款（CLO） | ★★★★★ | 静态CLO |
| 融资租赁应收款 | ★★★★ | 静态租赁ABS |
| 个人消费贷款（一次性购买） | ★★★★ | 静态消金ABS |

**分析步骤：**

1. **资产池画像（Pool Profile）：**
   - 贷款笔数及规模分布统计
   - 加权平均利率（WAC）与加权平均期限（WAM）
   - 地理/行业/担保方式分布
   - 信用评分/影子评级分布
   - 账龄（Seasoning）分布分析

2. **历史表现数据分析：**
   - 不同账龄段的违约率曲线
   - 不同信用等级的违约迁移矩阵
   - 早偿率（CPR/PSA）历史曲线
   - 回收率与回收时间分布
   - 逾期滚动率（Roll Rate）分析

3. **基准情景参数设定：**
   - 基于历史均值设定违约率基准
   - 基于近期趋势调整基准预期
   - 早偿率基准设定（参考行业惯例或历史均值）
   - 回收率基准（区分不同担保方式）

4. **损失分布模拟：**
   - 蒙特卡洛模拟（Monte Carlo Simulation）
   - 基于违约率/回收率双维度模拟
   - 引入资产相关性矩阵
   - 输出各置信区间下的组合损失

### 2.2 动态池分析（Dynamic Pool Analysis）

**定义：** 资产池在特定期间（循环期）内持续购入新资产，资产组合不断变化，需要分析循环购买机制对资产池信用质量的影响。

**适用场景：**

| 资产类型 | 适用性 | 循环购买特征 |
|---------|:----:|-------------|
| 信用卡分期应收款 | ★★★★★ | 短期资产、高频循环 |
| 个人消费贷款（循环型） | ★★★★★ | 持续新增购买 |
| 应收账款/保理 | ★★★★ | 账期短、持续周转 |
| 供应链反向保理 | ★★★★ | 核心企业持续供货产生新资产 |
| 小额贷款（经营贷循环额度） | ★★★ | 额度循环使用 |

**核心分析维度：**

1. **购买标准（Eligibility Criteria）设定与模拟：**
   - 合格资产须满足的最低标准
   - 每一轮购买后资产池特征是否偏移
   - 模拟多轮购买后资产池特征稳定性

2. **资产池特征漂移风险（Drift Risk）：**
   - 加权平均利率漂移（收入端）
   - 加权平均期限漂移（期限端）
   - 信用质量漂移（风险端）
   - 集中度漂移（分散度端）

3. **购买能力测试：**
   - 每月可购买资产规模下限
   - 合格资产供给充足性评估
   - 原始权益人业务持续性分析

4. **动态池压力情景：**
   - 合格资产供给不足（导致部分资金闲置/低收益）
   - 循环期资产质量持续恶化
   - 提前摊还触发后的资产处置假设

### 2.3 静态池与动态池方法对比

| 对比维度 | 静态池方法 | 动态池方法 |
|---------|-----------|-----------|
| 资产组合稳定性 | 封闭期内不变 | 持续变化 |
| 模拟复杂度 | 中等 | 较高 |
| 关键风险 | 资产池初始质量 | 资产池漂移风险 + 购买标准执行力 |
| 历史数据需求 | 静态资产池的历史表现 | 多期资产池变化趋势 + 新资产特征 |
| 模型输入变量 | 固定参数集 | 时变参数集 |
| 现金流结构 | 过手型或计划摊还型 | 循环购买 + 摊还期 |

## 三、影子评级方法论（五步法）

影子评级（Shadow Rating）是对未获得公开信用评级的基础资产参照评级机构方法论进行的内部信用评估。其结果并非公开发布的信用评级，而是用于 ABS 分层评级的中间变量。

### 3.1 五步法框架

#### 第一步：资产筛选与数据准备 —— 确定评级池（Rated Pool）

**操作要点：**
- 将原始资产池中每一笔资产按照评级机构方法论要求的维度整理数据
- 剔除数据不完整、无法进行影子评级的资产（单独归入「未评级池」）
- 未评级池的资产通常归入次级档吸收（或在保守假设下统一给予最低影子评级）

**数据维度的完整性核查清单：**

| 数据维度 | 企业信贷类资产 | 个人信贷类资产 | 核查标准 |
|---------|:------------:|:------------:|---------|
| 债务人基本信息 | ✓ | ✓ | 识别唯一性、无重复 |
| 贷款/资产信息 | ✓ | ✓ | 本金、利率、期限、还款方式 |
| 担保/增信信息 | ✓ | ✓ | 保证人、抵质押物类型和价值 |
| 债务人财务信息 | ✓ | — | 近三年审计报告/财务报表 |
| 信用记录/行为信息 | — | ✓ | 央行征信报告、逾期记录 |
| 行业/地区分类 | ✓ | ✓（部分） | 行业分类标准一致 |

#### 第二步：单笔资产影子评级 —— 逐笔评估信用质量

**企业类资产（CLO/融资租赁/供应链）：**

| 评估维度 | 权重参考 | 评估内容 |
|---------|:------:|---------|
| 经营与竞争地位 | 20%-25% | 市场份额、行业地位、技术壁垒、经营历史 |
| 财务实力 | 30%-35% | 资产负债率、利息保障倍数、现金流债务比、盈利能力 |
| 流动性 | 10%-15% | 短期偿债能力、融资渠道、备用流动性 |
| 公司治理与战略 | 10%-15% | 股东背景、管理团队经验、战略清晰度 |
| 行业风险 | 10%-15% | 行业周期、竞争格局、监管风险、替代威胁 |
| 外部支持 | 5%-10% | 股东/政府支持意愿和能力 |

**个人类资产（RMBS/Auto ABS/消费贷款）：**

| 评估维度 | 权重参考 | 评估内容 |
|---------|:------:|---------|
| 债务人信用评分 | 35%-40% | 央行征信评分、内部信用评分 |
| 收入/负债比（DTI） | 20%-25% | 月收入/月还款额比例 |
| 贷款价值比（LTV/抵押率） | 20%-25% | 贷款金额/抵押物价值的比率 |
| 就业稳定性 | 10%-15% | 所属行业、工作年限、收入稳定性 |
| 历史行为数据 | 5%-10% | 该债务人历史还款、逾期记录 |

#### 第三步：相关性分析 —— 资产关联关系建模

**相关性来源识别：**

| 相关性类型 | 描述 | 典型来源 | 建模方式 |
|-----------|------|---------|---------|
| 行业相关性 | 同行业债务人受相同行业风险影响 | 同一行业（如房地产、批发零售） | 行业因子模型 |
| 区域相关性 | 同地区的债务人受区域经济波动影响 | 集中同一城市/省份 | 区域因子模型 |
| 债务人相关性 | 同一债务人/关联方的多笔资产 | 同一集团/同一实际控制人 | 合并计算敞口 |
| 担保/增信相关性 | 同一担保人担保的多笔资产 | 保证人集中 | 担保人信用替代 |
| 资产类型相关性 | 同类资产间的系统性风险 | 同一产品类型的总体风险暴露 | 全局因子/系统性因子 |

**相关性参数量化方法：**

| 方法 | 适用条件 | 典型参数范围 |
|------|---------|:----------:|
| 基于历史违约数据统计 | 有足够历史违约联合事件数据 | ρ = 0.05-0.30 |
| 行业/区域因子模型 | 可识别共同的系统性因子 | 因子载荷 β = 0.3-0.8 |
| Basel/Gordy模型框架 | 参照Basel框架推导 | 资产相关性基于PD的函数 |
| 评级机构方法论 | 中诚信/联合/标普/穆迪的公开参数 | 区分资产类别和风险等级 |

**单因子高斯 Copula 模型（行业标准模型框架）：**

在单因子模型中，第 i 笔资产的信用状态由系统性因子 Z 和特异性因子 εᵢ 驱动：

- 资产价值变量：Aᵢ = √ρᵢ · Z + √(1-ρᵢ) · εᵢ
- 其中 ρᵢ 为资产与系统因子的相关性系数
- Z ~ N(0,1)，εᵢ ~ N(0,1)，Z ⟂ εᵢ
- 违约条件：Aᵢ < Φ⁻¹(PDᵢ)
- 条件违约概率：PDᵢ(z) = Φ((Φ⁻¹(PDᵢ) - √ρᵢ·z) / √(1-ρᵢ))

#### 第四步：组合损失分布 —— 从单笔到组合

**组合损失计算框架：**

| 步骤 | 说明 |
|------|------|
| 1. 情景生成 | 生成大量系统性风险情景（通常10万-100万次模拟） |
| 2. 条件违约概率 | 每个情景下计算每笔资产的条件违约概率 |
| 3. 违约判定 | 生成特异性随机数，判定各资产是否违约 |
| 4. 损失计算 | 违约资产 × (1 - 回收率) = 单笔损失额 |
| 5. 组合加总 | 所有资产损失加总 = 该情景下的组合损失 |
| 6. 损失分布 | 汇总所有情景，得组合损失的经验分布 |

**关键输出指标：**

| 指标 | 定义 | 用途 |
|------|------|------|
| 预期损失（EL） | 组合损失在所有情景下的均值 | 优先级增信厚度参考下限 |
| 非预期损失（UL） | 特定置信水平下的损失分位数 - EL | 次级档厚度 + 超额利差覆盖 |
| 尾部风险损失 | 99.9% / 99.99% 置信水平下的尾部损失 | AAA/Aaa 档信用增级要求 |
| 损失波动率 | 组合损失的标准化标准差 | 分散度效果的量化度量 |

#### 第五步：分层评级映射 —— 从组合到证券

**映射逻辑：**

- 将组合损失分布的分位数映射至各档证券的信用增级（Credit Enhancement, CE）水平
- 信用增级 = 次级档 + 超额利差 + 其他增信措施所能吸收的首笔损失
- 目标：各档证券的目标信用等级对应其相应的偿付可靠性

**各等级对应的目标损失覆盖率要求（典型参数范围）：**

| 目标评级 | 最小损失覆盖率（置信区间） | 最大允许违约率 | 典型CE水平（%） |
|---------|:---------------------:|:-----------:|:-----------:|
| AAA(sf) / Aaa(sf) | ≥ 99.90%-99.99% | < 0.01% | 10%-30% (因资产类型而异) |
| AA+(sf) / Aa1(sf) | ≥ 99.50%-99.80% | < 0.05% | 7%-22% |
| AA(sf) / Aa2(sf) | ≥ 99.00%-99.50% | < 0.10% | 5%-18% |
| AA-(sf) / Aa3(sf) | ≥ 98.50%-99.00% | < 0.15% | 4%-15% |
| A+(sf) / A1(sf) | ≥ 97.50%-98.50% | < 0.25% | 3%-12% |
| A(sf) / A2(sf) | ≥ 96.00%-97.50% | < 0.50% | 2%-10% |
| A-(sf) / A3(sf) | ≥ 94.00%-96.00% | < 0.80% | 2%-8% |
| BBB+(sf) / Baa1(sf) | ≥ 90.00%-94.00% | < 1.50% | 1%-6% |
| BBB(sf) / Baa2(sf) | ≥ 85.00%-90.00% | < 3.00% | 1%-5% |
| BBB-(sf) / Baa3(sf) | ≥ 80.00%-85.00% | < 5.00% | 0.5%-4% |

## 四、标准压力情景参数体系

### 4.1 九大类标准压力情景

#### 情景一：基准情景（Base Case）

| 参数 | 设定方法 | 典型取值 |
|------|---------|---------|
| 违约率 | 历史均值 ± 0.25标准差 | 按资产类别取历史均值 |
| 早偿率 | 历史均值 | 按资产类别取历史均值（CPR 5%-25%） |
| 回收率 | 历史均值 | 按资产类别取历史均值（20%-60%） |
| 利率 | 当前市场利率曲线 | 各期限点当前即期利率 |
| 集中度 | 实际资产池 | 无额外调整 |

#### 情景二：温和压力（Mild Stress）

| 参数 | 设定方法 | 典型取值（相对基准调整） |
|------|---------|:----------------------:|
| 违约率 | 基准 × 1.5 | +50% |
| 早偿率 | 基准 × 0.8 | -20% |
| 回收率 | 基准 × 0.85 | -15% |
| 利率 | 各期限 +50bp | +50bp 平移 |
| 集中度 | 实际 + 10%前5大借款人 | +10% |

#### 情景三：中等压力（Moderate Stress）

| 参数 | 设定方法 | 典型取值（相对基准调整） |
|------|---------|:----------------------:|
| 违约率 | 基准 × 2.5 | +150% |
| 早偿率 | 基准 × 0.6 | -40% |
| 回收率 | 基准 × 0.7 | -30% |
| 利率 | 各期限 +150bp | +150bp 平移 |
| 集中度 | 实际 × 1.3 | +30% |
| 宏观经济 | GDP增速-2pp | 引入宏观经济因子 |

#### 情景四：严重压力（Severe Stress）

| 参数 | 设定方法 | 典型取值（相对基准调整） |
|------|---------|:----------------------:|
| 违约率 | 基准 × 4.0 | +300% |
| 早偿率 | 基准 × 0.4 | -60% |
| 回收率 | 基准 × 0.5 | -50% |
| 利率 | 各期限 +300bp | +300bp 平移 |
| 集中度 | 实际 × 1.5 | +50% |
| 宏观经济 | GDP增速-4pp | 深度衰退情景 |

#### 情景五：极端压力——违约率激增

| 参数 | 设定方法 | 典型取值 |
|------|---------|---------|
| 违约率 | 基准 × 5.0-8.0 | +400% 至 +700% |
| 其他参数 | 同严重压力 | — |

#### 情景六：极端压力——早偿率突变

| 参数 | 设定方法 | 典型取值 |
|------|---------|---------|
| 早偿率 | 基准 × 2.0-3.0 | CPR 可飙升至 50%-70% |
| 再投资收益率 | 基准 × 0.5 | -50% |
| 其他参数 | 同基准或温和压力 | — |

**注：** 该情景主要用于评估早偿加速导致的超额利差降低、加权平均期限缩短对投资者收益率的影响。

#### 情景七：极端压力——利率冲击

| 参数 | 设定方法 | 典型取值 |
|------|---------|---------|
| 利率平移 | 各期限 +400bp 至 +500bp | 极端加息周期 |
| 利率曲线形态 | 陡峭化或倒挂 | 特定形态下的尾部风险 |
| 违约率 | 基准 × 2.0 | 利率冲击引发的信用恶化 |

#### 情景八：尾部风险——回收率崩溃

| 参数 | 设定方法 | 典型取值 |
|------|---------|---------|
| 回收率 | 基准 × 0.25-0.35 | -65% 至 -75% |
| 回收周期 | 基准 × 2.0-3.0 | 回收时间翻倍/三倍 |
| 违约率 | 基准 × 3.0-4.0 | — |

#### 情景九：复合极端压力——Perfect Storm

| 参数 | 组合方式 | 典型取值 |
|------|---------|---------|
| 违约率 | 严重压力上限 | 基准 × 4.0-5.0 |
| 早偿率 | 严重压力下限 | 基准 × 0.4 |
| 回收率 | 严重压力下限 | 基准 × 0.4-0.5 |
| 利率 | 严重压力上限 | +300bp 至 +400bp |
| 集中度 | 极端值 | +100% |
| 宏观经济 | 严重衰退 | GDP增速-5pp 以上 |

### 4.2 压力情景矩阵（不同资产类型差异化参数）

| 资产类型 | 违约率乘数（严重压力） | 回收率乘数 | 早偿率乘数 | 特殊压力关注点 |
|---------|:------------------:|:--------:|:--------:|--------------|
| RMBS | 3.0-5.0× | 0.7-0.5× | 0.3-2.0× | 房价下跌+收入下降的联动效应 |
| Auto ABS | 3.0-6.0× | 0.5-0.3× | 0.5-1.5× | 二手车价格暴跌的回收率影响 |
| 消费贷款 | 4.0-7.0× | 0.4-0.2× | — | 无抵押、回收率极低，对违约率极敏感 |
| CLO | 2.0-4.0× | 0.6-0.4× | — | 行业集中度、最大债务人违约的连锁反应 |
| 融资租赁ABS | 3.0-5.0× | 0.6-0.4× | 0.5-1.5× | 租赁物处置价值和周期 |
| 信用卡ABS | 4.0-8.0× | 0.3-0.1× | 大规模波动的收益率冲击 | 信用循环+消费行为突变 |
| 应收账款/供应链 | 3.0-6.0× | 0.5-0.3× | — | 核心企业信用风险、债务人集中度 |
| CMBS/类REITs | 2.0-3.0× | 0.6-0.5× | — | 物业估值下跌、出租率下降 |
| 基础设施收费权 | 1.5-3.0× | — | — | 收入预测偏差、政策变化风险 |

## 五、资产池合格标准 Checklist

### 5.1 通用准入标准

| 序号 | 检查项目 | 合格标准（参考） | 核查方式 |
|:--:|---------|---------------|---------|
| 1 | 资产真实性 | 基础资产真实、合法、有效 | 合同、发票、付款凭证三单核对 |
| 2 | 无权属争议 | 无诉讼、仲裁、查封、冻结等权利限制 | 登记系统查询、法律尽职调查 |
| 3 | 可转让性 | 无合同限制转让条款（或已获豁免） | 逐一审阅底层合同 |
| 4 | 原始权益人合法持有 | 原始权益人为资产的合法所有权人 | 权属文件核查 |
| 5 | 无历史违约（选择） | 入池时点无逾期30天以上记录 | 原始权益人系统数据 |
| 6 | 资产合规性 | 不涉及违法违规领域（如违规房地产贷款） | 监管政策排查 |

### 5.2 资产池分散度标准

| 指标 | 计算方法 | 合格标准 | 优秀标准 |
|------|---------|---------|---------|
| 单一债务人集中度 | 最大债务人敞口 / 资产池总额 | ≤ 15%（CLO）/ ≤ 5%（消金） | ≤ 5%（CLO）/ ≤ 1%（消金） |
| 前5大债务人集中度 | 前5大债务人敞口合计 / 资产池总额 | ≤ 30%（CLO） | ≤ 20%（CLO） |
| 前10大债务人集中度 | 前10大债务人敞口合计 / 资产池总额 | ≤ 40%（CLO） | ≤ 30%（CLO） |
| 单一行业集中度 | 最大行业敞口 / 资产池总额 | ≤ 30%（CLO） | ≤ 20%（CLO） |
| 前3大行业集中度 | 前3大行业敞口合计 / 资产池总额 | ≤ 60% | ≤ 45% |
| 单一地区集中度 | 最大地区敞口 / 资产池总额 | ≤ 40% | ≤ 25% |
| 资产笔数 | 基础资产总笔数 | ≥ 20笔（CLO）/ ≥ 100笔（消金） | ≥ 100笔（CLO）/ ≥ 1,000笔（消金） |
| Herfindahl指数 | Σ(wi)² （wi为各债务人占比） | ≤ 0.05 | ≤ 0.02 |

### 5.3 资产池收益与期限标准

| 指标 | 计算方法 | 合格标准 | 说明 |
|------|---------|---------|------|
| 加权平均利率（WAC） | Σ(本金i × 利率i) / Σ(本金i) | 高于证券加权平均票面利率 + 税费 + 服务费，通常需 ≥ 1.5%-2.0% 超额利差 | 超额利差是重要内部增信 |
| 加权平均期限（WAM） | Σ(本金i × 剩余期限i) / Σ(本金i) | 与证券期限匹配；循环型 ≤ 36个月 | 期限错配产生再投资风险 |
| 加权平均账龄（WAS） | Σ(本金i × 已存续期i) / Σ(本金i) | ≥ 3-6个月 | 排除新发放贷款的季节效应 |
| 加权平均影子评级 | 按影子评级映射至数值后加权 | ≥ BBB-/Baa3 等效（CLO）；≥ BB等效（消金） | 资产整体信用质量门槛 |
| 加权平均LTV | Σ(贷款额i) / Σ(抵押物价值i) | RMBS: ≤ 70%；CMBS: ≤ 60% | 抵押率越低，抗跌能力越强 |

### 5.4 资产池禁止性特征

| 类别 | 禁止性特征 | 原因 |
|------|-----------|------|
| 信用 | 入池时已处于违约/逾期90天以上 | 初始损失过高 |
| 担保 | 基础资产无明确还款来源、纯信用无任何担保（对CLO类） | 回收率不可预估 |
| 期限 | 剩余期限 < 1个月（对循环购买型） | 有效存续期不足 |
| 集中度 | 单一债务人超过资产池总额的25%（CLO类） | 过度集中，AAA档要求更高的CE |
| 合规 | 资产涉及监管明令禁止的行业/领域 | 合规一票否决 |
| 关联 | 债务人/担保人与原始权益人存在重大关联关系且无公允定价 | 利益输送风险 |
| 合同 | 底层合同约定不可转让、或转让须经债务人书面同意而未获取 | 转让效力瑕疵 |

## 六、现金流瀑布（Waterfall）优先级规则

### 6.1 典型现金流分配顺序（优先级从高到低）

| 优先级 | 现金流支出项目 | 性质 | 说明 |
|:----:|---------------|------|------|
| 1 | 税、费（托管费、登记费、清算费等） | 税费 | 独立于证券偿付优先级，最为优先 |
| 2 | 资产服务机构报酬 | 服务费 | 服务费率通常为基础资产本息回收额的0.5%-1.5%/年 |
| 3 | 计划管理人/受托人报酬 | 管理费 | 含承销费（一次性）和管理费（年度约0.1%-0.3%） |
| 4 | 优先级证券当期应付利息 | 优先级利息 | 同优先级之间按比例分配 |
| 5 | 优先级证券当期应付本金 | 优先级本金 | 过手型按份额比例偿付；计划摊还型按期定额 |
| 6 | 中间级证券当期应付利息 | 中间级利息 | 如有中间级，按约定利率偿付 |
| 7 | 中间级证券当期应付本金 | 中间级本金 | 优先级全额清偿后才开始偿付中间级本金 |
| 8 | 次级证券当期应付收益 | 次级收益 | 在所有优先级和中间级本金清偿完毕后 |
| 9 | 次级证券本金偿还 | 次级本金 | 偿付完所有高优先级后取得剩余现金流 |

### 6.2 加速清偿事件触发后的瀑布调整

当加速清偿事件（Acceleration Event）触发后，现金流分配顺序发生如下调整：

| 优先级 | 调整后分配顺序 |
|:----:|--------------|
| 1 | 税、费（不变） |
| 2 | 资产服务机构报酬（不变） |
| 3 | 计划管理人报酬（不变） |
| 4 | **优先级证券本金（提前至利息前）** |
| 5 | 优先级证券当期应付利息 |
| 6 | 中间级证券当期应付本金 |
| 7 | 中间级证券当期应付利息 |
| 8 | 次级证券（剩余现金流） |

### 6.3 现金流模型中的关键缓冲机制

| 机制 | 运作原理 | 规模设定参考 |
|------|---------|------------|
| 流动性储备账户（Liquidity Reserve） | 在每期分配时预留一定金额至储备账户，应对下一期税费、服务费及优先级利息的支付风险 | 通常为下期优先级利息的1-3倍 |
| 服务转移储备 | 以备在服务商更换时支付过渡期服务成本 | 通常为3-6个月服务费 |
| 混同风险储备 | 防范服务商收取资金与自有资金混同的风险 | 按监管要求在收款账户中单独核算，或要求定期扫款 |
| 抵销风险储备 | 基础资产借款人享有对原始权益人的法定抵销权时的准备金 | 按有抵销风险部分的敞口计算 |
| 现金储备账户（Cash Reserve） | 存放于第三方托管银行的专用现金储备 | 通常占资产池总额的0.5%-1.5% |

### 6.4 两档优先级并存的瀑布规则

当产品设有优先级A档（通常AAA）和优先级B档（通常AA至A）时：

| 分配顺序 | 正常状态 | 优先A档加速清偿触发后（若只有A档触发事件） |
|:----:|---------|--------------------------------------|
| A档利息 | 最优先 | 最优先 |
| B档利息 | A档利息后 | A档利息后 |
| A档本金 | B档利息后 | 优先于B档利息 |
| B档本金 | A档本金后 | A档本金全额清偿后才轮到B档 |

# 尽调底稿归档标准与清单

> **适用范围**：A股IPO、再融资（定增/可转债）、债券（公募/私募）、并购等投行项目  
> **版本**：v1.0 | **生效日期**：2026年5月  
> **维护人**：ib-execution skill

---

## 一、DD-01至DD-07底稿体系详解

### 1.1 底稿分类总表

| 底稿编号 | 类别 | 核心内容 | 合规要求 | 保存年限 | 责任人 |
|----------|------|----------|---------|---------|--------|
| DD-01 | 公司主体 | 营业执照/章程/工商档案/股权结构 | 必备 | 永久 | 律师 |
| DD-02 | 财务账 | 三年一期财务报表/明细账/银行流水/科目余额表 | 必备+核查 | 永久 | 会计师 |
| DD-03 | 法律文件 | 合同/协议/决议/授权/证照许可 | 必备 | 永久 | 律师 |
| DD-04 | 诉讼/处罚 | 判决书/仲裁裁决/行政处罚决定书 | 必备 | 永久 | 律师 |
| DD-05 | 访谈纪要 | 管理层/客户/供应商/中介访谈记录 | 必备 | 永久 | 项目组 |
| DD-06 | 行业数据 | 行业报告/可比公司数据/政策文件 | 参考 | 5年 | 项目组 |
| DD-07 | 其他 | 计算底稿/分析表/问题追踪表 | 参考 | 5年 | 项目组 |

> **注**："必备"指项目内核/申报前必须齐全；"核查"指需经独立核查程序确认真实性。

### 1.2 DD-01 公司主体底稿细化清单

| 文件名称 | 份数要求 | 盖章要求 | 获取方式 | 注意事项 |
|---------|---------|---------|---------|---------|
| 营业执照（最新版） | 正本+副本各1 | 加盖公章 | 公司提供 | 核实经营范围是否涵盖主营业务 |
| 公司章程（现行版） | 1份 | 加盖公章 | 工商调档 | 需含全部修正案 |
| 历史沿革说明 | 1份 | — | 律师整理 | 含历次股权/资本变更 |
| 股权结构图（穿透版） | 1份 | — | 律师整理 | 穿透至最终受益人（自然人/国资委） |
| 工商登记基本信息 | 1份 | — | 国家企业信用信息公示系统 | 自行打印，加盖项目组章 |
| 商标/专利证书 | 全部 | — | 公司提供 | 仅收录与主营业务相关者 |
| 重要资质文件 | 全部 | 加盖公章 | 公司提供 | 特殊行业（医药/建筑等）重点关注 |
| 房屋租赁合同 | 全部 | — | 公司提供 | 核实租赁面积占比及到期时间 |
| 关联方清单 | 1份 | — | 公司提供+律师核查 | 含持股5%以上股东及其关联方 |

**归档路径示例**：
```
DD-01/
├── 01_营业执照/
├── 02_公司章程/
├── 03_历史沿革/
├── 04_股权结构/
├── 05_工商信息/
├── 06_知识产权/
└── 07_资质证照/
```

### 1.3 DD-02 财务账归档规范

| 子目录编号 | 内容 | 命名规范 | 覆盖期间 |
|-----------|------|---------|---------|
| 01_审计报告 | 三年一期无保留审计报告 | `YYYY审计报告_事务所名.pdf` | 三年一期 |
| 02_财务报表 | 资产负债表/利润表/现金流量表 | `YYYYQ[X]_报表.xlsx` | 三年一期每期 |
| 03_明细账 | 主要科目（收入/成本/应收账款等） | `YYYY_MXX_科目名称.xlsx` | 三年一期 |
| 04_银行流水 | 全账户银行对账单（逐月） | `银行名_账号_YYYYMM.pdf` | 开户至最近月末 |
| 05_银行余额调节表 | 全账户调节表 | `银行名_账号_YYYYMM_调节表.xlsx` | 三年一期每期末 |
| 06_科目余额表 | 全级次科目余额 | `YYYY12_科目余额表.xlsx` | 三年一期每年末 |
| 07_纳税申报 | 国税+地税+个税+社保 | `YYYY_MM_税种_申报表.pdf` | 三年一期 |
| 08_银行授信 | 授信合同+使用记录+担保合同 | `授信机构_额度_起止日期.pdf` | 当前有效 |
| 09_大额往来 | 与关联方/关键第三方的往来明细 | `YYYY_对手方_往来.xlsx` | 三年一期 |

**核查要点**：
- [ ] 银行流水的银行印章清晰可辨
- [ ] 银行流水账户数量与财务账核算的银行科目数量一致
- [ ] 银行余额调节表调节后余额与财务报表货币资金一致
- [ ] 审计报告与财务明细账数据钩稽一致

### 1.6 DD-05 访谈纪要归档规范

**命名格式**：`[访谈对象姓名]_[职位简称]_[YYYYMMDD].docx`

**目录结构**：
```
DD-05/
├── 01_管理层访谈/        # CEO/COO/CFO/董秘

├── 05_中介协调访谈/      # 律师/会计师/评估师沟通纪要
└── 06_监管/政府部门访谈/  # 如有
```

**访谈纪要要素清单**：
- [ ] 访谈日期、时间、地点
- [ ] 访谈对象姓名、职务、联系方式
- [ ] 访谈人姓名
- [ ] 主要问题及回复摘要
- [ ] 访谈对象签字或盖章（纸质版）
- [ ] 访谈问卷底稿（含电子版）

## 二、底稿完整性标准

### 2.1 各项目类型底稿数量基准

| 项目类型 | 最低底稿数量 | 核心关注领域 | 法规依据 |
|---------|------------|-------------|---------|
| IPO（主板/科创板/创业板） | ≥500份 | 收入确认/关联交易的完整证据链 | 证券法第19条 |
| 再融资（定增） | ≥200份 | 募投项目+财务数据一致性 | 发行监管问答 |
| 债券（公募公司债/企业债） | ≥150份 | 信用分析+担保/抵押权利完整性 | 公司债券条例 |
| 债券（私募） | ≥100份 | 核心文件+简化核查 | 私募办法 |
| 并购（重大资产重组） | ≥300份 | 财务+法律+业务三线并行 | 重组办法 |

> **注**：最低数量仅供参考，底稿完整性以内核意见及监管审核要求为准。

---

### 2.2 底稿核查 checklist（内核前必查）

**DD-01 公司主体**
- [ ] 营业执照正副本齐全，在有效期，经营范围覆盖主营
- [ ] 公司章程工商调档版与公司提供版内容一致
- [ ] 历史沿革无重大遗漏，股权变动有完整法律文件支撑
- [ ] 穿透后股权结构与工商登记一致

**DD-02 财务账**
- [ ] 银行流水全账户覆盖（须覆盖发行人及纳入合并报表的全部子公司）
- [ ] 审计报告三年一期无断档，签字盖章齐全
- [ ] 银行流水与财务账货币资金科目余额核对一致
- [ ] 应收账款/收入确认有完整的穿行测试底稿

**DD-03 法律文件**
- [ ] 前5大客户合同（前十大另计）合同要素完整（签章/金额/期限）
- [ ] 前5大供应商合同合同要素完整
- [ ] 全部银行授信合同已归档，含担保/抵押合同及登记证明

**DD-04 诉讼/处罚**
- [ ] 检索覆盖全部子公司（含已注销/转让）
- [ ] 检索结果与公司提供的重大诉讼清单一致
- [ ] 期后事项：至申报前持续关注新增诉讼/处罚

**DD-05 访谈纪要**
- [ ] 管理层访谈覆盖全部董事/监事/高管及核心技术人员
- [ ] 客户访谈覆盖前5大客户（含收入占比测算支撑）
- [ ] 访谈纪要有访谈对象签字或录音/录像确认

---

### 2.3 底稿质量评级标准

| 等级 | 定义 | 特征 | 处置要求 |
|------|------|------|---------|
| **A级** | 完整+可追溯+逻辑清晰 | 全部满足归档规范，引用关系清晰 | 可直接用于内核 |
| **B级** | 完整但整理较乱 | 有文件但索引不清，查找耗时 | 内核前整理至A级 |
| **C级** | 部分缺失 | 关键文件缺失≥5% | 暂停内核，补充完善 |
| **D级** | 不合格 | 缺失核心文件或存在造假迹象 | 全面整改，必要时重新申报 |

## 三、底稿交叉索引体系

### 3.1 索引编号规则

**格式**：`DD-[XX]-[类别]-[NNNN]`

| 字段 | 说明 | 示例 |
|------|------|------|
| DD-[XX] | 底稿大类（01-07） | DD-02 |
| [类别] | 子类：FIN=财务，LEG=法律，OPR=运营 | FIN |
| [NNNN] | 流水序号（4位） | 0007 |

**完整示例**：`DD-02-FIN-0007` = DD-02财务类第7号索引文件

**分类对照**：
| 类别代码 | 含义 | 涵盖范围 |
|---------|------|---------|
| FIN | 财务 | 财务报表、审计报告、银行流水、科目余额 |
| LEG | 法律 | 合同、决议、证照、诉讼文书 |
| OPR | 运营 | 访谈纪要、行业数据、生产经营数据 |
| TAX | 税务 | 纳税申报、税务报告、完税凭证 |
| ASR | 资产 | 房产证、土地证、设备清单、无形资产 |

---

### 3.2 快速定位表（常用路径）

| 查找目标 | 归档路径 |
|---------|---------|
| 某客户销售合同 | `DD-03/01_重大合同/客户合同/[客户名].pdf` |
| 某银行账户流水 | `DD-02/04_银行流水/[银行名]_[账号]_[年月].pdf` |
| 某高管访谈纪要 | `DD-05/01_管理层/[姓名]_[职位]_[日期].docx` |
| 某科目计算底稿 | `DD-07/01_计算底稿/[科目]_[期间].xlsx` |
| 某诉讼文书 | `DD-04/[主体名]_[文书类型]_[案号].pdf` |
| 某审计报告 | `DD-02/01_审计报告/[年份]审计报告_[事务所名].pdf` |
| 房产/土地权属 | `DD-03/06_土地使用权/` 或 `DD-03/07_房产/` |

## 四、版本控制规范

### 4.1 版本命名规范

**格式**：`文件名_v[M].[N].[YYYYMMDD].[状态]`

| 字段 | 说明 | 示例 |
|------|------|------|
| v[M].[N] | 主版本.次版本 | v1.3 |
| YYYYMMDD | 更新日期 | 20260301 |
| 状态（可选） | draft/draft2/review/clean | draft |

**示例**：
```
招股书_第一章_公司概况_v1.3.20260301.draft.docx
尽职调查报告_v2.0.20260315.review.docx
财务尽调报告_v1.0.20260320.clean.docx
```

> **建议**：终稿版本统一命名为`v1.0.最终版`并移除draft标记。

## 五、底稿归档流程与质量控制

### 5.1 归档时间要求

| 节点 | 要求 |
|------|------|
| 项目启动 | 建立底稿目录框架（DD-01至DD-07全目录） |
| 尽职调查期间 | 随获取随归档，不得积压 |
| 内核前1个月 | 完成全部底稿整理及索引编制 |
| 内核会 | 纸质底稿已整理完毕，电子底稿已刻盘 |
| 申报前 | 最终版底稿归档，移交档案室 |

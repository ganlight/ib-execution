## Token 预算原则

> ⚠️ **核心原则：按需加载，不要一次性加载所有文件！**

| 原则 | 说明 |
|------|------|
| **主入口轻量化** | `SKILL.md` 仅做路由，轻量 (< 2,000 tokens) |
| **场景驱动** | 每个场景只加载 3-5 个相关文件 |
| **典型单次消耗** | 3,000 - 5,000 tokens（路由 + 场景文件） |
| **META 按需加载** | META 方法论文件只在需要方法指导时加载 |
| **参考文件延迟加载** | 参考知识库文件在处理具体问题时才加载 |

**错误做法 ❌：**
- 一次性加载所有 SKILL 文件（47个 = ~80,000 tokens）
- 每个问题都全部加载 META 方法论
- 把所有参考文件当作上下文塞入

**正确做法 ✅：**
- 先让 SKILL.md 匹配关键词决定路由
- 只加载当前场景必需的文件
- 用户追问时再按需加载更多文件

```
Phase 1 ──── IPO全流程掌握 ──── 场景1-4
  │  新项目冷启动 → 尽调 → 写招股书 → 监管反馈回复
  │  覆盖投行最核心的80%工作场景

  │
Phase 2 ──── 加入并购能力 ──── +场景5
  │  理解交易结构设计、估值方法、重组审核
  │  适用：承做人员扩展至并购重组业务
  │
Phase 3 ──── 强化项目管理 ──── +场景6
  │  掌握时间表管理、中介协调、里程碑追踪
  │  适用：向项目负责人角色过渡
  │

  │  柳叶刀工作法、交叉校验、技能演化
  │  适用：追求极致效率和质量控制
  │
  └── 🏆 全栈投行AI承做能力
```

|-------|---------|
| Phase 1 | 能独立用 QClaw 完成IPO冷启动、尽调规划、招股书章节撰写、反馈回复起草 |

## 三、RAG 升级规则（Escalation Rules）

### 🔴 Red Issue 升级路径

```
触发条件（满足任一）：
  · 构成上市法律障碍（如VIE合规、实际控制人认定不清）
  · 可能影响报告期财务数据的重大调整（收入确认、减值等）
  · 涉及实际控制人/董监高的重大诉讼或处罚
  · 审计师出具保留意见/无法表示意见

升级路径：
  Issue发现 → 24h内通知项目负责人 → 48h内召开专题会议
  → 制定解决时间表 → 每周跟踪 → 影响判断升级至投行管理层

决策层级：
  是否构成上市障碍 → 保荐人内核委员会判断
  是否继续推进 → 发行人与投行联合决策
```

### 🟡 Amber Issue 升级路径

```
触发条件（满足任一）：
  · 客户集中度/关联交易等需重点披露事项
  · 社保/税务等历史合规问题，量化影响可控
  · 知识产权、商标等在途申请

升级路径：
  Issue发现 → 纳入周例会跟踪 → 责任人确认解决时间
  → 连续2周无进展自动升级为Red → 项目负责人介入
```

### 🟢 Green Issue 处理流程

```
触发条件：
  · 影响轻微、可快速解决的形式性/程序性问题
  · 已解决事项的跟踪归档

处理流程：
  记录在案 → 按计划关闭 → 无需升级
  → Resolved后保留归档备查（联交所/证监会问询时可引用）
```

## 四、Issue 状态定义

| 状态 | 含义 | 更新规则 |
|:---|:---|:---|
| **Open** | 问题已识别，尚未开始解决 | 新Issue创建时默认状态 |
| **In Progress** | 解决方案执行中 | 开始行动后更新，附进度说明 |
| **Resolved** | 已解决，待确认归档 | 方案执行完毕后更新，附支持文件 |
| **Closed** | 经确认归档完毕 | 项目负责人确认后关闭 |

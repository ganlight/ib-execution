## 二、问题状态流转图

### 2.1 状态定义

| 状态 | 含义 | 进入条件 | 退出条件 |
|:---|:---|:---|:---|
| **Open** | 问题已识别，尚未开始解决 | 新Issue创建时默认状态 | 开始执行整改方案 |
| **InProgress** | 解决方案执行中 | 整改方案已确定 + 责任人已确认 + 开始落地 | 方案执行完毕 |
| **PendingReview** | 已提交解决，等待复核 | 方案执行完毕 + 责任人提交复核材料 | 复核人确认关闭条件满足 |
| **Closed** | 经确认归档完毕 | 复核人确认关闭条件全部满足 + 支持文件齐全 | |
| **Reopen** | 已关闭问题重新激活 | 已关闭的Issue再次出现问题或发现未彻底解决 | 重新评估后回到Open或InProgress |

### 2.2 状态流转规则

```
                    ┌─────────────────────────┐
                    │                         │
                    ▼                         │
  Open ──→ InProgress ──→ PendingReview ──→ Closed
    ▲           │              │               │
    │           │              │（复核不通过）  │
    │           │              ▼               │
    │           └──────→  (回到执行)  ─────────┘
    │                                             │
    └──────────── Reopen ◀───────────────────────┘
                     │
               (重新评估后)
```

### 2.3 流转时限要求

| 流转 | 时限要求 | 超时处理 |
|:---|:---|:---|
| Open → InProgress | 识别后5个工作日内 | 自动升级至项目执行负责人介入 |
| InProgress → PendingReview | 按目标完成日期 | 逾期1周→🟡预警；逾期2周→🔴升级、专题讨论 |
| PendingReview → Closed | 提交复核后5个工作日内 | 催促复核人，超10日未回复→升级 |
| Closed → Reopen | 随时（发现重复/反弹） | 重新评估后24小时内更新状态 |

## 四、RAG 升级与降级规则

### 4.1 RAG判断标准总表

| 等级 | 判定标准 | 对申报的影响 | 时限要求 |
|:---:|:---|:---|:---|
| 🔴 Red | ①构成上市法律障碍；②影响报告期财务数据重大调整（>净利润10%）；③涉及实控人/董监高重大诉讼/处罚；④审计师出具保留意见/无法表示意见 | 构成上市/申报障碍 | 24h内通知项目负责人，48h内专题会议 |
| 🟡 Amber | ①客户集中度/关联交易等需重点披露（但可解决）；②社保/税务等历史合规问题（量化影响可控）；③知识产权/商标等在途申请；④会计政策判断差异（但可协商一致） | 需在文件中重点披露或补充说明，但可解决 | 纳入周例会跟踪，连续2周无进展自动升级🔴 |
| 🟢 Green | ①形式性/程序性/文档性问题（不影响结论）；②已解决事项的例行跟踪归档（可随时关闭）；③信息披露措辞的优化（不涉及实质内容） | 对申报无实质影响 | 正常处理即可 |

### 4.2 🔴 Red Issue 升级路径

```
升级路径：
  Issue发现 → 24h内通知项目执行负责人
           → 48h内召开专题会议
           → 制定解决时间表
           → 每周跟踪（直至降级或关闭）
           → 如1个月无进展 → 升级至项目总负责人（MD）
           → 如解决可能性<50% → 升级至保荐人内核/风控 → 评估是否终止项目

决策层级：
  是否构成上市障碍 → 保荐人内核委员会判断
  是否继续推进 → 发行人与投行联合决策（需记录在案）
```

### 4.3 🟡 Amber Issue 升级路径

```
升级路径：
  Issue发现 → 纳入周例会跟踪
           → 责任人确认解决时间
           → 每周更新进度
           → 连续2周无进展 → 自动升级为🔴
           → 项目负责人介入 → 制定加速计划

降级条件：
  Amber → Green：方案的可行性确认 + 执行进度按计划推进 + 无新增风险
```

### 4.4 🟢 Green Issue 处理流程

```
处理流程：
  Issue发现 → 登记在案
           → 按计划解决
           → 无需周会跟踪（仅在统计表中显示）
           → 满足关闭条件 → 提交复核 → Closed
           → Closed Issue 保留归档备查

> 注意：Green Issue 连续2周无进展也建议关注，但无需升级
```

| 升级级别 | 联系人 | 角色 | 联系电话 | 升级时限 |
|:---|:---|:---|:---|:---|
| 第一级：项目执行负责人 | 【姓名，D/VP】 | 日常管理 | 【】 | 问题发现后24h |
